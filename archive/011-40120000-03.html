This page contains the process for reporting telephony faults.

Reporting telephony faults to STaWM

This table describes the process for telephony fault logging and reporting. This implements a consistent fault reporting process that ensures all relevant information is collected and provided to STaWM, for timely resolution and to provide the business with consistent escalation of unresolved faults.

Step

Action

Role

1

Information required to log a fault + Read more ...

Before logging a fault, check the following:

  • Are the cables plugged in?
  • Is the computer working?
  • Is anyone else experiencing the same problem?

When logging a fault the following details are required:

  • the extension number of the faulty handset
  • the MAC address of the phone, this is located underneath the phone next to the MAC embossing
  • details of any error messages

Service Officer

2

Notify of fault + Read more ...

Notify team coach or Manager of the fault.

Service Officer

3

Identify if issue is widespread + Read more ...

Check if any other Service Officers are affected.

Team coach or Manager

4

Troubleshoot issue and report fault + Read more ...

Using MySupport access the Services Australia Workspace – DHS Workspace article, troubleshoot the issue to try and locate a fix.

If unable to locate a fix use the link at the bottom of the article titled ‘Services Australia Workspace; – DHS workspace issues’ and register the issue.

Notify STaWM, in the relevant Microsoft Teams Chat, of the fault number once the MySupport form is lodged.

Note: faults occurring on weekends or after 17:00 AEST/AEDST are to be escalated to the relevant Service Support Manager.
