Helping users reset their password

Table 2

Step

Action

1

Profile - My Details + Read more ...

The user logs on to their PRODA account and selects:

  • Profile from the right hand side of the header
  • Update next to Password in the Security section

2

Change your password + Read more ...

The user:

  • enters the current password
  • enters and verifies the new password
  • selects Next

Passwords should be known only to the user and are case-sensitive. Strong passwords are difficult to guess and must contain:

  • at least 10 characters
  • a mix of upper and lower case letters, and
  • at least 1 number or special character

The user should try not to include:

  • repeated characters
  • personal information
  • a password they have previously used, or are using elsewhere

Note: if the password has been previously used, an error message will display - The new password entered matches a password already used. Please enter another password.

The system will advise them about how strong the password is considered to be as they enter it to help guide them to a password that is appropriate.

3

Confirm your changes + Read more ...

The user selects Yes to confirm the details.

Successful message displays Your account has been successfully updated.

Procedure ends here.

Helping users amend their security questions

Table 3

Step

Action

1

Profile - My Details + Read more ...

The user logs on to their PRODA account and selects:

  • Profile from the right hand side of the header
  • Update next to Security Questions and Answers in the Security section

2

Change your security questions and answers + Read more ...

Advise the user to change their security questions by selecting a question from the dropdown list and typing the answer in the Answer field.

If the user just wants to update the answers, they just enter the new answer to the question in the Answer field.

The user selects Next.

The user must select 3 pre-determined questions from the dropdown boxes and then type their answer to each question. Their answers are not case sensitive.

The answers should be:

  • memorable - something they will remember
  • constant - not changed over time
  • safe -not be something that is easily guessed or researched online (for example via social media)
  • different - not the same answer as other questions

Their answer must be at least 3 characters long and cannot be consecutive letters or numbers (for example, 'abc' or '123').

3

Confirm your changes + Read more ...

The user selects Yes to confirm the details.

Successful message displays Your account has been successfully updated

Procedure ends here.

Advise the user:

'Cancelling your account

You can only self-recover your account up to the date of cancellation or on the same day of cancellation.

From the date after cancellation you will no longer be able to access your PRODA account or any linked services.

You will, if within 90 days from the date of cancellation, be able to re-activate your account by calling Services Australia. (Do not create a new account within the 90 days. You will not be able to use your identity documents previously used).

After 90 days from the date of cancellation you will need to create a new account if you wish to access any services within PRODA.'

8

Finalise + Read more ...

The user selects the Search button.

Does the system advise the number does not match?

  • Yes, ask the user to check the number is correct and try again. If the number still won't match, go to Step 9
  • No, when the search for the identifier is successful the user will see the green tick next to the added identifiers:
    • Note: ask the user to check that the search has returned all their identifiers stored with Medicare. Access to HPOS is determined based on the identifiers found during the search and will be limited by the type of identifiers returned within the list. (For example: Ahpra number may not be stored in all databases and only return the HPI-I and not the users Medicare provider numbers. Ahpra numbers can only be used to search Services Australia data bases and will not display in the list.) It is also important to note that a HPI-I number will not give a provider full HPOS access without the users Medicare provider numbers linked as well
    • If all identifiers have been linked, they then select Save
    • A successful message displays The Health Professional Online Services has been successfully added to your account. Congratulations. You can now use your PRODA account to access your services
    • If the user wants to link more identifiers to the HPOS service, they select Link identifiers or select Go to service on the Health Professional Online Services tile that will now display in My linked services
    • Procedure ends here

9

Number does not match - request help + Read more ...

It is possible that there are some discrepancies in the data held about the user in different areas of the agency. For example, the personal information they have entered for this account may be different to the data saved against their Medicare provider number registration due to a change in circumstances or timing. Therefore, it may not be possible to link all numbers in every case.

  • If the Service Officer can access the data source, check the information. If unable to determine why the identifier is not matching, advise the user to select the checkbox to request help from a Service Officer
  • Alternatively, if a match cannot be found for the number yet the user is sure they entered the number correctly and it has been issued to them, advise them to select the checkbox to request help from a Service Officer

They should add a short explanation in the comments box advising what kind of number it is or which area of Services Australia issued it (for example, Medicare provider number), then select Save.

Note: 'Flagged Identifiers' are automatically escalated to Tier 3 for processing.

Note: users can only add identifiers that belong to them. If the user is entering the Registration Authority (RA) number from their Individual Public Key Infrastructure (PKI) certificate, advise them this is not their Responsible Officer (RO) or Organisation Maintenance Officer (OMO) number and they will need to contact either their OMO or the HI Service to obtain their correct identifier.
