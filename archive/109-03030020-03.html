Step

Action

1

Request a case and create a customer case folder + Read more ...

In Customer First:

  • go to Inbox
  • change Agent Box to Work Management
  • select Ready for Work

This will allocate a work item in the Result List.

Select the work item to enter the customer's record. Go to the AAT1 Request DOC located on the Doc List (DL) screen.

Check the DOC for any relevant information, including:

  • the Appeal (APL) number
  • the AAT1 Review Number (AMS)
  • if the appeal is a Priority request (complete within 7 days)
  • if the appeal is a Subject Matter Expert (SME) finalised decision
  • any other relevant information

Note: Multi-Party reviews see Table 2.

On the customer’s Activity List (AL) screen:

  • Select 'O' next to the relevant DOC to display the Transfer Activity (TA) screen
  • On the TA screen, key the following details:
    • To Regional Office: key 'ARO'
    • To Workgroup: key 'AAT1'
    • To Position: staff name (as it appears in the User Profile)

This will assign the activity in AL. The activity must remain open.

Create a customer case folder in the relevant Individual Cases folder located in the R drive. Each AMS needs a separate folder.

Save the folder using the following naming conventions:

  • ‘SURNAME CRN APL AMS’ for standard cases
  • ‘PRIORITY SURNAME CRN APL AMS’ for priority cases

The due date is

  • 28 days after the date of request, or
  • 7 days after the date of request for priority cases

The date of request is the date listed at the bottom of the scanned AAT1REQ.

Utilise ERoD checklist via Resources > Templates and Checklists.

Save the checklist using the following naming conventions:

  • ‘SURNAME CRN APL AMS DUE DATE EC’ for standard cases
  • ‘PRIORITY SURNAME CRN APL AMS DUE DATE EC’ for priority cases

Note: Multi-party reviews have different naming conventions, see Table 2, Step 6.

2

Locate AAT1 Request and Authorised Review Officer (ARO) decision + Read more ...

In Customer First:

  • Go to the customer's record
  • Choose Document Tools from the Quick Links menu
  • Find and open the AAT1REQ
  • Match the AMS on the AAT1REQ to the AAT1REQ DOC on the customer's DOC List (DL)

If the AAT1REQ cannot be found:

  • contact the officer who created the AAT1 REQUEST DOC use the Email template Decision Statement Required in Resources
  • if the officer is not available, contact the Workload Manager via the AAT1 Appeals Mailbox
  • email ERoD Mailing team

Find the relevant ARO decision (ARODEC) and ARO Notes (AROEVD) in Document Tools. Make sure the APL number matches the AAT1 REQUEST DOC.

Note: for AAT1 reviews of Subject Matter Expert (SME) decisions, include:

  • Q351 or Q449 issued by the SME, and
  • the SME DOC

Check the ARO Decision, ARO Notes and SME DOC for all evidence used to make the decision, including:

  • Evidence listed in the Information Considered section of the ARO decision statement
  • Other evidence mentioned in the ARO decision statement
  • ARO notes detailing contact with the customer and any other evidence
  • Evidence used by the SME located in the SME/Decision Maker DOCs
  • Evidence received after the review was finalised
  • For Disability Support Pension (DSP) medical rejections and cancellations:
    • Evidence listed in Job Capacity Assessment (JCA), Employment Services Assessment (ESAT) and Assessment Services Recommendation reports,
    • Health Professional Advisory Unit (HPAU), Disability Medical Assessment (DMA) and other internal medical documents.
  • For DSP medical rejections, all historical medical evidence and previous DSP medical AAT1 decisions. If not available on the customer's record, see ERoD AAT Decision Location Request Guide on the Resources page.

Collect all relevant scanned evidence. Make sure:

  • There is a scan cover page for each document
  • To remove irrelevant pages from the scan before to saving
  • To save details obtained from third party records in a subfolder(s) in the customer case folder
  • Create subfolder(s) based on the number of third parties, using the following naming conventions:
    • ‘THIRD PARTY 1’,
    • ‘THIRD PARTY 2’, etc.
  • Save all scanned third party evidence in the relevant subfolder

For more information, see Treatment of Evidence in Section 37 Hearing Papers and Treatment of Poor Quality Documents in s37 Hearing Papers on the Resources page.

Note:

  • Australian Transaction Reports and Analysis Centre (AUSTRAC) data may be used for intelligence purposes. Do not release this to the AAT or any parties to the review. See the Resources page for details
  • Remove duplicate documents from Hearing Papers, unless identified as relevant by the ARO or SME. See Treatment of Duplicates in s37 Hearing Papers on the Resources page
  • Medical reports and certificates not available on the customer’s electronic record may be listed in the Medical Conditions Summary Screens (MCSS) screen. Check the Medical Conditions details (MC) and Additional Medical Condition (MCA) screen
  • The ARO may have provided documents that could not be scanned to the customer's record to the EROD Appeals branch mailbox. For example, Customer Record Access Monitor (CRAM). Check for these emails in:
    EROD.APPEALS.REVIEW.BRANCH mailbox > Team Folders > 03. Documents from AROs

Is all evidence relevant to the review available on the customer and/or third party’s electronic record?

  • Yes,
    • Go to Step 8
    • If some of the evidence is not in English, go to Step 6
    • If the appeal is a result of an Intelligence and Investigations Branch case, go to Step 7
  • No,
    • If help is needed from the ARO, go to Step 3
    • If the customer’s paper file is needed, go to Step 4
    • If the missing information is an internal social worker report or case notes, go to Step 5

3

Assistance or clarification required from ARO + Read more ...

Send an email to the ARO outlining the clarification or support required

From the EROD Appeals branch mailbox:

  • use the Review Evidence Required template in Resources>Emails
  • send an email to the ARO
  • CC in the ARO Team AD mailbox and the EROD Appeals branch mailbox

The ARO has 24 hours (one working day) to provide a response.

If no response is received:

  • forward the initial email to the ARO
  • use the Review Evidence Required template in Resources>Emails
  • send an email to the ARO
  • CC in the ARO Team AD mailbox and the EROD Appeals branch mailbox
  • do not change the subject line

If the ARO advises the information can be accessed on the customer’s paper file, go to Step 4.

If the ARO advises the information is incorrect, unavailable, or recorded in error, note on the ERoD checklist. Prepare an AAT1 cover letter. Go to Step 11.

Move the emails to ERoD mailbox > Team Folders > 01 Review Evidence Required (RER) > RER Finalised.

If the review involves:

  • Information from Social Work Services Branch, go to Step 5
  • Untranslated documents, go to Step 6
  • An Intelligence and Investigations Branch case, go to Step 7
  • None of these, go to Step 8

4

Request paper files and scan to customer record + Read more ...

If the information is in the customer or third party paper file, get the file from Records Management Unit (RMU).

  • Go to My Support
  • Search for ‘Create, Transfer or Retrieve a Corporate File’
  • Choose the following options from the dropdown menus:
    • What would you like to do – Retrieve a Record
    • What type of file do you want to receive - Centrelink Customer File
    • What is the Customer Reference Number - key Customer’s CRN
    • Please provide the address you would like to send the record to - key site address, including floor
  • Include additional information about the file if required. Clearly document:
    • the file is required for an AAT1 review
    • where the case is a priority request an overnight courier is required

Note: there may be multiple paper files required such as Partner, Trust/Company, Investigation or Assuree/Assurer.

  • Update the AAT1 REQUEST DOC with the date of the RMU file request
  • Wait for the file (2 days for a priority case, 7 days for a standard case)
  • Proceed with all other steps while waiting for the file

The file may be

  • uploaded to the customer’s electronic record by RMU, or
  • delivered to ERoD

Scan the relevant documents from the paper file to the customer’s record as AROEVD. Return the paper file to RMU.

Note: do not upload Investigation files to the customers record.

Check the paper file/s:

  • Annotate the AAT1 request DOC with:
    • ‘RMU file/s have been checked and relevant evidence identified and actioned’
  • If the hearing papers have already been sent to the AAT, see Table 3 to send documents as Additional Information
  • If information cannot be located in the paper file, prepare a cover letter. Go to Step 10.

If the appeal involves:

  • Information from Social Work Services Branch, go to Step 5
  • Untranslated documents, go to Step 6
  • An Intelligence and Investigations Branch case, go to Step 7
  • None of these, go to Step 8

5

Relevant social worker reports and notes + Read more ...

The Social Work Services Branch

  • provide Social worker reports/case notes to the ERoD team
  • consider documents for non-disclosure and give details if requested

Send the following to Social Work Services Branch for non-disclosure consideration:

  • All DOCs created or annotated by a Social Worker
  • Social Work reports
  • Social Work Information System (SWIS) notes, and
  • DOCs containing sensitive information

See Requesting non-disclosure of information during an AAT first review.

Send an email from the EROD Appeals branch mailbox to the Social Work Services Branch

  • Use the SWO Requests template in Resources>Emails
  • Complete a separate table for each document
  • CC in the EROD Appeals branch mailbox
  • Wait for a response
  • Send a follow up email if necessary after 48 hours, replying to the initial email to preserve the email trail
  • File the response and associated emails in:
    ERoD mailbox > Team Folders > 06. Social Worker > Social Worker - Finalised folder

If the appeal involves:

  • Untranslated documents, go to Step 6
  • An Intelligence and Investigations Branch case, go to Step 7
  • None of these, go to Step 8

6

Untranslated documents + Read more ...

Translate all documents in a language other than English. Do not send untranslated documents.

For evidence not yet translated, request an urgent translation from the Language Service Unit. Follow the process outlined in Translation of documents. The Language Services Unit will translate the document and upload it to the customer's record within 5 business days.

If the hearing papers cannot be held until after the translation has been completed, prepare an AAT1 cover letter. Go to Step 11.

Send the translated evidence as additional information. See Table 3.

Does the appeal involve an Intelligence and Investigations Branch case?

  • Yes, go to Step 7
  • No, go to Step 8

7

Intelligence and Investigations Branch case + Read more ...

If the decision under review was made by the Intelligence and Investigations Branch:

  • check for pending or completed prosecution action, and
  • advise the AAT

Send an email to the Intelligence and Investigations Branch from the EROD Appeals branch mailbox:

  • Use the Request for Prosecution information template in Resources>Emails
  • Complete the email and attached Word template
  • CC in the EROD Appeals branch mailbox
  • Await response email
  • Send a follow up email if needed (after 48 hours, 24 hours for a priority case)
  • When received, move the initial request and associated emails to:
    ERoD Mailbox > Team Folders > 07. I&I Prosecution Action Finalised folder
  • If the Intelligence and Investigations Branch advise of pending or completed prosecution action, insert the Word template in the hearing papers behind the AAT1REQ
  • Paginate and apply redactions as normal

8

Collect relevant information from the customer’s record and check for sensitive information + Read more ...

Australian Transaction Reports and Analysis Centre (AUSTRAC) data may be used for intelligence purposes. Do not release this to the AAT or any parties to the review. See the Resources page for details.

Relevant screens

  • Find the relevant Reference Guide in the Master Reference Guide in Resources
  • Use the ERoD Multi-tool to collect relevant screens listed on the reference guide from the customer record
  • Include other relevant screens:
    • listed in the ARO Notes, or
    • that contain information referred to by the ARO or SME

Save screens collected from third party records in the relevant third party subfolder.

Seek help from nominated support staff in the first instance, before requesting help from the ARO.

See the Resources page for:

  • Order of Screen Prints within Section 37 Hearing Papers
  • Collection of Related Explanation, and
  • ESS web POS task card policies

Relevant documents

Collect all relevant DOCs using the ERoD Multi-tool:

  • Collect DOCs in date order, oldest to newest
  • Include Document List (DL)/Archiving and Culling Engine (ACE) summary where required
  • Save DOCs taken from a third party record in the relevant third party subfolder

Relevant letters

Identify and collect relevant letters from HS, ACE, Customer Archive Retrieval (CAR), Outbound Correspondence and Communication History. Where letters are available in multiple locations letters should be sourced from ACE to make sure they are formatted correctly.

For debt appeals, collect all letters, SMS and email messages sent to the customer during the debt period. Also collect the last letter sent to the customer about the relevant payment before the debt period began, if available.

Note: where ACE letters are unavailable, consult your Team Leader. If hearing papers must be sent to the AAT before the ACE letters are available, note on the ERoD checklist. Complete an AAT1 cover letter, go to Step 11. When the letters are available, send as Additional Information. See Table 3.

Sensitive Information

For potentially sensitive information, prepare a section 35 application for Non-Disclosure Order. See Requesting non-disclosure of information during an AAT first review.

Sensitive information may include:

  • DSP medical qualification reviews
  • Job Capacity Assessment (JCA)
  • other internal medical reports that may not to be released to the customer
  • Social worker DOCs, case notes and reports
  • Other sensitive information that might cause harm to the customer or third parties

Note: see Mandatory Section 37 Hearing Paper Inclusions for mandatory inclusions on the Resources page.

9

Combine all evidence into the hearing papers document + Read more ...

Select all files using [Ctrl] + [A] in the customer folder. Combine files in Adobe Acrobat.

Drag each document into the correct order as follows:

  • AAT1 cover letter (if applicable)
  • AAT1 request email and accompanying documents
  • For ARO decisions include:
    • ARO decision statement, and
    • ARO otes
  • For SME decisions include:
    • Q351 or Q449 letter
    • SME decision DOC
    • ARO decision, and
    • Notes (if applicable)
  • Scanned evidence from oldest to newest as per the date of receipt
  • Screen prints in order, as per Master Reference Guide in Resources
  • DOCs collected from different systems:
    • in order from oldest to newest
  • Letters collected from different systems:
    • in order from oldest to newest
    • by the date of the first letter collected in each system
  • Place third party information in the same order, appearing directly after customer information
    • For example, third party scanned evidence is placed immediately after the customer scanned evidence

When all documents are in the correct order, combine files to form 1 document.

  • Save the newly created document using the following naming conventions:
    • ‘SURNAME CRN APL AMS DUE DATE’ for normal cases, or
    • ‘PRIORITY SURNAME CRN APL AMS DUE DATE’ for priority cases
  • From Adobe Acrobat > Tools - Action Wizard > run Step 1 to:
    • Convert colour to grey scale
    • Apply text recognition to the file
    • Rotate landscape pages to left to portrait mode
      Note: the top of landscape page when rotated should be on the left side of the page
    • Search for blank pages and prompts to confirm deletion. See Treatment of Blank Pages in s37 Hearing Papers on the Resources page
    • Shrink pages to allow for pagination
    • Save the file as ‘SURNAME CRN APL AMS DUE DATE_1’
  • Re-save the file as ‘SURNAME CRN APL AMS DUE DATE’
  • Apply relevant bookmarks to the file on the first page of that section:
    • ARO DECISION (for SME decisions this will be on the Q351 or Q449 letter or SME DOC)
    • EVIDENCE
    • SCREEN PRINTS
    • DOCS
    • LETTERS/NOTICES
  • Place an additional sub-bookmark 'THIRD PARTY' at the commencement of the third party information for each of the above bookmarks
    • Where there are multiple third parties, number each sub-bookmark 'THIRD PARTY 1', 'THIRD PARTY 2', etc.
  • Re-save the file

10

Check the hearing papers for privacy + Read more ...

Mark the hearing papers for privacy. See the Resources page for:

  • Guidelines for preparation of documents for the AAT, and
  • Redactions within Section 37 hearing papers

To mark irrelevant information for redaction:

  • select red for the redaction outline colour
  • mark the specified content for redaction
  • do not apply the marked redactions

For any material identified for non-disclosure under section 35:

  • see Requesting non-disclosure of information during an AAT first review to mark the sensitive content, and
  • prepare the section 35 application
  • re-save the file

11

Create an AAT1 cover letter + Read more ...

If required create a cover letter for the AAT using the relevant AAT cover letter templates. See AAT1 Cover Letters and Cover Letter Process guide on the Resources page.

Include any errors identified in the hearing papers.

This may include:

  • date/detail discrepancies (for example in ARODEC, AROEVD, JCAR)
  • unavailable evidence
  • evidence listed in error
  • poor quality documents

If ACE letters are delayed, send the hearing papers without these letters.

  • if the letter codes and dates for the hearing papers are not known, prepare AAT1 Cover Letter ‘ACE NOT AVAILABLE’
  • if the date range of letters for the hearing papers are known, prepare AAT1 Cover Letter ‘ACE NOT AVAILABLE - DATE RANGE LETTER’

Convert the letter to PDF and insert at the front of the hearing papers.

12

Create customer cover letter and complete file + Read more ...

See Creating an Online Advice (OLA), including within an existing activity to create a Q402 cover letter via Online Advice (OLA).

  • Insert the payment type in the Service Reason (i.e. DSP, NSA)
  • Insert 2 for client
  • Insert 2 for customer
  • Insert the date of request (found in the middle of the scanned AAT1REQ)
  • Insert the date of decision (found in the ARO decision statement or APL system)
  • Insert 1 for standard hearing papers or 2 for hearing papers with section 35 redactions
  • Insert 1 for enclosed items
  • Enter ‘DATA’ in the Command field
  • Insert a space then the internal review number (APL) in the letter header after ‘Q402 /’
  • Check the customer’s address against the address on the AAT1REQ
  • Enter ‘TEXT’ in the Command field to view the letter
  • Enter ‘FIN’ in the Command field to finalise the letter

Send the letter to the address on the AAT request for statement. If the customer has updated their address after the AAT1 request was received, confirm address with AAT1. See Applicant Cover Letter for Section 37 Hearing Papers on the Resources page.

Delete uncompleted letters that contain errors:

  • Go to HS
    • Find the letter
    • Key ‘D’ next to the letter, and
  • Go to AL
    • Find the Q402 activity
    • Key ‘D’ next to the letter
    • Press [Enter]

If the letter is complete:

  • Go to HS,
    • Find the letter
    • Key ‘N’ next to the letter
    • Press [Enter]
  • Create a new letter

Insert the customer cover letter in front of the AAT1REQ document (behind the AAT1 cover letter if applicable).

13

Complete the hearing papers + Read more ...

Paginate the file, in Adobe > Tools > run Action Wizard Step 2

Check page numbers:

  • are centred at the bottom of the page
  • start from page number 1
  • start from the first page of the AAT1REQ document, and
  • are not obscured by any other page content

Re-save the paginated file in the Current folder.

Save a copy of the file as ‘SURNAME CRN APL AMS DUE DATE UNREDACTED’.

In the unredacted ‘SURNAME CRN APL AMS DUE DATE’ file:

  • apply the redactions and
  • re-save the file as 'SURNAME CRN APL AMS DUE DATE S37'. The two copies must be identical (the only difference being the redactions).

When the file is correct and complete:

  • complete the ERoD checklist
  • including any notes, and
  • re-save

Save the redacted file and the ERoD checklist to:
R:\NAT\DEBTPROGAPP\APPEALREVIEW\AROOSU\SSAT OSU\AAT1 Cases\Completed Cases\File preparation complete

Upload a copy of the redacted and unredacted file to HPE Content Manager. Keep the standard file naming convention but remove the DUE DATE when uploading.

Complete the ERoD Activity Tracker with the file details and make sure all details entered are correct.

  • Go to the AL screen
  • Select 'O' next to the relevant DOC to open the TA screen
  • On the TA screen, key the following details:
    • To Regional Office: key 'ARO'
    • To Workgroup: key 'AAT1'
    • To Position: key ‘RTG’

In Customer First:

  • Go to Inbox
  • Change Agent Box to Work Management
  • Select the case
  • Click Unassign
  • Select recategorised from the reallocation menu and work type ZIAM_EROD_WRK

Request a case and create a customer case folder + Read more ...

5

Collect relevant information from the customer’s record and check for sensitive information + Read more ...

Relevant screens

For FTB and Parenting Payment appeals:

  • Collect child care screens for relevant children for the relevant period from the customer’s record. For example:
    • FSC, CHC for FTB appeals
    • SSC for PPS appeals
  • If the ARO has listed any other screens or details from the customer’s record, consider section 35 application. See Requesting non-disclosure of information during an AAT first review. For example:
    • Address, or
    • Financial details

For Carers Payment appeals:

  • Collect relevant screens showing care details from the customer and/or third party care recipient’s record
  • If the ARO has listed any screens or information from the customer’s record, consider section 35 application. See Requesting non-disclosure of information during an AAT first review. For example:
    • Address, or
    • Financial details

Nominated support staff may be able to assist where required. Seek support from them before asking for help from the ARO.

Relevant documents

Collect all relevant DOCs using the ERoD Multi-tool, ensuring:

  • Only to include relevant DOCs containing information about care arrangements and the internal review
  • Collect DOCs in date order, oldest to newest
  • Include Document List (DL)/Archiving and Culling Engine (ACE) summary where required (for the customer only)
  • Save DOCs taken from a third party record in the relevant third party subfolder
    Note: DOC summaries are not required for third party DOCs

Relevant letters

Identify and collect all relevant letters from HS, ACE, Customer Archive Retrieval (CAR), Outbound Correspondence and Communication History. Where letters are available in multiple locations, letters should be sourced from ACE to make sure they are formatted correctly.

  • For most appeals, only letters sent to the customer about care of a child and the Q351 or Q449 are relevant
  • Include any letters listed in the ARO decision statement and ARO Notes
  • Collect letters from third party records if
  • not available on the customer record (i.e. the Q351) or
  • listed in the ARO decision statement and ARO Notes
    Note: letter summaries are not required for third party letters.

Do not consider letters for privacy as they will not be sent to third parties.

Sensitive Information

For potentially sensitive information, prepare a section 35 application for Non-Disclosure Order. See Requesting non-disclosure of information during an AAT first review.

Sensitive information may include:

  • Social worker DOCs, case notes and reports
  • Other sensitive information that might cause harm to the customer or third parties

6

Combine all evidence into the hearing papers document + Read more ...

Prepare multi-party hearing papers into the following sections:

  • Section I – evidence to be released to all parties
  • Section II – evidence to be released to customer only (include all of customer’s relevant letters and summaries)
  • Section III – evidence to be released to the first affected third party (include any relevant third party letters)
  • Section IV, V, etc. – evidence to be released to additional affected third parties if required (include any relevant letters)

Combine the files into the following order for Section I:

  • Section I Cover Page, located on Resources
  • AAT1 request email and accompanying documents
  • For ARO decisions:
    • ARO decision statement
    • ARO Notes
  • For SME decisions:
    • Q351 or Q449
    • SME decision DOC
    • ARO decision, and
    • notes (if applicable)
  • Affected third party ARO Decision and ARO Notes
  • Scanned evidence from oldest to newest, as per the date of receipt
  • Affected third party scanned evidence
  • Screen prints
  • Affected third party screens (carer payment appeals only)
  • DOCs from oldest to newest

Section II includes:

  • Section II Cover Page, located in Resources
  • Details collected from the customer's record that was heavily redacted in Section I. This includes information redacted under section 35 that is safe to release to the customer
  • The customer’s Document List
  • Letters collected from different systems:
    • in order from oldest to newest
    • by the date of the first letter collected in each system
  • Customer letter summaries (including mandatory HS)

Section III includes:

  • Section III Cover Page, located in Resources
  • Details collected from the customer's record that was heavily redacted in Section I. This includes information redacted under section 35 that is safe to release to the customer
  • Third party letters (if applicable)

Section IV, V, etc. includes:

  • Section IV Cover Page, located in Resources
    • For further affected third parties, amend the cover page to list the correct section
  • Details collected from the customer's record that was heavily redacted in Section I. This includes information redacted under section 35 that is safe to release to the customer
  • Third party letters (if applicable)

When all documents are in the correct order, select combine files to form 1 document.

  • Save the newly created document using the following naming conventions:
    • ‘AAT1 SURNAME CRN APL AMS DUE DATE’ for normal cases, or
    • ‘PRIORITY AAT1 SURNAME CRN APL AMS DUE DATE’ for priority cases
  • From Adobe Acrobat > Tools - Action Wizard > run Step 1 to:
    • convert colour to grey scale
    • apply text recognition to the file
    • rotate landscape pages to left to portrait mode
      Note: the top of landscape page when rotated should be on the left side of the page
    • search for blank pages and prompts to confirm deletion. See Treatment of Blank Pages in s37 Hearing Papers on the Resources page
    • shrink pages to allow for pagination
    • save the file as ‘AAT1 SURNAME CRN APL AMS DUE DATE 1’
  • Re-save the file as ‘AAT1 SURNAME CRN APL AMS DUE DATE’
  • Apply the relevant bookmarks to the file on the first page of that section:
    • ARO DECISION (for SME decisions this will be on the Q351 or Q449 or SME DOC)
    • EVIDENCE
    • SCREEN PRINTS
    • DOCS
    • LETTERS/NOTICES
  • Place an additional sub-bookmark 'THIRD PARTY' at the commencement of the third party information for each of the above bookmarks
    • Where there are multiple third parties, number each sub-bookmark 'THIRD PARTY 1', 'THIRD PARTY 2', etc.
  • Re-save the file
