Coding changes to periodic compensation payments

Table 1

Step

Action

1

Check Compensation Management System (CMS) record + Read more ...

Compensation staff, go to Step 3.

For all other staff, if the CMS record is:

  • open, go to Step 4
  • closed or customer is starting payments, go to Step 2

Some workers compensation and transport accident authorities can make payments to the surviving partner of a deceased worker, this is not compensation. See Assessment of income for Centrelink payments.

2

Create AYR activity and scan documents + Read more ...

Create a manual CMM/CLE activity. Go to the Activity Registration (AYR) screen in Customer First. Manually key the fields:

  • Service Reason: CMM
  • Activity Type: CLE
  • Source: MOD C or other valid source (for example, LET, FAX, PHO)
  • Date of Receipt: date of the 'Source' document or information was received
  • Proceed with Activity: Select Yes
  • Transfer to Region: key region code CTN to transfer activity to CRT
  • Notes: enter comments in the free text box, for example:
    • 'Starting periodic payments'
    • 'Periodic payments with arrears'
    • 'Periodic payment rates have varied'
  • Select Continue to generate CMM/CLE activity to AL screen

Scan supporting documents to the customer's record. For example:

  • MOD C
  • letter from the compensation payer advising payments have commenced, changed or ceased
  • Compensation Remittance/Payment advice

Procedure ends here.

Periodic compensation payments + Read more ...

For Compensation Recovery Teams (CRT) only

To:

  • change existing ongoing rate of periodic compensation, go to Step 5
  • stop ongoing periodic compensation, go to Step 6
  • add new ongoing periodic compensation, go to Step 7
  • add new arrears periodic compensation, go to Step 8
  • code foreign periodic compensation, see Step 10 in Table 2

4

Change periodic compensation payments details + Read more ...

To:

  • change existing ongoing rate of periodic payment, go to Step 5
  • stop ongoing periodic payment, go to Step 6

5

Change an existing ongoing rate of periodic payment + Read more ...

Go to the Compensation Management Summary screen. Select:

  • Periodic Payments
  • the relevant Claim ID
  • Change Details under Ongoing Details
  • Add a new line for extra lines

The following fields will appear in edit mode and can be changed:

  • Compensation Payer Incident Reference ID - key the reference number
  • Incident after CAP grant - can only be changed by the CRT:
    • Yes - treat as ordinary income
    • No - treat as direct deductions
  • Compensation paid in Australian currency - defaults to 'Yes'. If 'No' is selected, see Step 10 in Table 2
  • Change To Date and From Date. As compensation affects as a daily rate, the date of event is the date of effect:
    • To Date - end date the previous rate and add a line for new details
    • From Date - key the actual date the rate changed
    • Leave the end date blank
  • Gross Amount - change rate of payment
  • Payment Frequency - select from the dropdown box
  • Reason ceased - select from the dropdown box
  • Recipient Assessment/Pension Rate and Recipient Assessment CAP Qualified – can only be changed by CRT. See Resources page for coding
  • Complete Source and Receipt date
  • Select Finalise
  • Select Continue. The Assessment Results (AR) screen will show

Has a debt been created?

  • Yes,
    • CRT staff, See Table 2
    • For all other staff, transfer debt shell to Debt Raising region code XWA
  • No, record the details of the update in a DOC. Procedure ends here

6

Stop ongoing periodic payments + Read more ...

Staff outside of a compensation team will need CMS02 access to make updates to periodic payments in CMS. See Resources on how to request access.

If the compensation record is closed, create an AYR activity (see Step 2, then return to complete this step).

Periodic payments cannot be coded off for a date in future. Set a review to code the day after the periodic payments stop.

Before stopping ongoing periodic payments:

  • ask the customer the reason why payments have stopped to ensure they are not in breach of section 1166. See Resources page of Reasonable action to claim compensation
  • ask the customer to provide supporting documents, showing the date they were paid up to and including
  • scan supporting documents to customer record, for example, a letter from compensation payer advising payments have ceased and reason why they have ceased

Go to the Compensation Management Summary screen

  • Select:
  • Periodic Payments
  • relevant Claim ID
  • Change Details under Ongoing Details
  • Update the following fields:
    • To Date - with the last date of compensation entitlement
    • Reason Ceased - select from the dropdown box
    • Source
    • Receipt Date
  • Select Continue - the Assessment Results (AR) screen will display
    • If payment of arrears is being generated, ensure customer and/or partner are entitled to the amount. Refer to Payment of arrears after reassessment and stopping a payment.
  • Select:
    • Finalise
    • Continue to finalise the activity. The Compensation Management Summary screen will show. Make any changes if needed

If the AR screen does not show the option to finalise the activity:

  • select the started activity on the Activity List (AL) screen
  • go to Next
  • key 'CMMAR'
  • press [Enter]
    • the CMS Assessment Result screen will show
  • select Finalise
  • select Continue

Record details in a Fast Note.

Procedure ends here.

7

Add new ongoing details + Read more ...

For Compensation Recovery Teams (CRT) only

During the new claim process record the ongoing rate on CMS from the date paid to plus one. This ensures the customer and/or partner receive the correct rate. After coding the payments, go back and complete the CMM/CLE activity.

If the customer was only paid for a period before they got a CAP and are not partnered:

  • record the details on a Fast Note
  • do not record the details on the Periodic Payment Details screen

Record the periodic payments in CMS if a non-CAP customer has a partner:

  • who gets paid a CAP, and
  • has been since the date of the incident

This will ensure the partner gets the correct rate.

Record the whole period of compensation onto CMS if the customer got periodic payments prior to and after the grant of a CAP

  • Go to the Compensation Management Summary screen. Select:
    • relevant Claim ID
    • Periodic Payments
    • Add New Ongoing Details
    • Add a new line for extra lines

The following fields will appear in edit mode and can be changed:

  • Compensation Payer Incident Reference ID - code reference number
  • Incident after CAP grant - can only be changed by the CRT:
    • Yes - treat as ordinary income
    • No - treat as direct deductions
  • Compensation paid in Australian currency - defaults to 'Yes'. If 'No' is selected, see Step 10 in Table 2
  • From Date - key the correct start date the periodic payments started
  • To Date - only key this field when ongoing periodic payments have stopped or changed
  • Gross Amount - key the correct gross rate of the periodic payments
  • Payment Frequency - select from the drop down box
  • Reason ceased - only key this field when ongoing payment has stopped
  • Recipient Assessment Benefit/Pension Rate and Recipient Assessment CAP Qualified fields, see the Resources page for coding
  • Complete Source and Receipt Date
  • Select Continue, the Assessment Results (AR) screen will show

Check the amount is correct. If not, manually:

  • adjust the AR screen, or
  • calculate the debt

Check to see if:

  • there is nil entitlement due to compensation, and
  • a debt(s) exists for the same period as the compensation debt

If so:

  • zero the existing debt on the Debt Management and Information System (DMIS), and
  • include the debt in the compensation debt amount

See Step 8 in Table 2.

8

Adding new arrears periodic compensation + Read more ...

For Compensation Recovery Teams only

If the arrears period is before 20 September 2001:

  • complete a manual calculation, see Using the Multical debt calculator
  • adjust the Assessment Results (AR) screen

Staff should check the address is the same as recorded on the customer record. If different, complete relevant address screen checks. For example:

  • Address Summary (ADS)
  • Address History (ADH)
  • Online Search (OLS) facility

If in doubt phone the customer.

Once the details are confirmed or address is the same as the SS445, update the customers address. See Updating address details.

Has the customer and/or partner got a CAP in the period the arrears payment represents?

  • Yes, see Step 1 Table 2
  • No:
    • do not record arrears of periodic payments on the Periodic Payment Details screen
    • record the details on a Fast Note
    • record the end date of the periodic payment on the Claim Details screen
    • see Step 11 in Table 2

3

Enter arrears details to calculate debt + Read more ...

If the partner was in receipt of a CAP for the period before the customer started a CAP, see Raising periodic compensation charges for partnered customers.

  • Select Add New Arrears Details
  • Select Add a new line - for multiple lines

The following fields will appear in edit mode and can be changed:

  • Compensation Payer Incident Reference ID
  • Incident after CAP grant field - can only be changed by the CRT:
    • Yes - treat as ordinary income
    • No - treat as direct deductions
  • Compensation paid in Australian currency - defaults to 'Yes'. If 'No' is selected, go to Step 9
  • From Date - key the start date of the periodic payments
  • To Date - key the date the periodic payments arrears rates stopped or changed
  • Gross Amount - key the amount of the periodic payments
  • Payment Frequency - select from the drop down box
  • Recipient Assessment Benefit/Pension Rate and Recipient Assessment CAP Qualified fields, see the Resources page for coding
  • Gradual return to work - select from the drop down box. If 'Yes':
    • Complete the Return to Work Date field. Use the calendar to select the date
    • A Manual Follow up (MFU) will be created and transferred to the customer's home service centre, with the following text:
      'The customer has a current compensation claim pending. The Compensation Recovery Team has been advised the customer has started on a Gradual Return to Work programme and will start receiving part of their earnings. Check with the customer that their earnings details are up to date'
  • Issue Advice - select from the drop down box
  • Advice issued automatically - automatically sends letters to the compensation payer and the customer. Note: if the Compensation Payer screen has been updated to 'M' leave this field as 'A'
  • Advice issued manually where needed.
  • Advice not issued - no letters will be sent
  • S1184H debt - select 'Yes' if the compensation payer is in breach of an earlier notice. An automatic section 1184H debt advice will go to the compensation payer
  • Complete Source and Receipt Date
  • Select Continue. The Assessment Results (AR) screen will show. The system will calculate the total arrears debt amount for the correct period in DMIS
  • Check the AR screen shows the periodic debt amount:
    • if debt amount is correct, go to Step 5
    • no debt raised, go to Step 4

Calculate the debt using the Multical debt calculator.

Manually adjust the AR screen.

The $600 Carer Supplement is a recoverable payment when the underlying payment is a CAP. This is to be included in the debt calculations where applicable.

5

Is there an indicator for family and domestic violence? + Read more ...

A Display on Access (DOA) Fast Note ‘Special Contact Considerations (SCC) – Sensitive Issue - ‘FDV Concern Identified’ ‘will show for customer records with a family and domestic violence (FDV) concern.

Is there an indicator on the customer’s record?

  • Yes, any outstanding customer debt/s garnishee should not be considered in these circumstances, go to Step 7
  • No, go to Step 6

6

Do you need to contact the customer? + Read more ...

If the debt is raised under section 1184F or 11841 or if the customer debt/s meet the following criteria:

  • a single high value debt of more than $10,000
  • 3 debts with a value of more than $5,000
  • 4 or more debts, or
  • if the customer has identified vulnerabilities including, but not limited to:
    • customers under 16 years of age
    • customers over 80 years of age
    • homelessness
    • mental illness
    • family and domestic violence, or
    • low literacy or numeracy skills

Before calling a customer, send a pre-call SMS message if the customer has a telephone number on record and has subscribed to received SMS messages via Electronic Messaging.

Make genuine attempts to contact the customer prior to finalising the activity, and record details of each contact attempt on a Fast Note.

See General debt raising information.

Go to Step 7

7

Finalise the debt + Read more ...

Adjust any supplementary payments that are not recoverable under the compensation provisions.

Clean Energy Advance (CEA) and Income Support Bonus (ISB) payments are not CAPs. Raise a separate manual debt under Section 1223(1) of the Social Security Act.

  • from the AR screen adjust the primary payment by the CEA and ISB amount
  • finalise the activity
    • the correct debt amount will show on the letter and the Debt List (OPDL) screen
  • select the debt and from the Component Amount (OPCA) screen:
    • add the CEA and the ISB amount back to the primary payment
    • adjust the CEA and ISB components to $0

Make manual changes to payment summaries for debt changes before 1 July 2003.

  • Select Finalise
  • Select Continue. The Compensation Management Summary screen will show
  • If there is a break between payment dates, a separate debt for each payment period
  • record the details on a Fast Note

See General debt raising information

8

Are payments of periodic compensation to continue into the future? + Read more ...

  • Yes, go to Step 9
  • No, go to Step 11

9

Payments of periodic compensation are to continue + Read more ...

Do not finalise a claim when the:

  • customer and/or partner are receiving a CAP
  • periodic payments are continuing into the future, and
  • CAP continues to be paid to the customer or their partner, or a Low Income Health Care Card (LIC) is current

This will allow the rate to be changed.

Do not send a preliminary notice if the customer:

  • has ongoing periodic payments, and
  • claims or getting a CAP

This is because the compensation payer has already accepted liability.

Go to Step 11.

10

Coding Foreign Periodic Compensation Payments + Read more ...

Components of foreign payments may not be compensation.

Enter these details elsewhere, for example:

  • on the Foreign Income/Asset Details (FID) screen, enter family allowance components of Italian INAIL payments as Type 'OIN'
  • see Foreign income and assets

Make sure the Compensation paid in Australian currency field is 'No'.

Complete the following fields:

  • From date- key the start date of the periodic payments
  • To date
  • Country of Payment - select the country from the drop down box
  • Foreign Amount - the amount paid in foreign currency
  • Exchange Rate - the system will populate
  • Gross Amount ($AUD) - the system will populate
  • Payment Frequency - select from the drop down box
  • Recipient Assessment Benefit/Pension Rate and Recipient Assessment CAP Qualified fields, see the Resources page for coding
  • select Retrieve Exchange Data. This will convert the foreign payment amount into Australian dollars
  • select Continue. The Assessment Results (AR) screen will show

A Manual Follow-up (MFU) will appear on the Activity List (AL) screen if an incomplete CMM activity exists. It will show the following text:

  • 'Foreign compensation assessment activity cannot be completed due to existing compensation activity'

Foreign periodic payments update each month when changes in the exchange rates are applied. See Foreign currency and exchange rates.

The customer will get a Compensation Foreign Exchange letter every 3 months. It shows the exchange rates used.

11

Urgent requests + Read more ...

Most letters will be delivered by the customer’s or nominated third party’s preferred delivery channel. The customer or third party must be given a reasonable amount of time to receive the letter by post.

As the delegate, determine if the request is 'urgent' and/or a letter is required to be given on the same day.

Is an urgent or same day letter required?

  • Yes, fax a copy of the letter on the same day, see Faxing letters using Outlook and Customer First
  • No, procedure ends here
