Processing a new Power of Attorney (POA)

Table 1

Step

Action

1

On CDMS home page + Read more ...

Select Amend Consumer Details

  • key the Medicare card number in ID Number field
  • select Entitlement

The Amendment Search screen displays.

2

Select the customer the POA is to be added to + Read more ...

In Consumers:

  • select the radio button to select the customer to be amended
  • select POA tab

The new/Additional POA screen displays.

Note: this screen will confirm if the customer already has an existing POA.

3

Adding POA details + Read more ...

To add POA details, in New/Additional POA, key or select the details of the customer's POA eligibility:

  • POA Name (name of the person that holds the POA)
  • POA Type (select from the drop down list)
  • POA Text (key identity document)
  • POA Start Date (refer to the POA authority and key the date the POA commenced). Note: the POA start date will generally not be today's date
  • POA End Date (see Note)
  • Sight POA Due Date: (do not complete, field is obsolete)

Note:

  • General POA - do not record an end date unless it is recorded in the document
  • Enduring POA - do not record an end date
  • Limited POA - record an end date

Select Validate.

An information message displays informing whether the details were acceptable or not. Select OK to acknowledge the message. If not acceptable, change the details as required.

Select Return to return to the Amend Entitlement Details screen, Validate, Update.

Add comments in Personal tab in CDMS – Photo Identity sighted/provided for attorney/donee.

Step

Action

1

Protect personal details + Read more ...

If a customer is updating their Power of Attorney (POA), Guardianship or Administration order due to separating from a relationship, there are key tasks they should consider to protect their personal details.

2

On CDMS home page + Read more ...

Select Amend Consumer Details

  • key the Medicare card number in ID Number field
  • select Entitlement

The Amendment Search screen displays.

3

Select the customer to be amended + Read more ...

In Consumers:

  • select the radio button to select the customer to be amended
  • select POA tab

4

Amending POA details + Read more ...

To amend POA, in Recorded POAs:

  • select the relevant POA details
  • select Amend
  • change the details as required
  • select Validate

An information message displays informing whether the amendments were acceptable or not. Select OK to acknowledge the message. If not acceptable, change as required.

Select Return to return to the Amend Entitlement Details screen, Validate, Update.

5

Deleting POA details + Read more ...

To delete POA details, in Recorded POAs:

  • select the POA details to be deleted
  • select Delete
  • a message displays confirming the deletion. Select Yes to continue
  • a message displays informing of the deletion. Select OK to acknowledge the message

Select Return to return to the Amend Entitlement Details screen, Validate, Update

Processing Public Trustee (entity) appointment, or Guardianship and Administrative Orders

Table 3

Step

Action

1

On CDMS home page + Read more ...

Select Amend Consumer Details

  • key the Medicare card number in the ID Number field
  • select Group
  • select Members tab to confirm the person is the only person on the Medicare card

Note: where there is more than one person enrolled on the Medicare card and only one person on the card has been appointed a guardianship/administrative order, the person who has been appointed the order must be copied onto their own card. See Copy or transfer to a new group.

2

Record address and group contact + Read more ...

  • select Name/Address tab
  • untick the Group Association button
  • remove the Contact Reference Number
  • key either one of the following fields:
    • guardian or administration name field: 'Surname, Firstname'
    • Public Trustee field: 'Reference number,Public Trustee'. For example, 123456,Public Trustee
  • Record Address listed on the order
    • select Address Source from the drop down list
    • over-type the locality or postcode
    • key the new address details

Select Validate, Validate, Update

Note: where a guardianship/administration order is in place, the guardian/administrator address is recorded at group level. The Personal mailing address must be recorded as the customer's mailing address. Go to Step 4

3

Adding Public Trustee, Guardianship/Administrative orders details + Read more ...

  • select Entitlement
  • select POA tab

To add order details, in New/Additional POA, key or select the details of the customer's POA eligibility:

  • POA Name (full name or entity name)
  • POA Type - select from the drop down list either:
    • Guardianship Order
    • Administration Order
    • for Public Trustee - select Other
  • POA Text (only complete for Public Trustee)
    • key Public Trustee, reference number. For example Public Trustee, 123456
  • POA Start Date (refer to the order and key the date commenced)
  • POA End Date (do not complete)
  • Sight POA Due Date: field (do not complete, field is obsolete)

Select Validate.

An information message displays stating whether the details were acceptable or not. Select OK to acknowledge the message. If not acceptable, change as required.

Select Return to return to the Amend Entitlement Details screen, Validate, Update.

See Amend address in the Consumer Directory Maintenance System (CDMS) for Medicare.

4

Change the Personal address + Read more ...

  • select Personal
  • select Address tab
  • key the previous mailing address
  • If unknown
    • select Summary tab
    • click on History
    • in All Events, select Address, Filter Results
    • select Amended address radio button, click on History Details

This will display the previous address. Return to Personal address screen and update to reflect customers address:

  • over-type the locality or postcode
  • key the address details

Note: the mailing address must be an Australian address.

Select Validate, Validate, Update.

Adding Authorised Representative details

Table 4: this process is only for the Medicare Public team in Blacktown.

Step

Action

1

Check documentation + Read more ...

Ensure all appropriate documentation is provided to complete the CDMS, see Proof of identification table on the Resources page for the types of documentation required.

Verify identity documents in DVS tool.

Is all documentation provided?

  • Yes, go to Step 3
  • No, go to Step 2

2

Missing documentation + Read more ...

Where documentation is missing or not all required sections of the form have been completed, contact the customer.

If unable to contact:

  • leave a comment in PaNDA
  • send the Authorised Representative more information letter. Include:
    • all submitted documents
    • a blank statutory declaration or Authorisation to Act form(MO050) if required
  • attach letter to PaNDA work item

Procedure ends here.

3

Verify identity documents + Read more ...

Is verification successful?

  • Yes, go to Step 4
  • No, contact a HSDD PSO to request a second attempt at DVS verification. Go to Step 9

4

Log on to the CDMS + Read more ...

On the CDMS Home page, select Amend Consumer Details.

The Amendment Search screen is displayed.

5

Locate the customer + Read more ...

Select the following fields:

  • Program (select Medicare)
  • ID Number (key the customer’s Medicare card number)
  • ID Number Type (select Medicare card number)
  • select Group

The Group screen is displayed.

Select the Consumer who is the Authorised Representative.

6

Add POA details + Read more ...

Key the following Authorised Representative information from the form:

  • POA Name (Authorised Representative’s name)
  • POA Type (select Authorised Representative in the drop down box)
  • POA Text (key relationship details from the form)
  • POA Start Date (key the date in the Authorised Representative acceptance section)
  • POA End Date (do not complete)
  • Sight POA End Date (do not complete)
  • select Validate
  • press OK to the information message
  • select Return
  • select Validate
  • select Update
  • press OK

The POA is now recorded in Recorded POAs section of CDMS.

7

Finalise amendments + Read more ...

Check the form details and update any changes, see Amend customer details in the Consumer Directory Maintenance System (CDMS) for Medicare.

8

Send letter + Read more ...

Complete the Authorised representative confirmed letter and mail to the authorised representative.

9

PSO second attempt at document verification + Read more ...

Was HSDD PSO document verification successful?

  • Yes, process application. Go to Step 4
  • No:
    • Attempt to contact customer for alternative identity document. If customer provides alternative identification document, go to Step 4
    • Customer contact unsuccessful and/or second verification fails, return original document to the customer with Medicare Enrolment application more information required (DVS interim only) letter
