Process online registration for change of ownership - seed organisation

Table 2

Step

Action

1

Process application + Read more ...

  • Select the relevant tab to list any issues that need investigation
  • Check Steps 2, 3, 4, 5, 6 and 7 for each tab
  • Include comments in the Comments box

The one Comments box appears at the bottom of the screen for all tabs, not one for each tab. Add a new line in the Comments box for all comments.

Do not select the Save Comments button. If selected the application closes and it will need to be reopened.

2

Current Organisation tab + Read more ...

This tab is used to identify the existing organisation in EHP where the ownership details needs to be updated.

HPI-O number supplied by applicant:

  • If there are issues to investigate, cross check any issues using the second screen if needed
  • Override the HPI-O on this tab if the applicant supplied the incorrect HPI-O
  • If a new HPI-O is entered, select on the Validate button
  • Once all issues resolved, go to Step 3

HPI-O not supplied by applicant:

  • Search for the organisation using the details provided in the application:
    • trading name
    • ABN/ACN
    • phone number
    • email
    • business address
  • If one unique match to an existing seed organisation in EHP is found:
    • enter the correct HPI-O in the HPI-O Override field
    • select the Validate button
    • continue to process and check other issues if needed
    • if there are no further issues, go to Step 3
  • If multiple matches to existing seed organisations are found in EHP:
    • contact the applicant to determine the seed organisation which requires a change of ownership
    • if the one organisation is identified, enter the correct HPI-O in the HPI-O Override field
    • select the Validate button
    • Continue to process and check other issues if needed
    • If no further issues, go to Step 3
  • If no organisation matches:
    • contact the applicant to advise them the application must be rejected
    • copy and paste and update the text ‘No organisation in EHP matches organisation details in application’ into the Comments box
    • procedure ends here

3

Organisation Updates tab + Read more ...

If there are issues to process, cross check any issues using the second screen.

Evidence of authority to commit and change of ownership:

The applicant must provide both evidence of authority to commit and evidence of change of ownership.

For evidence of authority to commit, the online form does an upfront check to see if the applicant is on the organisation’s ABR record as a person with authority to commit.

If they are:

  • able to commit, the applicant will only need to provide document(s) to support the evidence of change of ownership
  • not able to commit, the applicant will need to provide both document(s) to support the evidence of change of ownership and evidence of authority to commit

If either of the above evidence has not been provided or is incorrect:

  • contact the applicant and request the applicant to upload additional evidence of authority to commit documentation and/or change of ownership documentation for the new ABN/ACN
  • copy and paste and update the text ‘Applicant needs to provide evidence of authority to commit documentation and/or change of ownership documentation for the new ABN/ACN’ into the Comments box
  • place application on hold, select the Hold radio button
  • procedure ends here

If correct evidence of authority to commit documentation and/or change of ownership documentation has been supplied, go to Step 4.

4

Networks tab + Read more ...

This tab displays:

  • all networked organisations (including the seed organisation identified in the application) which are in the seed organisations hierarchy
  • the selection the applicant has made on what updates are to be made to any organisations within the hierarchy

No networked organisations in hierarchy

Go to Step 5.

Networked organisations in hierarchy

Applicant selected:

  • ‘I do not want any of these networks to be updated’
    • select the Yes radio button on the seed organisation HPI-O which matches the HPI-O from the Current Organisation tab
    • go to Step 5
  • ‘I want all of the networks to be updated’
    • select the Update All button
    • this will automatically set the radio button to ‘Yes’ for updating all of the networked organisations
    • select the Yes radio button on the seed organisation HPI-O which matches the HPI-O from the Current Organisation tab
    • go to Step 5
  • ‘I only want to update the networks whose ABN/ACN details match those of the organisation I have nominated’
    • select the Update Matching button
    • this will automatically set the radio button to ‘Yes’ for updating all of the networked organisations whose ABN/ACN matched the seed organisations ABN/ACN
    • select the seed organisation HPI-O which matches the HPI-O from the Current Organisation tab
    • go to Step 5
  • ‘I only want to update the networks that I have identified in documentation and attached in the application’
    • from the list of organisations in this tab, select the Yes radio button for the organisations the applicant has identified for updating from the documentation they attached in their application
    • select the seed organisation HPI-O which matches the HPI-O from the Current Organisation tab
    • go to Step 5

5

Responsible Officer tab + Read more ...

If there are errors to process, cross check any duplicate details using the second screen.

Check if the Responsible Officer (RO) already exists in EHP or another application.

RO in EHP:

  • Copy the Healthcare Provider Identifier – Individual (HPI-I)/Admin number from the second screen
  • Select the relevant option from the Override option: drop down list
  • Enter the HPI-I/Admin number into the HPI-I/ADMIN number field
  • Copy, paste and update the text ‘RO linked to existing record <HPI-I/ADMIN number>’ into the Comments box
  • Go to Step 6

If the issue is not resolved when the HPI-I/Admin number has been entered:

  • escalate the application ID to the Provider HI Service and My Health Record Subject Matter Expert (SME) via email
  • re-categorise the work item in WLM to Healthcare Identifier Tier 3 Escalation

RO in another application:

If the other application is:

  • an exact duplicate and was successfully processed:
    • copy this application’s application ID
    • find and open the other application
    • copy, paste and update the text ‘Duplicate application with <other application ID>’ into the Comments box
    • reject application, select Reject
    • procedure ends here
  • not a duplicate and the RO is updating the RO for a different organisation:
    • override issues, select Override tick box that relates to each individual issue
    • copy and paste the text ‘RO details checked. Overrode issues.’ into the Comments box
    • go to Step 6

No RO exists:

  • Override issues
  • Copy and paste the text ‘RO details checked. Overrode issues.’ into the Comments box
  • Continue processing this application
  • Select Ignore ID as per Application and treat as New Individual from the drop down box on the RO details tab

Documents tab + Read more ...

Note: one document can be supplied if it satisfies both change of ownership and authority to commit for the ABN/ACN.

Has the applicant attached the correct documentation?

Current Organisation tab + Read more ...

If there are issues to investigate, cross check any issues using the second screen if needed.

Applicant is already the RO for the organisation in EHP:

  • Contact the applicant to advise they are already the RO for the organisation
  • Leave the below message if unable to speak to the applicant:
    • ‘We were unable to process your request to become the RO for your organisation because you are already the RO for this organisation in the HI Service. You can look up your healthcare identifier in Health Professional Online Services (HPOS), available on the Services Australia website.’
  • Copy and paste the text ‘Applicant already RO for this organisation. Rejected application.’ into the Comments box
  • Reject application, select Reject. Do not re-categorise in WLM
  • An email is sent to the applicant’s HPOS mail centre advising them of the unsuccessful outcome
  • Procedure ends here

HPI-O and ABN/ACN in application does not match existing HPI-O and/or ABN/ACN organisation details in the HI Service:

For each of the 3 scenarios contact the applicant for advice on how to process the application.

Scenario 1 – HPI-O in both the application and EHP are identical, the ABN/ACN provided in the application does not match the ABN/ACN in EHP, but the applicant needs to also update the ABN/ACN in EHP:

  • Reject the application, select Reject
  • Advise the applicant to complete the Healthcare Identifiers – Change Organisation Ownership application form online
  • Copy, paste and update the text ‘Applicant also needs to update ABN/ACN. Must submit a Change of Organisation Seed online application form’ into the Comments box
  • Procedure ends here

Scenario 2 – HPI-O in both the application and EHP are identical, the ABN/ACN provided in the application does not match the ABN/ACN in EHP, but the applicant does not need to update the ABN/ACN in EHP:

  • Advise the applicant to upload additional evidence of authority to commit documentation for the ABN/ACN in EHP
  • Copy, paste and update the text ‘Applicant needs to provide evidence of authority to commit documentation for the ABN/ACN in EHP’ into the Comments box
  • Place application on hold, select the Hold radio button
  • Procedure ends here

If the HPI-O in the application was incorrect, override the HPI-O on the online form to the correct HPI-O. In the HPI-O Override field, enter the correct HPI-O.

Note: any new HPI-O can be provided by the applicant as long as they can also supply the required evidence of authority to commit documentation for the ABN/ACN

  • Continue to process and check other issues if needed
  • If there are no further issues, go to Step 3

Scenario 3 – HPI-O in the application was incorrect.

  • Override the HPI-O on the online form to the correct HPI-O. In the HPI-O Override field, enter the correct HPI-O
    Note: any new HPI-O can be provided by the applicant as long as they can also supply the required evidence of authority to commit documentation for the ABN/ACN in EHP
  • Continue to process and check other issues if needed
  • If there are no further issues, go to Step 3

Application where trading name and address is provided by applicant (no HPI-O):

Search for the organisation using the details provided in the application:

  • Trading name
  • ABN/ACN
  • Phone number
  • Email
  • Business address

If one unique match to an existing seed organisation in EHP is found and the authority to commit documentation is:

  • correct:
    • in the HPI-O Override field, enter the correct HPI-O
    • continue to process and check other issues if needed
    • if there are no further issues, go to Step 3
  • incorrect:
    • contact the applicant to upload additional evidence of authority to commit documentation for the ABN/ACN in EHP
    • copy, paste and update the text ‘Applicant must provide evidence of authority to commit documentation for the organisation in EHP’ into the Comments box
    • place application on hold, select the Hold radio button
    • procedure ends here

If multiple matches to existing seed organisations are found in EHP:

  • contact the applicant to determine the seed organisation needing the RO update
  • if the one required organisation is identified, in the HPI-O Override field, enter the correct HPI-O (ensure that authority to commit is correct for the identified organisation)
  • continue to process and check other issues if needed
  • if there are no further issues, go to Step 3

If no organisation matches:

  • contact the applicant to advise them the application must be rejected
  • copy, paste and update the text ‘No organisation in EHP matches organisation details in application’ into the Comments box
  • procedure ends here
