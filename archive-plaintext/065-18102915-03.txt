This page contains information on processing CSHC online claims,
assisted customer claims and paper claims.

Staff trained in Process Direct select the Process Direct tab, staff not
yet trained select Customer First.

|                                   | generally default to the Date of  |
|                                   | Receipt. Do not manually change   |
|                                   | the start date on the SHC         |
|                                   | Assessment (SHA) screen.          |
|                                   |                                   |
|                                   | The SHA date should not be        |
|                                   | changed to a date after the       |
|                                   | determined start date, for        |
|                                   | example, the date of claim        |
|                                   | lodgement or the date the         |
|                                   | customer contacted the Services   |
|                                   | Australia in relation to a claim  |
|                                   | for a concession card.            |
|                                   |                                   |
|                                   | Early claim provisions do not     |
|                                   | apply to the CSHC. Any CSHC       |
|                                   | claims lodged prior to the        |
|                                   | customer being eligible           |
|                                   | (including before they reach Age  |
|                                   | Pension age) before the           |
|                                   | cancellation date of Age Pension  |
|                                   | has been reached or before the    |
|                                   | end of a NARWP, must be rejected. |
|                                   |                                   |
|                                   | Check the start date is correct,  |
|                                   | or change to an earlier date      |
|                                   | using the ‘general start date’    |
|                                   | provisions.                       |
|                                   |                                   |

|                                   | return if:                        |

|                                   |     the date of return to         |
|                                   |     Australia, and                |
|                                   | -   the date of return is less    |
|                                   |     than 26 weeks since their     |
|                                   |     CSHC cancellation date        |
|                                   |                                   |
|                                   | This can apply to CSHC online     |
|                                   | claims, customised claims, paper  |
|                                   | claims and verbal re-claims.      |
|                                   | Customers whose CSHC has been     |
|                                   | cancelled for more than 26 weeks  |
|                                   | will need to complete a full new  |
|                                   | claim.                            |

| 9                                 | Seniors Health Card income        |
|                                   | summary and details (SHIS/SHID)   |
|                                   | screen + Read more ...            |
|                                   |                                   |
|                                   | This is a compulsory screen,      |
|                                   | except for manual rejections.     |
|                                   |                                   |
|                                   | This screen records all customer  |
|                                   | and partner income details. It    |
|                                   | will display any deemed income    |
|                                   | from account based income streams |
|                                   | automatically once the assessment |
|                                   | has been completed on the         |
|                                   | Entitlements (ELD) screen.        |
|                                   |                                   |
|                                   | The Initiate DVA Check (if        |
|                                   | displayed), Date of Event,        |
|                                   | Reference Tax Year, Income Type   |
|                                   | and Taxable Income: fields are    |
|                                   | mandatory:                        |
|                                   |                                   |
|                                   | Date of Event (DOV): For new      |
|                                   | claims this DOV must be the same  |
|                                   | as the Date of receipt (DOR). The |
|                                   | DOV cannot be a date after the    |
|                                   | DOR and should be the date of     |
|                                   | claim as recorded on the SHA      |
|                                   | screen                            |
|                                   |                                   |
|                                   | Receiving a DVA Service Pension?: |
|                                   | This is auto loaded from the      |
|                                   | Veterans Affairs pension screen   |
|                                   | (DVA). All DVA payment data must  |
|                                   | be coded on the DVA screen. For   |
|                                   | more information on how to record |
|                                   | DVA income see Income from        |
|                                   | Department of Veterans' Affairs   |
|                                   | (DVA)                             |
|                                   |                                   |
|                                   | Reference Tax Year. Note: (1)     |
|                                   | Members of a couple need to       |
|                                   | provide information relating to   |
|                                   | the same financial year, and (2)  |
|                                   | the date of receipt (DOR) of the  |
|                                   | claim determines the reference    |
|                                   | tax year. Where a claim is        |
|                                   | started before 1 July, but lodged |
|                                   | on or after 1 July, the reference |
|                                   | tax year selected may no longer   |
|                                   | be valid. Where this occurs       |
|                                   | contact the customer and if       |
|                                   | required, make a further request  |
|                                   | for information to obtain the     |
|                                   | correct income details to assess  |
|                                   | the claim                         |
|                                   |                                   |
|                                   | -   TNA – Tax Notice of           |
|                                   |     Assessment: Record the        |
|                                   |     financial year of the Notice  |
|                                   |     of Assessment (NOA) or the    |
|                                   |     ATO lodgement extension,      |
|                                   |     which must be within 2 years  |
|                                   |     of the current financial      |
|                                   |     year. The most recent NOA     |
|                                   |     should be used. Use the year  |
|                                   |     the financial year ended,     |
|                                   |     e.g. for 2019/20, code 2020   |
|                                   | -   NRL - Not required to lodge   |
|                                   |     an income tax return: Record  |
|                                   |     the relevant financial year.  |
|                                   |     This must be 1 of the         |
|                                   |     previous 2 financial years,   |
|                                   |     not the current financial     |
|                                   |     year                          |
|                                   | -   EST - Estimate: If an         |
|                                   |     estimate is used it needs to  |
|                                   |     be for the current financial  |
|                                   |     year and for an acceptable    |
|                                   |     condition for using an        |
|                                   |     estimate of income for the    |
|                                   |     CSHC income test. Record the  |
|                                   |     year that the current         |
|                                   |     financial year is to end, for |
|                                   |     example, for 2020/21 code     |
|                                   |     2021                          |
|                                   |                                   |
|                                   | Income Type: Code 1 of the        |
|                                   | following:                        |
|                                   |                                   |
|                                   | -   'TNA' for a NOA and for       |
|                                   |     customers who have been       |
|                                   |     granted an ATO lodgement      |
|                                   |     extension                     |
|                                   | -   'NRL' when a customer does    |
|                                   |     not have a NOA because their  |
|                                   |     income is either below the    |
|                                   |     tax lodgement threshold or as |
|                                   |     a result of an ATO tax        |
|                                   |     offset. NRL must not be used  |
|                                   |     when the customer has income  |
|                                   |     above the tax lodgement       |
|                                   |     threshold, or when the        |
|                                   |     customer has had TAX withheld |
|                                   |     from any income, in that      |
|                                   |     financial year                |
|                                   | -   'EST' for an estimate. If the |
|                                   |     customer was not required to  |
|                                   |     lodge a tax return either     |
|                                   |     because their income was      |
|                                   |     below the tax free threshold  |
|                                   |     or as a result of an ATO tax  |
|                                   |     offset, it is not considered  |
|                                   |     an estimate (EST), code as    |
|                                   |     NRL                           |
|                                   |                                   |
|                                   | Taxable Income: Code 1 of the     |
|                                   | following:                        |
|                                   |                                   |
|                                   | -   Enter the amount on the NOA   |
|                                   |     or for customers been granted |
|                                   |     an ATO lodgement extension an |
|                                   |     estimate of their taxable     |
|                                   |     income. Note: when a customer |
|                                   |     provides an estimate, a       |
|                                   |     superannuation withdrawal     |
|                                   |     under the First Home Super    |
|                                   |     Saver (FHSS) Scheme is not    |
|                                   |     counted towards their taxable |
|                                   |     income                        |
|                                   | -   The current year estimate, or |
|                                   | -   The customer's income below   |
|                                   |     the tax lodgement threshold   |
|                                   |                                   |
|                                   | Customers who have provided an    |
|                                   | accepted estimate for the current |
|                                   | financial year must also have     |
|                                   | their verified income that was    |
|                                   | over the income limit from either |
|                                   | of the last 2 financial years     |
|                                   | recorded on the SHIS/SHID screen. |
|                                   | After the estimate has been       |
|                                   | recorded using the above coding,  |
|                                   | record the NOA using the date     |
|                                   | before the claim DOV, that is,    |
|                                   | DOV - 1 day.                      |
|                                   |                                   |
|                                   | Customers who are required to     |
|                                   | lodge a tax return but cannot     |
|                                   | provide income details from       |
|                                   | either of the previous 2          |
|                                   | financial years because they have |
|                                   | not yet lodged their tax return   |
|                                   | with the ATO and do not have an   |
|                                   | ATO lodgement extension, are not  |
|                                   | permitted to use a current year   |
|                                   | estimate for their income.        |
|                                   | Manually reject their CSHC claim. |
|                                   | See Rejecting a claim for CSHC.   |
|                                   |                                   |
|                                   | The following fields are optional |
|                                   | and must only be completed if     |
|                                   | they are relevant to the customer |
|                                   | claim:                            |
|                                   |                                   |
|                                   | -   Foreign Income: key the       |
|                                   |     Australian dollar value from  |
|                                   |     section IT4 on the income tax |
|                                   |     return or other foreign       |
|                                   |     income documents if not       |
|                                   |     required to lodge an          |
|                                   |     Australian tax return         |
|                                   | -   Net Investment Loss: key net  |
|                                   |     rental property or financial  |
|                                   |     investment loss details from  |
|                                   |     section IT5/6 on the income   |
|                                   |     tax return or other financial |
|                                   |     documents if not required to  |
|                                   |     lodge                         |
|                                   | -   Employer Provided Benefit:    |
|                                   |     (also known as fringe         |
|                                   |     benefits). Key the details    |
|                                   |     from the PAYG payment summary |
|                                   |     or section IT1 on the income  |
|                                   |     tax return minus the first    |
|                                   |     $1,000, e.g. if the customer  |
|                                   |     received $3,200, only record  |
|                                   |     $2,200                        |
|                                   | -   Reportable Super              |
|                                   |     Contributions: key reportable |
|                                   |     employer or personal          |
|                                   |     deductable superannuation     |
|                                   |     contribution details from the |
|                                   |     PAYG payment summary, section |
|                                   |     IT2 or D12 on the income tax  |
|                                   |     return or statement from      |
|                                   |     superannuation provider       |
|                                   |                                   |
|                                   | Complete the details as required. |
|                                   |                                   |
|                                   | Once all information is updated   |
|                                   | press [Next].                     |
|                                   |                                   |
|                                   | If CSHC is rejected due to an     |
|                                   | income estimate for the current   |
|                                   | financial year being over the     |
|                                   | limit, they cannot lodge a second |
|                                   | claim and provide previous        |
|                                   | financial year income details.    |
|                                   | The customer must provide a new   |
|                                   | estimate and reasons why their    |
|                                   | estimate has been changed. The    |
|                                   | revised estimate should only be   |
|                                   | accepted if details provided and  |
|                                   | the new estimate is reasonable in |
|                                   | comparison with the previous      |
|                                   | estimate.                         |
|                                   |                                   |
|                                   | Note: the SHIS/SHID screen        |
|                                   | includes a Deemed Income Account: |
|                                   | field. This is a protected field  |
|                                   | that cannot be updated manually.  |
|                                   | If a customer or partner has      |
|                                   | account-based income streams      |
|                                   | recorded that are subject to      |
|                                   | deeming, the claim activity will  |
|                                   | need to be taken through          |
|                                   | assessment to ensure the deemed   |
|                                   | income figure (if applicable) on  |
|                                   | the SHIS/SHID screen is correct.  |
|                                   | To do this, go to the ELD screen  |
|                                   | and then return to the SHIS/SHID  |
|                                   | screen to view the updated Deemed |
|                                   | Income Account.                   |

| 13                                | Finalising a claim for CSHC +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | After selecting the Assess        |
|                                   | button, the Entitlement (ELD)     |
|                                   | screen displays. Ensure the claim |
|                                   | outcome is correct.               |
|                                   |                                   |
|                                   | If a customer has died before     |
|                                   | their claim was finalised, extra  |
|                                   | coding is needed to end           |
|                                   | Electronic Messaging (EM)         |
|                                   | subscription.                     |
|                                   |                                   |
|                                   | -   Select Finish button          |
|                                   | -   A dialogue box will give the  |
|                                   |     Service Officer an            |
|                                   |     opportunity to add additional |
|                                   |     comments/information about    |
|                                   |     the claim outcome             |
|                                   | -   Select Finalise to complete   |
|                                   |     the claim assessment          |
|                                   |                                   |
|                                   | Note: all scans associated with   |
|                                   | the claim must be actioned once   |
|                                   | the claim is finalised.           |
|                                   |                                   |
|                                   | When a CSHC claim is finalised    |
|                                   | using Process Direct, an          |
|                                   | automatic DOC is created on the   |
|                                   | customer’s record to reflect the  |
|                                   | outcome.                          |
|                                   |                                   |
|                                   | Service Officers will need to     |
|                                   | check that the DOC meets:         |
|                                   |                                   |
|                                   | -   Getting it Right (GIR)        |
|                                   |     Recording reasons for         |
|                                   |     decisions                     |
|                                   | -   Online Document Recording     |
|                                   |     (ODR)                         |
|                                   |                                   |
|                                   | Note: if a customer already holds |
