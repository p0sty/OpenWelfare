+-----------------------------------+-----------------------------------+
| Category                          | Description                       |
+-----------------------------------+-----------------------------------+
| Address Details (AD) screen       | Recording the customer's          |
|                                   | address + Read more ...           |
|                                   |                                   |
|                                   | The customer's address details    |
|                                   | are recorded on the AD screen.    |
|                                   | This information assists with     |
|                                   | assessment of eligibility for:    |
|                                   |                                   |
|                                   | -   Remote Area Allowance (RAA)   |
|                                   | -   ABSTUDY Fares Allowance       |
|                                   | -   on-going qualification for    |
|                                   |     away from home rate           |
|                                   |                                   |
|                                   | To facilitate correct payment of  |
|                                   | away from home entitlements for   |
|                                   | ABSTUDY, the following coding is  |
|                                   | required for this screen:         |
|                                   |                                   |
|                                   | Secondary school student          |
|                                   |                                   |
|                                   | A secondary school student is     |
|                                   | entitled to away from home rate   |
|                                   | allowances (Living Allowance,     |
|                                   | Rent Assistance (RA) and Remote   |
|                                   | Area Allowance (RAA)) across      |
|                                   | vacation periods, even when the   |
|                                   | student returns to the permanent  |
|                                   | home for the holidays. The AD     |
|                                   | screen must be coded to match the |
|                                   | eligibility period for away from  |
|                                   | home rate to ensure qualification |
|                                   | for RAA matches the eligibility   |
|                                   | period for away from home rate.   |
|                                   |                                   |
|                                   | If the student's home address is  |
|                                   | in a RAA specified area, record   |
|                                   | only a Home (HOM) address on the  |
|                                   | AD screen. Do not record the Term |
|                                   | (TER) address. This facilitates   |
|                                   | correct payment of RAA for the    |
|                                   | full entitlement period.          |
|                                   |                                   |
|                                   | To ensure the term address is     |
|                                   | recorded on the customer's        |
|                                   | record, a Display on Access (DOA) |
|                                   | DOC can be created. If the        |
|                                   | student wants to receive mail at  |
|                                   | their term address, code a POS    |
|                                   | (postal) address with the term    |
|                                   | address details.                  |
|                                   |                                   |
|                                   | If the student's home address is  |
|                                   | not in a RAA specified area,      |
|                                   | record a HOM address and details  |
|                                   | of the location the student is    |
|                                   | residing whilst studying as a TER |
|                                   | address.                          |
|                                   |                                   |
|                                   | Secondary school student –        |
|                                   | Private Board Arrangement         |
|                                   |                                   |
|                                   | If the student's home address is  |
|                                   | in a RAA specified area, to       |
|                                   | generate a Rent Certificate       |
|                                   | SU523HD with TER address details  |
|                                   | and ensure correct payment of RAA |
|                                   | for the full entitlement period,  |
|                                   | use the following coding:         |
|                                   |                                   |
|                                   | Record details of the location    |
|                                   | the student is residing whilst    |
|                                   | studying on ADH as a TER address. |
|                                   |                                   |
|                                   | Address Type: TER                 |
|                                   |                                   |
|                                   | Start Date: code the commencement |
|                                   | date of the board arrangement     |
|                                   |                                   |
|                                   | End Date: code start Date + 2     |
|                                   | days                              |
|                                   |                                   |
|                                   | Address: Code the students term   |
|                                   | address                           |
|                                   |                                   |
|                                   | Is Rent Assistance being claimed  |
|                                   | on this Address: Yes              |
|                                   |                                   |
|                                   | Is this address in a Remote Area  |
|                                   | for the purpose of Remote Area    |
|                                   | Allowance: No                     |
|                                   |                                   |
|                                   | Should this address be used for   |
|                                   | all mail: Yes                     |
|                                   |                                   |
|                                   | The above coding will send the    |
|                                   | Rent Certificate SU523HD to the   |
|                                   | students TER address and will     |
|                                   | populate with the TER address     |
|                                   | details.                          |
|                                   |                                   |
|                                   | Tertiary and secondary            |
|                                   | non-schooling students            |
|                                   |                                   |
|                                   | The customer's address details    |
|                                   | coded must include the student's  |
|                                   | actual HOM address as well as the |
|                                   | TER address at the study          |
|                                   | location. Tertiary students are   |
|                                   | subject to normal Remote Area     |
|                                   | Allowance entitlement provisions. |
|                                   |                                   |
|                                   | Australian Apprentices            |
|                                   |                                   |
|                                   | Australian Apprentices who need   |
|                                   | to live away from home can have a |
|                                   | TER address coded.                |

|                                   | OTAC Release Dates                |
|                                   |                                   |
|                                   | A Payment on Advice request is    |
|                                   | sent to the school or boarding    |
|                                   | institution (based on where the   |
|                                   | student resides) in the second    |
|                                   | week of each school term. The     |
|                                   | request contains a One Time       |
|                                   | Access Code (OTAC) for            |
|                                   | schools/boarding institutions to  |
|                                   | log into the Education            |
|                                   | Institution eService and provide  |
|                                   | term commencement dates for       |
|                                   | ABSTUDY students. In the          |
|                                   | eService, schools/boarding        |
|                                   | institutions can view their       |
|                                   | estimated ABSTUDY boarding        |
|                                   | instalment payments for students  |
|                                   | and the delivery date for these   |
|                                   | entitlements.                     |
|                                   |                                   |
|                                   | A student processed as eligible   |
|                                   | for the away from home rate after |
|                                   | the SAS extraction date for the   |
|                                   | Payment on Advice request for the |

|                                   | ABSTUDY away from home rate for   |

|                                   | Third Friday                      |
|                                   |                                   |
|                                   | ABSTUDY secondary school students |

|                                   |     control, they are treated the |

|                                   | Calculating the student start     |
|                                   | date on the EDC screen            |
|                                   |                                   |
|                                   | The OTAC release dates can be     |
|                                   | found in the resources tab of     |
|                                   | Payment on Advice for ABSTUDY     |
|                                   | away from home rate. Within this  |
|                                   | document is the SAS extraction    |
|                                   | date for each term and state.     |
|                                   | This date is very important when  |
|                                   | choosing which date to code in    |
|                                   | the Student start date field on   |
|                                   | the EDC screen. The SAS           |
|                                   | extraction date is the date used  |
|                                   | to compile the payment on advice  |
|                                   | list that is sent to schools and  |
|                                   | hostels. If a claim is granted    |
|                                   | for the current term entitlement  |
|                                   | period after the SAS extraction   |
|                                   | date, then that student will not  |
|                                   | appear on the Payment on Advice   |
|                                   | list.                             |
|                                   |                                   |
|                                   | If a claim is being assessed      |
|                                   | before the SAS extraction date in |
|                                   | the current term, the student     |
|                                   | start date should be coded as the |
|                                   | first day of the term entitlement |
|                                   | period.                           |
|                                   |                                   |
|                                   | Note: Term 1 will need to be      |
|                                   | coded as the course start date    |
|                                   | because it is not possible to     |
|                                   | code 1 Jan.                       |
|                                   |                                   |
|                                   | If a claim is being assessed      |
|                                   | after the SAS extraction date in  |
|                                   | the current term and no first day |
|                                   | of attendance has been provided   |
|                                   | from the school or hostel for the |
|                                   | terms then the student start date |
|                                   | should be coded as the first day  |
|                                   | of the next term entitlement      |
|                                   | period. Once the first day of     |
|                                   | attendance has been provided, the |
|                                   | EDC can be updated with the date  |
|                                   | the student commenced so that the |
|                                   | correct payments are issued. The  |
|                                   | school may need to be contacted   |
|                                   | or a manual request sent to       |
|                                   | collect this information.         |
|                                   |                                   |
|                                   | Coding for Term 4 differs         |
|                                   | slightly when processing after    |
|                                   | the SAS extraction date. The      |
|                                   | coding will depend on if ABSTUDY  |
|                                   | is going to continue in the next  |
|                                   | calendar year. If the student is  |
|                                   | continuing study in the next      |
|                                   | calendar year, study will need to |
|                                   | be coded for both calendar years. |
|                                   | Term 4 should be coded from the   |
|                                   | start of the Term 4 entitlement   |
|                                   | period. The EPDC should be coded  |
|                                   | from 1 January to avoid payments  |
|                                   | being issued. If the student is   |
|                                   | not continuing in the next        |
|                                   | calendar year, the claim cannot   |
|                                   | be granted until the Term 4 first |
|                                   | day of attendance is confirmed    |
|                                   | with the school.                  |
|                                   |                                   |
|                                   | When assessing previous terms, or |
|                                   | having been provided with the     |
|                                   | first day of attendance to pay    |
|                                   | arrears in the current term, the  |
|                                   | student start date on EDC will    |
|                                   | need to be calculated using the   |
|                                   | third Friday rule.                |
|                                   |                                   |
|                                   | In some cases, multiple EDC       |
|                                   | screens may be needed if there    |
|                                   | has been a gap in entitlement.    |
|                                   |                                   |
|                                   | If there is a gap in entitlement  |
|                                   | and the student was previously    |
|                                   | attending the same school in the  |
|                                   | same calendar year then a new EDC |
|                                   | screen will need to be inserted   |
|                                   | under the same course ID. This    |
|                                   | can be done by selecting the      |
|                                   | existing course, adjusting the    |
|                                   | participation status event date   |
|                                   | field to the first day after the  |
|                                   | gap in entitlement and either:    |
|                                   |                                   |
|                                   | -   in Process Direct selecting   |
|                                   |     Edit                          |
|                                   | -   in Customer First coding 'I'  |
|                                   |     in the Action field           |
|                                   |                                   |
|                                   | The student start and end dates   |
|                                   | within each EDC screen will need  |
|                                   | to be adjusted to ensure that it  |
|                                   | reflects the gap of entitlement.  |

| Coding on the School Fees and     | Coding for transfer of excess     |
| Boarding Details (EYSB) screen to | SFA + Read more ...               |
| allow transfer of excess School   |                                   |
| Fees Allowance (SFA) to 'top-up'  | Secondary school students         |
| Living Allowance shortfall        | boarding at a school, an approved |
|                                   | ABSTUDY Homestay arrangement, or  |
|                                   | at a hostel that is a signatory   |
|                                   | to the ABSTUDY Hostel Agreement,  |
|                                   | can qualify for any excess SFA to |
|                                   | be transferred to assist with any |
|                                   | shortfall in Living Allowance.    |
|                                   | This is known as 'top-up' and is  |
|                                   | not available to students in any  |
|                                   | other living arrangement.         |
|                                   |                                   |
|                                   | Where a student's total annual    |
|                                   | entitlement of combined Living    |
|                                   | Allowance, Rent Assistance and    |
|                                   | Remote Area Allowance is          |
|                                   | insufficient to meet the board    |
|                                   | charge of the school or hostel,   |
|                                   | any excess SFA, up to the maximum |
|                                   | available, is transferred across  |
|                                   | to top-up the shortfall in board. |
|                                   |                                   |
|                                   | The system automatically          |
|                                   | determines top-up eligibility and |
|                                   | amount, based on the coding on    |
|                                   | the EYSB screen.                  |
|                                   |                                   |
|                                   | Mandatory coding of the actual    |
|                                   | board and school fee charges is   |
|                                   | needed to allow top-up to occur   |
|                                   | where applicable.                 |
|                                   |                                   |
|                                   | Note: both board and school fee   |
|                                   | charges must be obtained prior to |
|                                   | the first ABSTUDY boarding        |
|                                   | instalment being paid. If the     |
|                                   | school fee charge is not coded,   |
|                                   | the system will assume that no    |
|                                   | school fees are charged and       |
|                                   | therefore the full amount of SFA  |
|                                   | will be available as a top-up     |
|                                   | towards boarding fees. This may   |
|                                   | mean that insufficient SFA is     |
|                                   | available to pay the school fees. |
|                                   |                                   |
|                                   | See the Resources page for        |
|                                   | examples of how the system        |
|                                   | automatically calculates top-up   |
|                                   | based on the coding of board and  |
|                                   | school fee charges on EYSB        |
|                                   | screen.                           |

| Checking assessment outcome at    | Checking the outcome on the AR    |
| the Assessment Results (AR)       | screen (Customer First) or ELD    |
| screen                            | screen (Process Direct) + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Upon completion of coding for the |
|                                   | away from home rate, the AR/ELD   |
|                                   | screen will display the outcome   |
|                                   | of the assessment. The            |
|                                   | information displayed on the      |
|                                   | AR/ELD screen should be checked   |
|                                   | to ensure the assessment has      |
|                                   | produced the correct outcome.     |
|                                   |                                   |
|                                   | The AR/ELD screen displays the    |
|                                   | student's entitlements,           |
|                                   | including:                        |
|                                   |                                   |
|                                   | -   allowance type                |
|                                   | -   period of entitlement for     |
|                                   |     each allowance                |
|                                   | -   total amount of each          |
|                                   |     allowance for the entitlement |
|                                   |     period (including any arrears |
|                                   |     owing)                        |
|                                   | -   end date of eligibility       |
|                                   |                                   |
|                                   | Check the display to ensure       |
|                                   | correct entitlement is payable    |
|                                   | before finalising the assessment. |
|                                   | A quick calculation may be useful |
|                                   | to determine whether the coding   |
|                                   | has correctly released all        |
|                                   | necessary payments. The process   |
|                                   | to calculate entitlements is:     |
|                                   |                                   |
|                                   | add up total annual entitlement   |
|                                   | of Living Allowance, Rent         |
|                                   | Assistance and Remote Area        |
|                                   | Allowance (where eligible) based  |
|                                   | on age and living circumstances   |
|                                   |                                   |
|                                   | apply applicable means test to    |
|                                   | entitlements to determine         |
|                                   | adjusted total entitlement of     |
|                                   | boarding allowances and School    |
|                                   | Fees Allowance (SFA) (for         |
|                                   | secondary school level students)  |
|                                   |                                   |
|                                   | for secondary school boarders -   |
|                                   | match total adjusted entitlements |
|                                   | (boarding and school fee) against |
|                                   | board and school fee charges of   |
|                                   | school/hostel or private board    |
|                                   | provider:                         |
|                                   |                                   |
|                                   | -   if board entitlement is in    |
|                                   |     excess of board charge,       |
|                                   |     residual boarding             |
|                                   |     entitlements payable to       |
|                                   |     student or parent             |
|                                   | -   if boarding entitlements are  |
|                                   |     insufficient to cover board   |
|                                   |     charge, determine if excess   |
|                                   |     SFA exists                    |
|                                   | -   if SFA is in excess of school |
|                                   |     fee charge, transfer unused   |
|                                   |     SFA up to maximum available   |
|                                   |     to meet shortfall in board    |
|                                   |     charge                        |
|                                   | -   if SFA is insufficient to     |
|                                   |     meet the school fee charge,   |
|                                   |     full charge cannot be met     |
|                                   |                                   |
|                                   | determine if correct payee and    |
|                                   | payment frequency will occur      |
|                                   |                                   |
|                                   | If correct complete the activity  |
|                                   | on the AR/ELD screen, and record  |
|                                   | all details of the assessment on  |
|                                   | the new claim DOC.                |
