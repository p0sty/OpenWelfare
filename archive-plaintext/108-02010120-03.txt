'Manual Rejection of YAL Pending Sibling NCL Action'

Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Youth Allowance (YA) benefit      |
|                                   | status YAL/PSA + Read more ...    |
|                                   |                                   |
|                                   | A customer's benefit status will  |
|                                   | reflect YAL/PSA where a YA claim  |
|                                   | is finalised and the dependent YA |
|                                   | customer exceeded the 'Parental   |
|                                   | income cut-off limit'. This       |
|                                   | status will remain for 1 day      |
|                                   | (until the end of the next        |
|                                   | working date after the claim was  |
|                                   | processed). Once the PSA          |
|                                   | timeframe has elapsed, the claim  |
|                                   | will reject (PIH - Parental       |
|                                   | Income too High) if there are no  |
|                                   | changes to parental income or the |
|                                   | family pool which would entitle   |
|                                   | the customer to receive payment.  |
|                                   |                                   |
|                                   | There are circumstances where the |
|                                   | system cannot automatically       |
|                                   | reject these claims and their     |
|                                   | status will remain as YAL/PSA.    |
|                                   | This results in the allocation of |
|                                   | a new claim (NCL) activity in     |
|                                   | Customer First with a Review      |
|                                   | Reason: 'YAL Pending Sibling NCL  |
|                                   | Review'.                          |
|                                   |                                   |
|                                   | Before updating the customer's    |
|                                   | record, Service Officers must:    |
|                                   |                                   |
|                                   | -   investigate the customer's    |
|                                   |     record to establish if there  |
|                                   |     are any siblings in the       |
|                                   |     family pool or the sibling    |
|                                   |     has submitted a new claim,    |
|                                   |     and                           |
|                                   | -   check the combined income is  |
|                                   |     under the PIT threshold       |
+-----------------------------------+-----------------------------------+
| 2                                 | Processing YAL/PSA New Claim work |
|                                   | item + Read more ...              |
|                                   |                                   |
|                                   | Is the customer eligible for      |
|                                   | Youth Allowance (YA)?             |
|                                   |                                   |
|                                   | Yes,                              |
|                                   |                                   |
|                                   | -   Re-index rejected claim       |
|                                   | -   See, Assessing and coding     |
|                                   |     student and Australian        |
|                                   |     Apprentice claims for Youth   |
|                                   |     Allowance (YA)                |
|                                   | -   Procedure ends here           |
|                                   |                                   |
|                                   | No, go to Step 3                  |
+-----------------------------------+-----------------------------------+
| 3                                 | Customer is not entitled to YA +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | If the customer is not entitled   |
|                                   | to YA, code as shown below in     |
|                                   | either:                           |
|                                   |                                   |
|                                   | Process Direct                    |
|                                   |                                   |
|                                   | Go to the Benefit Action (BA)     |
|                                   | screen                            |
|                                   |                                   |
|                                   | Select Add. In the Create Benefit |
|                                   | Action screen, complete the       |
|                                   | fields:                           |
|                                   |                                   |
|                                   | -   Benefit Type Code: YAL        |
|                                   | -   Benefit Action: REJ – REJECT  |
|                                   | -   Reason: PIH – Parental Income |
|                                   |     too High                      |
|                                   | -   Date of Effect: as            |
|                                   |     appropriate                   |
|                                   | -   Select Save                   |
|                                   |                                   |
|                                   | Complete the Receipt Date and     |
|                                   | Channel. Select Save              |
|                                   |                                   |
|                                   | Select Assess and address any     |
|                                   | errors/warnings                   |
|                                   |                                   |
|                                   | Selecting Assess again            |
|                                   |                                   |
|                                   | Select Finish. Record details of  |
|                                   | the update                        |
|                                   |                                   |
|                                   | Select Finalise                   |
|                                   |                                   |
|                                   | Customer First                    |
|                                   |                                   |
|                                   | -   Select the activity from the  |
|                                   |     Activity List (AL) and cancel |
|                                   |     the activity                  |
|                                   | -   Next: field, Benefit Action   |
|                                   |     (BA)                          |
|                                   | -   Service Reason: field, YAL -  |
|                                   |     Youth Allowance               |
|                                   | -   Action: field, REJ - Reject   |
|                                   | -   Reason: field, PIH - Parental |
|                                   |     Income too High               |
|                                   | -   Effect Date: field, as        |
|                                   |     appropriate                   |
|                                   | -   Source: field, as appropriate |
|                                   | -   Receipt Date: field, as       |
|                                   |     appropriate                   |
|                                   | -   Next: field, Activity Results |
|                                   |     (AR)                          |
|                                   | -   If the customer is not        |
|                                   |     eligible, do not inhibit      |
|                                   |     letter                        |
|                                   | -   Finalise the activity         |
|                                   | -   DOC the record                |
