For Compliance Review Officers (RO) only.

This page contains the process for actioning a Service Update or Review
in the Integrated Review System (IRS).

Actioning a Service Update or Review in the IRS

Integrated Review System (IRS) Service Updates and Reviews

Actioning a Service Update or Review in the IRS

This table describes the process for actioning Service Updates and
Reviews in the Integrated Review System (IRS).

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Recording a customer contact in   |
|                                   | IRS + Read more ...               |
|                                   |                                   |
|                                   | This first section describes how  |
|                                   | to initiate and record a customer |
|                                   | contact and how to action a       |
|                                   | customer response in the IRS.     |
|                                   |                                   |
|                                   | To commence a customer contact on |
|                                   | a review, 'S'elect the review     |
|                                   | from the Review Summary (IRRS)    |
|                                   | screen.                           |
|                                   |                                   |
|                                   | This will navigate to the Review  |
|                                   | Details (IRRD) screen which gives |
|                                   | a summary of details of the       |
|                                   | review.                           |
|                                   |                                   |
|                                   | Each individual review for a      |
|                                   | customer will be represented by a |
|                                   | different IRRD screen and         |
|                                   | individual reviews can be         |
|                                   | identified by their unique Review |
|                                   | Identification number (Review ID) |
|                                   | which appears on this screen.     |
|                                   |                                   |
|                                   | Review contact details are        |
|                                   | recorded on the Review Contact    |
|                                   | Summary (IRCS) and Review Contact |
|                                   | Details (IRCD) screens.           |
|                                   |                                   |
|                                   | To initiate a contact with the    |
|                                   | customer or with a third party    |
|                                   | from the IRRD screen:             |
|                                   |                                   |
|                                   | -   select the 'go to' link by    |
|                                   |     placing an 'S' in the Review  |
|                                   |     Contacts: field, or           |
|                                   | -   go to the IRCS screen         |
|                                   |                                   |
|                                   | Note: the IRCS screen can be      |
|                                   | navigated to directly from any    |
|                                   | screen within a review.           |
+-----------------------------------+-----------------------------------+
| 2                                 | Recording contact details on the  |
|                                   | Review Contact Summary (IRCS)     |
|                                   | screen + Read more ...            |
|                                   |                                   |
|                                   | The IRCS screen is used to record |
|                                   | contacts to customers or their    |
|                                   | representatives and third         |
|                                   | parties.                          |
|                                   |                                   |
|                                   | This screen will also display a   |
|                                   | list of all previous contacts     |
|                                   | that have occurred over the life  |
|                                   | of the review and will display    |
|                                   | the method of contact, the date a |
|                                   | contact was initiated, an         |
|                                   | expected date of response and an  |
|                                   | actual date of response.          |
|                                   |                                   |
|                                   | Once in the IRCS screen, enter an |
|                                   | acceptable Method of Contact      |
|                                   | (Meth) and Date of Contact        |
|                                   | (Contacted).                      |
|                                   |                                   |
|                                   | Acceptable Methods of Contact     |
|                                   | are:                              |
|                                   |                                   |
|                                   | -   'MAI' (Mail)                  |
|                                   | -   'PHO' (Telephone)             |
|                                   | -   'FAX' (Facsimile)             |
|                                   | -   'FLD' (Field Visit)           |
|                                   | -   'OFF' (Office interview)      |
|                                   | -   'EML' (Email)                 |
|                                   | -   'SMS' (Short Message Service) |
|                                   | -   'EER' (Electronic Employer    |
|                                   |     Reference)                    |
|                                   | -   'OUT' (Outreach)              |
|                                   | -   'SEM' (Seminar)               |
|                                   | -   'REF' (Referral)              |
|                                   |                                   |
|                                   | Press [Enter]. The IRS will       |
|                                   | navigate to the Review Contact    |
|                                   | Details (IRCD) screen.            |
|                                   |                                   |
|                                   | Once a contact has been initiated |
|                                   | the review status will change     |
|                                   | from 'REG'istered to 'STA'rted. A |
|                                   | future date cannot be entered and |
|                                   | a date prior to the load date of  |
|                                   | the review (which is displayed on |
|                                   | the Review Details (IRRD) screen) |
|                                   | may not be entered. The date the  |
|                                   | review was 'STA'rted will be      |
|                                   | calculated only from the time the |
|                                   | action was performed and will not |
|                                   | be set to any previous date       |
|                                   | entered.                          |
|                                   |                                   |
|                                   | The IRCD screen is used to record |
|                                   | the actual details of the contact |
|                                   | after it is initiated from the    |
|                                   | IRCS screen. The IRCD screen also |
|                                   | shows details of a previous       |
|                                   | contact selected from the IRCS    |
|                                   | screen and can be used to record  |
|                                   | a response to the previous        |
|                                   | contact. The fields to be coded   |
|                                   | on this screen depend on the      |
|                                   | method of contact and response.   |
|                                   |                                   |
|                                   | The IRCS screen should be updated |
|                                   | on the actual date the contact is |
|                                   | issued.                           |
|                                   |                                   |
|                                   | A review must not be started by   |
|                                   | recording non-factual information |
|                                   | on the IRCS screen, for example,  |
|                                   | recording a contact took place    |
|                                   | when that contact did not exist.  |
|                                   |                                   |
|                                   | Note: the date detailed for the   |
|                                   | contact on the Review Action      |
|                                   | (IRRA) screen is not the date the |
|                                   | contact was initiated but the     |
|                                   | date the contact details were     |
|                                   | updated. The logging of 'contact  |
|                                   | initiated' on the IRRA screen is  |
|                                   | only reflecting that the IRCS and |
|                                   | IRCD screens were updated with    |
|                                   | details that a contact was        |
|                                   | initiated.                        |
+-----------------------------------+-----------------------------------+
| 3                                 | Initiating a                      |
|                                   | customer/representative/third     |
|                                   | party contact + Read more ...     |
|                                   |                                   |
|                                   | A valid method of contact must be |
|                                   | entered (see Item 2 in the        |
|                                   | Actioning a Service Update or     |
|                                   | Review in the IRS table) and      |
|                                   | person contacted must be entered  |
|                                   | for all contacts. Other details   |
|                                   | must also be recorded for the     |
|                                   | following contact methods:        |
|                                   |                                   |
|                                   | -   'MAI' (Mail) - a letter code  |
|                                   |     must also be recorded         |
|                                   | -   'PHO' (Telephone) - person    |
|                                   |     responded, date of response   |
|                                   |     and telephone or mobile       |
|                                   |     number must also be recorded  |
|                                   | -   'FAX' (Facsimile) - a fax     |
|                                   |     number must also be recorded  |
|                                   | -   'FLD' (Field Visit) - the     |
|                                   |     date of response, and address |
|                                   |     must also be recorded         |
|                                   | -   'OFF' (Office interview) -    |
|                                   |     the date of response, person  |
|                                   |     contacted and Interview (CSC) |
|                                   |     must also be recorded         |
|                                   | -   'EML' (email) - the date of   |
|                                   |     response must also be         |
|                                   |     recorded                      |
|                                   | -   'SMS' (Short Message          |
|                                   |     Service) - the date of        |
|                                   |     response and mobile number    |
|                                   |     must also be recorded         |
|                                   | -   'EER' (Electronic Employer    |
|                                   |     Reference) - the expected     |
|                                   |     date of response and letter   |
|                                   |     code must also be recorded    |
|                                   | -   'OUT' (Outreach) or 'SEM'     |
|                                   |     (Seminar) - a date of contact |
|                                   |     must be recorded              |
|                                   | -   'REF' (Referral) - a date of  |
|                                   |     contact must be recorded. A   |
|                                   |     Referral Type must also be    |
|                                   |     recorded                      |
|                                   |                                   |
|                                   | Interpreter name and language     |
|                                   | used by the interpreter may also  |
|                                   | be recorded on this screen.       |
|                                   |                                   |
|                                   | Note: for a contact method of     |
|                                   | 'MAI', the letter should be       |
|                                   | printed before the contact is     |
|                                   | recorded to ensure that the       |
|                                   | letter is actually sent.          |
|                                   |                                   |
|                                   | When the expected response date   |
|                                   | is coded the review is            |
|                                   | automatically updated to a status |
|                                   | of 'HLD' (held) and will further  |
|                                   | change status to 'DUE' (due) when |
|                                   | the expected response date is     |
|                                   | reached.                          |
|                                   |                                   |
|                                   | Each contact details screen has   |
|                                   | two lines of free text to record  |
|                                   | a brief, meaningful description   |
|                                   | of the contact. However, any      |
|                                   | customer contact that requires    |
|                                   | more detail will need to be       |
|                                   | recorded on a DOC taking into     |
|                                   | account Getting it Right minimum  |
|                                   | standards.                        |
|                                   |                                   |
|                                   | Once all valid contact details    |
|                                   | have been recorded, press         |
|                                   | [Enter].                          |
+-----------------------------------+-----------------------------------+
| 4                                 | Recording a contact method of     |
|                                   | 'REF' (Referral) + Read more ...  |
|                                   |                                   |
|                                   | A method of contact of 'REF'      |
|                                   | (Referral) may be recorded on all |
|                                   | reviews with the status of        |
|                                   | 'STA'rted, including the          |
|                                   | sub-status of 'DUE' (due) or      |
|                                   | 'HLD' (held).                     |
|                                   |                                   |
|                                   | Once 'REF' is recorded as the     |
|                                   | Method of Contact in the IRCS     |
|                                   | screen along with an acceptable   |
|                                   | Date of Contact, a Referral Type  |
|                                   | must also be recorded.            |
|                                   |                                   |
|                                   | Acceptable Referral Types are:    |
|                                   |                                   |
|                                   | -   'ATO' (Australian Taxation    |
|                                   |     Office)                       |
|                                   | -   'BDM' (Births, Deaths and     |
|                                   |     Marriages)                    |
|                                   | -   'CAO' (Complex Assessment     |
|                                   |     Officer)                      |
|                                   | -   'CIS' (International          |
|                                   |     Services)                     |
|                                   | -   'COM' (Compensation Team)     |
|                                   | -   'CSC' (Service Centre)        |
|                                   | -   'DIC' (Dept Immigration       |
|                                   |     Citizenship)                  |
|                                   | -   'DMO' (Debt Management        |
|                                   |     Officer)                      |
|                                   | -   'ECU' (Employer Contact Unit) |
|                                   | -   'FIE' (Financial Institution  |
|                                   |     Electronic)                   |
|                                   | -   'HLP' (National Help Desk)    |
|                                   | -   'IDF' (Identity Fraud Team)   |
|                                   | -   'JCA' (Assessor)              |
|                                   | -   'LSU' (Document Translation)  |
|                                   | -   'MIS' (Miscellaneous)         |
|                                   | -   'MOC' (Member of a Couple)    |
|                                   | -   'NSO' (National Support       |
|                                   |     Office)                       |
|                                   | -   'QTC' (RSS Quality Checking)  |
|                                   | -   'RMU' (Records Management     |
|                                   |     Unit)                         |
|                                   | -   'SWO' (Social Worker)         |
|                                   |                                   |
|                                   | Note: if the 'MIS' Referral Type  |
|                                   | is used, a description of the     |
|                                   | referral will be required in the  |
|                                   | text field.                       |
|                                   |                                   |
|                                   | Date of Contact can be backdated  |
|                                   | up to 7 days.                     |
|                                   |                                   |
|                                   | An expected response date must be |
|                                   | recorded.                         |
|                                   |                                   |
|                                   | Once all valid contact details    |
|                                   | have been recorded, press         |
|                                   | [Enter].                          |
+-----------------------------------+-----------------------------------+
| 5                                 | Recording a                       |
|                                   | customer/representative/third     |
|                                   | party response + Read more ...    |
|                                   |                                   |
|                                   | From the Review Contact Summary   |
|                                   | (IRCS) screen:                    |
|                                   |                                   |
|                                   | 'S'elect the appropriate contact  |
|                                   | and press [Enter]. The IRS will   |
|                                   | navigate to the Review Contact    |
|                                   | Details (IRCD) screen for that    |
|                                   | contact                           |
|                                   |                                   |
|                                   | Depending on the Method of        |
|                                   | Contact that was initiated some   |
|                                   | fields may not be able to be      |
|                                   | updated. The Person responded:    |
|                                   | field must be entered for all     |
|                                   | responses                         |
|                                   |                                   |
|                                   | For a Contact Method of 'MAI'     |
|                                   | (Mail), a date of response is     |
|                                   | required                          |
|                                   |                                   |
|                                   | For all other Methods of Contact, |
|                                   | a date of response is required if |
|                                   | an Expected Date of Response was  |
|                                   | recorded on the original contact  |
|                                   |                                   |
|                                   | Other details must also be        |
|                                   | recorded for the following        |
|                                   | Contact Methods:                  |
|                                   |                                   |
|                                   | -   'MAI' (Mail) - Date of        |
|                                   |     Response (as above) must also |
|                                   |     be recorded (For              |
|                                   |     Data-matching Program (DMP)   |
|                                   |     reviews, the response must    |
|                                   |     also indicate if the match    |
|                                   |     data was disputed and if so   |
|                                   |     if action proceeded on the    |
|                                   |     review)                       |
|                                   | -   'PHO' (Telephone) - Date of   |
|                                   |     Response must also be         |
|                                   |     recorded                      |
|                                   | -   'FAX' (Facsimile) - sender    |
|                                   |     fax number and Date of        |
|                                   |     Response must also be         |
|                                   |     recorded                      |
|                                   | -   For a contact method of 'FLD' |
|                                   |     (field visit) address and     |
|                                   |     date of response must also be |
|                                   |     recorded                      |
|                                   | -   'OFF' (Office interview) -    |
|                                   |     Date of Response must also be |
|                                   |     recorded                      |
|                                   | -   'EML' (e-mail) - e-mail       |
|                                   |     address and Date of Response  |
|                                   |     must also be recorded         |
|                                   | -   'SMS' (Short Message          |
|                                   |     Service) - mobile number and  |
|                                   |     Date of Response must also be |
|                                   |     recorded                      |
|                                   | -   'EER' (Electronic Employer    |
|                                   |     Reference) - there is no      |
|                                   |     other required coding         |
|                                   | -   'REF' (Referral) - date of    |
|                                   |     response must also be         |
|                                   |     recorded                      |
|                                   |                                   |
|                                   | A contact which has an expected   |
|                                   | date of response in the future    |
|                                   | recorded will give the review a   |
|                                   | status of 'HLD' (held).           |
|                                   |                                   |
|                                   | Recording a valid response on the |
|                                   | contact will change the Review    |
|                                   | Status back to a status of        |
|                                   | 'STA'rted.                        |
|                                   |                                   |
|                                   | Each contact details screen has   |
|                                   | two lines of free text to record  |
|                                   | a brief, meaningful description   |
|                                   | of the contact. However any       |
|                                   | customer contact that requires    |
|                                   | more detail will need to be       |
|                                   | recorded on a DOC taking into     |
|                                   | account Getting it Right minimum  |
|                                   | standards.                        |
|                                   |                                   |
|                                   | Once all valid contact details    |
|                                   | have been recorded press [Enter]. |
|                                   |                                   |
|                                   | The IRCD screen should be updated |
|                                   | on the actual date the contact is |
|                                   | received.                         |
|                                   |                                   |
|                                   | Note: to record a second or       |
|                                   | subsequent contact from the IRCS  |
|                                   | screen a new contact must be      |
|                                   | initiated.                        |
+-----------------------------------+-----------------------------------+
| 6                                 | Starting a review manually in the |
|                                   | IRS + Read more ...               |
|                                   |                                   |
|                                   | Certain reviews may be commenced  |
|                                   | manually in the IRS without       |
|                                   | initiating a contact.             |
|                                   |                                   |
|                                   | See the Resources page for Review |
|                                   | Types subject to a manual start.  |
|                                   |                                   |
|                                   | Manually starting a review is     |
|                                   | actioned by navigating to the     |
|                                   | Manage Review (IRMR) screen from  |
|                                   | within a customer's review.       |
|                                   |                                   |
|                                   | -   Update the Start Review:      |
|                                   |     field, by placing a 'Y' in    |
|                                   |     the field and press [Enter].  |
|                                   |     This will change the status   |
|                                   |     of the review from            |
|                                   |     'REG'istered to 'STA'rted and |
|                                   |     allow Attribution of the      |
|                                   |     activities to the review      |
|                                   |     without a contact having to   |
|                                   |     be initiated                  |
|                                   | -   This action will also update  |
|                                   |     the review action history on  |
|                                   |     the Review Action (IRRA)      |
|                                   |     screen                        |
+-----------------------------------+-----------------------------------+
| 7                                 | Updating the Review Reason + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | This procedure only applies to    |
|                                   | Data-Matching Program (DMP)       |
|                                   | reviews.                          |
|                                   |                                   |
|                                   | A DMP review may be changed to an |
|                                   | 'AMN' (administrative action)     |
|                                   | review in some circumstances.     |
|                                   |                                   |
|                                   | This is generally when an         |
|                                   | administration error has been     |
|                                   | made. For example, when a         |
|                                   | customer has advised details and  |
|                                   | the details were not validated or |
|                                   | recorded. The original details    |
|                                   | may be actioned and the review    |
|                                   | changed to Review Reason 'AMN'.   |
|                                   |                                   |
|                                   | To change the Review Reason:      |
|                                   |                                   |
|                                   | -   go to the Manage Review       |
|                                   |     (IRMR) screen                 |
|                                   | -   enter 'AMN' in the Review     |
|                                   |     Reason: field                 |
|                                   | -   press [Enter]                 |
|                                   |                                   |
|                                   | This will change the Category of  |
|                                   | Review from a Compliance Review   |
|                                   | (CMP) to an Administrative Action |
|                                   | (AAR) review.                     |
+-----------------------------------+-----------------------------------+
| 8                                 | Updating Review Region -          |
|                                   | Transferring a review + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Transferring a review in the IRS  |
|                                   | can be actioned on either the     |
|                                   | Manage Review (IRMR) screen or    |
|                                   | the Bulk Manage (IRBM) screen.    |
|                                   |                                   |
|                                   | To transfer a review on the IRMR  |
|                                   | screen:                           |
|                                   |                                   |
|                                   | -   update the Review Region:     |
|                                   |     field, to the gaining region  |
|                                   |     and press [Enter]             |
|                                   | -   the Workgroup and/or Position |
|                                   |     may be reset to 'Waiting' by  |
|                                   |     placing a 'W' in either field |
|                                   |     on this screen                |
|                                   |                                   |
|                                   | Note: when transferring a review, |
|                                   | the gaining office must be        |
|                                   | contacted, or a DOC recorded and  |
|                                   | a contact name added to any mail  |
|                                   | to be forwarded.                  |
|                                   |                                   |
|                                   | Generally, for Compliance and     |
|                                   | Fraud reviews, if the customer:   |
|                                   |                                   |
|                                   | -   no longer resides in the      |
|                                   |     geographical area of the      |
|                                   |     office conducting the review  |
|                                   | -   the review has not been       |
|                                   |     started (i.e. has a Review    |
|                                   |     Status of 'REG'istered), and  |
|                                   | -   the review is less than 30    |
|                                   |     days from the date loaded to  |
|                                   |     the IRS (see Review Details   |
|                                   |     (IRRD) screen)                |
|                                   |                                   |
|                                   | The review may be transferred to  |
|                                   | the gaining region/area which     |
|                                   | conducts reviews of that Review   |
|                                   | Type.                             |

Integrated Review System (IRS) Service Updates and Reviews

Table 2: This table describes the steps for processing Service Updates
and Reviews in the Integrated Review System (IRS).

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Bulk transferring of reviews +    |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | To transfer a group of reviews    |
|                                   | via the Bulk Manage (IRBM)        |
|                                   | screen, a valid search must be    |
|                                   | performed on the Review Totals    |
|                                   | (IRRT) screen.                    |
|                                   |                                   |
|                                   | This screen will return various   |
|                                   | totals of reviews depending on    |
|                                   | the search criteria entered.      |
|                                   |                                   |
|                                   | Reviews can also be transferred   |
|                                   | via the Review Enquiry (IRRE)     |
|                                   | screen. Up to 10 records may be   |
|                                   | selected at a time for 'BAF'ing   |
|                                   | (Bulk Action Facility) from this  |
|                                   | screen by placing a 'B' beside    |
|                                   | the desired record.               |
|                                   |                                   |
|                                   | To transfer a particular total of |
|                                   | reviews via the IRRT screen:      |
|                                   |                                   |
|                                   | -   select the BAF: field, with a |
|                                   |     'Y'                           |
|                                   | -   place the cursor on the total |
|                                   |     that is to be moved           |
|                                   | -   press [Enter]                 |
|                                   |                                   |
|                                   | The IRS will navigate to the IRBM |
|                                   | screen.                           |
|                                   |                                   |
|                                   | -   The Review Region may be      |
|                                   |     changed (and any other        |
|                                   |     details on this screen (e.g.  |
|                                   |     keywords). However, only one  |
|                                   |     action at a time is permitted |
|                                   |     other than keyword and        |
|                                   |     another action)               |
|                                   | -   Press [Enter]. The changes    |
|                                   |     will be made to all reviews   |
|                                   |     in the total selected and the |
|                                   |     IRS will return to the IRRT   |
|                                   |     screen with the details       |
|                                   |     refreshed                     |
|                                   | -   The Workgroup and/or Position |
|                                   |     may be reset to 'Waiting' by  |
|                                   |     placing a 'W' in either field |
|                                   |     on this screen                |
|                                   |                                   |
|                                   | Note: any actioning will appear   |
|                                   | on the logged details for each    |
|                                   | individual review on the Review   |
|                                   | Action (IRRA) screen.             |
+-----------------------------------+-----------------------------------+
| 2                                 | Updating Review Method + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Updating the Review Method is     |
|                                   | actioned from the Manage Review   |
|                                   | (IRMR) or Bulk Manage (IRBM)      |
|                                   | screens.                          |
|                                   |                                   |
|                                   | If the review is either           |
|                                   | 'CAN'celled, 'FIN'alised or       |
|                                   | 'COM'pleted, then the Review      |
|                                   | Method: field will be protected.  |
|                                   |                                   |
|                                   | To update, change the field to an |
|                                   | acceptable value. Acceptable      |
|                                   | Review Methods are:               |
|                                   |                                   |
|                                   | -   'CAO' (Complex Assessment     |
|                                   |     Officer)                      |
|                                   | -   'FIL' (File (see below)       |
|                                   | -   'FLD' (Field Visit)           |
|                                   | -   'MAI' (Mail)                  |
|                                   | -   'NET' (Internet)              |
|                                   | -   'OFF' (Office interview)      |
|                                   | -   'OUT' (Outreach)              |
|                                   | -   'PHO' (Telephone)             |
|                                   | -   'SEM' (Seminar)               |
|                                   | -   'SUR' (Surveillance) (see     |
|                                   |     below for further             |
|                                   |     information)                  |
|                                   |                                   |
|                                   | Changing of a Review Method will  |
|                                   | be logged as a review action on   |
|                                   | the Review Action (IRRA) screen.  |
|                                   |                                   |
|                                   | 'FIL'                             |
|                                   |                                   |
|                                   | A Review Method of 'FIL' should   |
|                                   | only be used for reviews which    |
