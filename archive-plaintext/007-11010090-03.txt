|                                   |     ex-partner income comparison  |

|                                   |     FAO system again applies the  |
|                                   |     more favourable income.       |
|                                   |     Before 8 December 2008, there |
|                                   |     were some problems with this  |
|                                   |     process; see Item 8 in the    |
|                                   |     Debts and other circumstances |
|                                   |                                   |
|                                   | See the Resources page for an     |
|                                   | example.                          |
|                                   |                                   |
|                                   | If the ex-partner's income falls  |
|                                   | into another interim              |

| 7                                 | Changes to system processing -    |
|                                   | customer separated at 30 June of  |
|                                   | relevant year + Read more ...     |
|                                   |                                   |
|                                   | The partner is treated as an      |
|                                   | ex-partner for reconciliation     |
|                                   | purposes.                         |
|                                   |                                   |
|                                   | Clause 3A, Schedule 3 of the A    |
|                                   | New Tax System (Family            |
|                                   | Assistance) Act 1999 limits the   |
|                                   | effect of ex-partner income at    |
|                                   | reconciliation to either a top-up |
|                                   | or nil adjustment. This means     |
|                                   | that ex-partner income cannot     |
|                                   | cause a debt for any period the   |
|                                   | customer was with their           |
|                                   | ex-partner in the financial year  |
|                                   | that the separation occurred. See |
|                                   | the References page.              |
|                                   |                                   |
|                                   | When family assistance            |
|                                   | reconciliation first occurred in  |
|                                   | 2001, the system solution used to |
|                                   | apply Clause 3A was to suppress   |
|                                   | any debts that occurred in an     |
|                                   | ex-partner period. Although this  |
|                                   | ensured that ex-partner income    |
|                                   | could not cause a reconciliation  |
|                                   | debt, it also meant that other    |
|                                   | non-income related debts were     |
|                                   | suppressed (for example debts     |
|                                   | caused by circumstance changes,   |
|                                   | maintenance income). There have   |
|                                   | been a number of changes to the   |
|                                   | way the system treats ex-partner  |
|                                   | periods since family assistance   |
|                                   | reconciliation first commenced.   |
|                                   |                                   |
|                                   | In the September and December     |
|                                   | 2005 releases (for FTB) and June  |
|                                   | 2006 (for CCB), a new process was |
|                                   | introduced for 2002-2003 onwards  |
|                                   | so that only income related debts |
|                                   | were prevented in ex-partner      |
|                                   | periods. The new process was      |
|                                   | known as the ex-partner period    |
|                                   | income comparison. See Item 8 in  |
|                                   | the Debts and other circumstances |
|                                   | table. Unfortunately, there were  |
|                                   | some flaws in this process and in |
|                                   | some circumstances it was still   |
|                                   | possible for a customer to incur  |
|                                   | an income related debt in an      |
|                                   | ex-partner period. Changes to     |
|                                   | correct this process were         |
|                                   | implemented in the December 2008  |
|                                   | release.                          |
|                                   |                                   |
|                                   | The following is a summary of the |
|                                   | system changes that have occurred |
|                                   | for customers who were separated  |
|                                   | at 30 June of relevant year       |
|                                   | (ex-partner):                     |
|                                   |                                   |
|                                   | -   September and December 2005   |
|                                   |     system releases: ex-partner   |
|                                   |     income comparison process     |
|                                   |     introduced for FTB customers  |
|                                   |     for 2002-2003 onwards         |
|                                   | -   16 June 2007 system release:  |
|                                   |     partial reconciliation        |
|                                   |     introduced and timing of      |
|                                   |     interim reconciliation was    |
|                                   |     changed for FTB customers     |
|                                   | -   10 September 2007 system      |
|                                   |     release: partial              |
|                                   |     reconciliation introduced and |
|                                   |     timing of interim             |
|                                   |     reconciliation was changed    |
|                                   |     for CCB reduced fees          |
|                                   |     customers (to align with FTB) |
|                                   | -   8 December 2008 system        |
|                                   |     release: changes introduced   |
|                                   |     to correct the ex-partner     |
|                                   |     income comparison process for |
|                                   |     FTB and CCB                   |
|                                   |                                   |
|                                   | Re-reconciliation may result in a |
|                                   | debt or reduced top-up due to     |
|                                   | loss of qualification or          |
|                                   | maintenance income.               |
|                                   |                                   |
|                                   | Before 8 December 2008 system     |
|                                   | release, re-reconciliation may    |
|                                   | have also resulted in a debt due  |
|                                   | to new income details, but this   |
|                                   | was not correct processing. See   |
|                                   | Item 8 in the Debts and other     |
|                                   | circumstances table.              |

| 8                                 | Ex-partner period income          |
|                                   | comparison + Read more ...        |
|                                   |                                   |
|                                   | The ex-partner period income      |
|                                   | comparison is performed whenever  |
|                                   | an interim or final               |
|                                   | reconciliation result is          |
|                                   | calculated for customers who      |
|                                   | separated from their partner      |
|                                   | during the relevant financial     |
|                                   | year.                             |
|                                   |                                   |
|                                   | This comparison is designed to    |
|                                   | ensure that the customer does not |
|                                   | incur an income related           |
|                                   | overpayment in an ex-partner      |
|                                   | period.                           |
|                                   |                                   |
|                                   | First ex-partner period income    |
|                                   | comparison                        |
|                                   |                                   |
|                                   | The FAO system compares the       |
|                                   | customer's entitlement using      |
|                                   | income estimates provided during  |
|                                   | the relevant year with their      |
|                                   | entitlement using adequate        |
|                                   | income. For FTB lump sum claims,  |
|                                   | the system compares the           |
|                                   | customer's entitlement using the  |
|                                   | grant income with their           |
|                                   | entitlement using new (non-grant) |
|                                   | income.                           |
|                                   |                                   |
|                                   | The income used is that which     |
|                                   | results in the most favourable    |
|                                   | outcome for the customer. This is |
|                                   | then known as the favourable      |
|                                   | income.                           |
|                                   |                                   |
|                                   | The FAO Calculation Period Income |
|                                   | Summary (FICS) screen displays    |
|                                   | the income used as a result of    |
|                                   | the ex-partner period income      |
|                                   | comparison process and includes a |
|                                   | description of the income that    |
|                                   | was used. For example:            |
|                                   |                                   |
|                                   | -   Ex-partner Comparison:        |
|                                   |     Favourable (Adequate)         |
|                                   |                                   |
|                                   | Subsequent ex-partner period      |
|                                   | income comparisons                |
|                                   |                                   |
|                                   | Another comparison will occur     |
|                                   | every time there is a             |
|                                   | re-reconciliation.                |
|                                   |                                   |
|                                   | From 8 December 2008, the FAO     |
|                                   | system compares the customer's    |
|                                   | entitlement using the new         |
|                                   | adequate income with their        |
|                                   | entitlement using the income      |
|                                   | deemed most favourable in the     |
|                                   | previous reconciliation, and      |
|                                   | again uses the more favourable    |
|                                   | income.                           |
|                                   |                                   |
|                                   | System limitation prior to 8      |
|                                   | December 2008                     |
|                                   |                                   |
|                                   | The system was only able to       |
|                                   | compare Estimates As Provided     |
|                                   | with the latest Adequate Income.  |
|                                   | This meant that in some           |
|                                   | circumstances the customer was    |
|                                   | disadvantaged. This occurred most |
|                                   | commonly where the customer's     |
|                                   | and/or ex-partner's Adequate      |
|                                   | Income has changed.               |
|                                   |                                   |
|                                   | See the Resources page for an     |
|                                   | example.                          |
|                                   |                                   |
|                                   | A clean-up of cases where the     |
|                                   | customer was disadvantaged due to |
|                                   | this system limitation was        |
|                                   | conducted in early 2009. Cases    |
|                                   | identified before the clean-up    |
|                                   | were manually adjusted.           |
|                                   |                                   |
|                                   | Ex-partner period income          |
|                                   | comparison overrides              |
|                                   |                                   |
|                                   | In some cases it will be          |
|                                   | necessary to override the income  |
|                                   | deemed most favourable in an      |
|                                   | ex-partner period, for example,   |
|                                   | where a customer has been         |
|                                   | disadvantaged due to the system   |
|                                   | limitation described above.       |
|                                   |                                   |
|                                   | This is done using the FTB        |
|                                   | Ex-Partner Period Override        |
|                                   | (FEXPO) and CCB Ex-Partner Period |
|                                   | Override (KEXPO) screens. Access  |
|                                   | to update these screens is        |
|                                   | limited to selected network and   |
|                                   | National Support Office staff,    |

|                                   | FICS screen as Override.          |

|                                   | If income details or an           |
|                                   | ex-partner period change after an |
|                                   | override has been coded, the      |
|                                   | record will be selected for       |
|                                   | manual intervention so that the   |
|                                   | override income can be updated if |
|                                   | required.                         |
