| 11                                | Calculate the start date for      |
|                                   | payment + Read more ...           |
|                                   |                                   |
|                                   | For a new claim where the         |
|                                   | customer was not previously on a  |
|                                   | payment the system will           |
|                                   | automatically calculate the start |
|                                   | date.                             |

|                                   | changing the system calculated    |
|                                   | start date.                       |
|                                   |                                   |
|                                   | The system will apply the         |
|                                   | backdating rule and calculate the |
|                                   | date of commencement as Date Paid |
|                                   | To (DPT)+1, even if the date of   |
|                                   | the claim is within 14 days of    |
|                                   | DPT.                              |
|                                   |                                   |
|                                   | When the cancellation date is     |
|                                   | outside 14 days of the claim      |
|                                   | date, do not apply DPT+1. The     |
|                                   | correct start date for these      |
|                                   | claims is the date the customer   |
|                                   | reclaims payment.                 |
|                                   |                                   |
|                                   | To correct the start date where   |
|                                   | system has applied an incorrect   |
|                                   | date                              |
|                                   |                                   |
|                                   | Where the correct date should be  |
|                                   | after the DPT+1, code the correct |
|                                   | start date in the Manual Start    |
|                                   | Date field on the NDC screen      |
|                                   |                                   |
|                                   | Where the start should be before  |
|                                   | the DPT+1 check the following:    |
|                                   |                                   |
|                                   | -   will the customer be paid for |
|                                   |     a period in which they were   |
|                                   |     previously receiving a        |
|                                   |     payment                       |
|                                   | -   was the previous payment      |
|                                   |     cancelled as the customer was |
|                                   |     not eligible, resulting in a  |
|                                   |     debt being raised             |
|                                   |                                   |
|                                   | Once checks are completed, apply  |
|                                   | the following to correct the      |
|                                   | date:                             |
|                                   |                                   |
|                                   | do not code a Manual Start Date   |
|                                   | field on the NDC screen. This     |
|                                   | will cause the start date to      |
|                                   | overlap with the previous payment |
|                                   | dates, which may:                 |
|                                   |                                   |
|                                   | -   prevent the claim from        |
|                                   |     granting properly, or         |
|                                   | -   lead to a data corruption on  |
|                                   |     the record                    |
|                                   |                                   |
|                                   | pay the days the customer was     |
|                                   | entitled to prior to the DPT+1,   |
|                                   | as a one-off payment              |
|                                   |                                   |
|                                   | code the DPT+1 date in the Manual |
|                                   | Start Date field on the NDC       |
|                                   | screen, and                       |
|                                   |                                   |
|                                   | record a DOC detailing the steps  |
|                                   | taken and include the reason for  |
|                                   | the start date determination      |
|                                   |                                   |
|                                   | Early Claims                      |
|                                   |                                   |

|                                   | date of receipt/submission of the |
|                                   | claim.                            |
|                                   |                                   |

|                                   | DOR.                              |

|                                   |     calculated start date on NDC  |
|                                   | -   consider any waiting periods  |
|                                   | -   if required, code the Manual  |
|                                   |     Start Date on the NDC screen  |
