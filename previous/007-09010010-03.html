1

Incorrect Nominated Start Date (NSD) + Read more ...

This procedure only applies when:

  • an administration error has occurred, and
  • the customer's Parental Leave Pay (PPL) claim has been processed with an incorrect nominated start date, and
  • payments have already commenced

It does not apply to customers who have requested a change to their NSD after the PPL has been delivered. A separate procedure applies if the customer requests a reassessment of their PPL claim.

For Service Centre and Smart Centre staff not trained in PPL claims processing:

  • Send details to FCC via Fast Note, select Auto text, use Families > Updates > PPL incorrect NSD. Note: if customer is overseas, manually add 'FICREQ' to the Fast Note before sending
  • Procedure ends here

Smart Centre staff processing claims

The customer must be contacted and the process discussed so the customer is aware that any PPL paid after the incorrect NSD may be raised as an overpayment.

Has the customer's nominated start date been incorrectly coded and the PPL payment has:

  • been cancelled, and:
    • the PPL period is prior to 1 July 2020 (pre Flexible PPL claim), go to Step 2
    • the PPL period is after 1 July 2020, go to Step 3
  • already commenced, go to Step 3
  • not yet commenced, see Step 3 in Table 1 of Customer notifies a change of circumstances for Parental Leave Pay (PPL). Procedure ends here

2

PPL cancelled and original claim granted for period prior to 1 July 2020 + Read more ...

If the PPL payment has been cancelled and the PPL claim was granted for a PPL period prior to 1 July 2020 a manual work around may be required to correct the Nominated Start Date (NSD).

Note: the below workaround is only to be used when the original claim was granted for a PPL period prior to 1 July 2020. Example: to implement an ARO/AAT decision. For all other claims, go to Step 3.

Work around:

  • In Customer First go to the Parental Leave Pay (PPL) Claim Summary (P1CS) page and code 'D' next to the relevant PPL claim line
  • This will display the PPL Employment Dates (P1ED) page
  • Code the P1ED page with a return to work date that is the same as the customers original grant date. Even if the cancellation is for another reason. This will then cancel the customer's payment from the date of grant
  • If it is 2 years or more from the child's date of birth, the PPL will not reassess. To get the PPL to reassess manually, code the PPL Claim Action (P1CA) screen with the customer's original grant date
  • Go to the Assessment Results (AR) screen. Any paid amounts of PPL on the AR will be calculated as a debt
    • If the paymaster was the employer the debt will be transferred to the customer's employer
    • If the paymaster was Services Australia the debt will be on the customer's record
    • Inhibit the customer's PPL advice via the Coding Assessment Consequences (ASC) screen
    • Finalise activity

Once a decision has been made to re-grant the customer’s PPL from the new nominated start date, index a new claim on the customer's record with the correct nominated start date. As this new claim will be more than 12 months in the past, see Exception processing of Paid Parental Leave scheme claims for processing.

3

Select the relevant PPL claim + Read more ...

Once a decision has been made that the customer's PPL claim has been processed with an incorrect nominated start date, reject the original PPL claim. This allows correct claim processing to occur.

In Customer First, go to the Parental Leave Pay (PPL) Claim Summary (P1CS) page and code 'A' next to the relevant PPL claim line.

This will display the PPL Claim Action (P1CA) page.

  • Code:
    • Action field using 'REJ' with the Reason field as 'OTH' (do not use 'CAN' as this will not produce the correct result)
    • Level field as 'Claim'
    • Effect date: Use the original grant date for rejection date
  • Record details of the reason in the Notes field
  • Complete the Source and DOR field and navigate to the Assessment Results (AR) screen

Go to Step 4.

4

Finalise activity + Read more ...

On the AR screen, any paid amounts of PPL will be calculated as a debt.

  • If the paymaster was:
    • the employer, the debt will be transferred to the customer's employer
    • the agency, the debt will be on the customer's record
  • Inhibit the customer's PPL advice via the Coding Assessment Consequences (ASC) screen
  • Take note of the calculated debt information as this will be required when re-granting

Finalise activity.

Go to Step 5.

5

PPL new claim activity and assessment + Read more ...

Once a decision has been made to grant the customer’s PPL from the new nominated start date, index a new claim on the customer's record with the correct nominated start date.

A Social Online Application (SOA) shell ‘Work Item’ in Process Direct will need to be created.

This contains no claim information apart from:

  • the CRN
  • personal details, and
  • the receipt date (date of claim)

Update the claim information manually within the Work Item using:

  • information from the original claim
  • new information, and
  • associated documentation

Contact the customer if there are any concerns that their circumstances have changed.

To create SOA shell in Process direct for a new claim:

  • Go to the Process Direct Landing page and select SOA shell tile
  • Key the appropriate details in the:
    • Benefit type
    • Customer CRN
    • Date of receipt (DOR is the date all the required information was provided)
  • Select Create Claim
  • An activity with a claim ID will be generated without any claim information
  • Update claim screens as required. See Processing standalone and combined claims for family assistance and/or Paid Parental Leave scheme payments

Is the customer's Nominated Start Date (NSD) changing to a date later than date paid to +1?

  • Yes, go to Step 6
  • No, go to Step 7

6

Finalise PPL claim + Read more ...

Go to the Entitlement (ELD) screen in Process Direct to finalise the activity when satisfied with the outcome.

When finalising, the PPL claim outcome should be:

  • PPL//CUR (PPL Current) adjusted arrears are to be issued to the customer. Go to Step 7
  • PPL/ASS-EIA (PPL Assessed - Employer acceptance required). Go to Step 9
  • PPL/ASS-ASD (PPL Assessed - Awaiting start date). Finalise the claim and document the record. Procedure ends here

7

Adjust arrears + Read more ...

The customer's PPL arrears must be adjusted for the same period in which a debt shell (either on the customer or employer’s record) has been raised after the rejection activity was finalised.

In Customer First:

  • go to the Debt List (OPDL) screen, and
  • select the debt activity from the original claim rejection. This opens General Information (OPGI) screen
  • go to the Activity List (AL) screen to check the amount and period of the debt

In Process Direct:

  • key 'DEBTDTL' (Debt Summary) in Super Key, and
  • 'view Details' for the amount and period of the debt

To reduce the arrears within the new claim, in Process Direct:

  • Key 'RAC' in the Super Key field
  • On the Customer Daily Rate Component (RAC) screen
    • Go to the relevant date range for the debt that was generated by the rejection activity
    • Key 0 in the $ Man Adj field and ADJ in the Man Adj Reason field for the relevant dates
    • Select Update to save the update
  • Go to the ELD screen to check the outcome

Note: this may need to be done for multiple date ranges.

The customer's arrears should be reduced by the amount of the debt that was raised.

Once the arrears have been adjusted, finalise the new claim activity. Record the details of the amount adjusted and the relevant period of adjustment in the Finalisation DOC/Notes.

In Customer First, go to the AL screen and select the new debt activity. Record notes on the activity advising no duplicate payment exists and arrears have been adjusted. Include the AMR of the activity.

Is the agency the payer for PPL?

  • Yes, go to Step 8
  • No, go to Step 9

8

Refer the debt to the debt team + Read more ...

Refer the debt to the appropriate debt team by sending a Fast Note to have the debt team finalise the debt No Debt (FND). Lookup local Processing Service details for: Debt Raising.

  • Auto Text use Debts > DMB Raising > Debt Raising ACTion DOC Request
  • Select Confirm
  • Edit auto text:
    • 'Customer’s PPL processed with incorrect Nominated Start Date.
    • Please FND Debt ID XX for PPL $XX for period XX.
    • Corrective action has been taken in AMR XX.
    • Arrears have been offset against undetermined debt.'
  • Select Continue
  • Procedure ends here

9

Hand off to Paid Parental Leave Employer Processing Team (PPLEPT) + Read more ...

When the agency is not the payer for PPL, the employer must be contacted by PPLEPT to provide acceptance.

In Customer First create a Fast Note select Auto Text use Families > Updates > PPL Regrant with correct NSD. This will automatically allocate a work item with the keyword 'REGRANT' to an appropriately trained member of the PPLEPT:

PPLEPT staff only - when allocated a 'PPL Re-grant with correct NSD', see Step 1 in Table 2

2

Employer already accepted their role to provide PPL to the customer via Business Hub? + Read more ...

Has the employer already accepted their role or provided written acceptance to provide PPL to the customer via Business Hub in the new claim?

  • Yes, arrears are unable to be offset against the employer debt.

    • If the employer has been paid duplicate arrears, see Parental Leave Pay (PPL) debt raising, debt offsetting and recovery. Procedure ends here
  • No, go to Step 3

3

Attempt phone contact with the employer + Read more ...

Use the information provided to make a genuine attempt to contact the employer to advise:

  • that there has been a change in the employee's NSD
  • if the new NSD is in the future, arrears are unable to be offset against any undetermined debt shell and the employer may have an outstanding debt
  • if the employer had provided an acceptance notice for their employee’s previous claim, they only need to provide a new written acceptance to confirm:
    • bank account, and
    • employee pay cycle details
  • if the employer had not previously provided acceptance, they can accept their role to provide PPL to the customer via Business Hub

Employers can log into PRODA to:

  • access the Business Hub, and
  • upload the written acceptance notice
  • accept their role to provide PPL to the customer

Is the first contact with the employer successful?

  • Yes, to allow adjustments to any arrears or debts manually, tell the employer not to provide online acceptance through Business Hub. They should upload written acceptance notice via Business Hub however.
    • Annotate the Fast Note with conversation details. If required, place Fast Note on hold for 2 business days to allow time for the employer to provide written acceptance
    • Go to Step 4
  • No:
    • Annotate the Fast Note with the first contact attempt, and
    • Hold the Fast Note until the next business day. Go to Step 5

4

When Fast Note follow-up is allocated or written acceptance provided + Read more ...

Has the employer provided written acceptance?

  • Yes, upload the acceptance to the employer's record, annotate and close the Fast Note. Go to Step 7
  • No, if 2 genuine contact attempts:
    • have been made with the employer, annotate and close the Fast Note. An MFU will create on day 15. When the MFU is due, see Assisted dispute resolution for employer determination for Parental Leave Pay (PPL)
    • have not yet been made with the employer, attempt a second contact. Go to Step 5

5

Attempt second phone contact with the employer + Read more ...

Attempt to phone the employer on 2 separate occasions. Use all available contact numbers recorded on the system, to advise the employer:

  • that there has been a change in the employee's NSD
  • if the new NSD is in the future, arrears are unable to be offset against any undetermined debt shell and the employer may have an outstanding debt
  • if the employer had provided an acceptance notice for their employee’s previous claim, they only need to provide a new written acceptance to confirm:
    • bank account, and
    • employee pay cycle details
  • if the employer had not previously provided acceptance, they can accept their role to provide PPL to the customer via Business Hub

Has phone contact with the employer been successful?

  • Yes, to allow adjustments to any arrears or debts manually, tell the employer not to provide online acceptance through Business Hub. They should upload written acceptance notice via Business Hub however.
    • Annotate the Fast Note with conversation details. If required, place Fast Note on hold for 2 business days to allow time for the employer to provide written acceptance
    • Go to Step 6
  • No:
    • Annotate the Fast Note with attempt made to contact and close the Fast Note
    • An MFU will create on Day 15, see Assisted dispute resolution for employer determination for Parental Leave Pay (PPL)

6

When Fast Note follow-up is allocated or written acceptance provided after second contact + Read more ...

Has the employer provided written acceptance?

  • Yes:
    • Upload the acceptance to the employer's record
    • Annotate and close the Fast Note
    • Go to Step 7
  • No, annotate and close the Fast Note. An MFU will create on day 15, see Assisted dispute resolution for employer determination for Parental Leave Pay (PPL)

7

Manual acceptance provided + Read more ...

Before coding manual acceptance, check and note the amount and period of the debt on the employer's record that relates to the original claim rejection. The customer's PPL arrears will need to be adjusted for the period in which a debt shell has been raised after the rejection activity was finalised.

The Paid Parental Leave Claim Summary (P1CS) will display the current PPL claim information. In Customer First enter 'N' beside the relevant claim to go to the Paid Parental Leave Employer Notified Details (P1ERD) screen to record the manual acceptance by completing the following fields:

  • Employer accepts to pay customer, select Accept
  • complete Source and Receipt Date
  • select Continue
  • on Paid Parental Leave Employer Payroll List (P1ERDL), select Continue
  • on Paid Parental Leave Employer Payroll Confirmation (P1ERDC), select Yes to confirm employer determination payroll, and select Continue
  • check existing information is correct. Make any necessary updates
  • go to the Assessment Results (AR) screen

Is the NSD in the future?

  • Yes, a future NSD will not generate arrears. Go to Step 9
  • No, manually adjust the arrears. Go to Step 8

8

Manually adjust arrears + Read more ...

To reduce the arrears, in Customer First:

  • within the activity go to the AR screen
  • on the AR screen select Menu tab > Paid Parental Leave > Rate details (RATS)
  • select the relevant period to be adjusted
  • Daily Rate Component (RAC) will display. Select the period to be adjusted which will expand the RAC details
  • adjust the amount in the Manual Adj field for the relevant period with a Reason of ADJ
  • select Continue. Note: this may need to be done for multiple date ranges
  • on the AR screen, Finalise the activity
  • select Continue
  • annotate and close the Fast Note

Is there a debt shell on the employer's record?

  • Yes, any arrears adjusted in the manual acceptance activity on the employee's record will also require adjusting in the undetermined debt shell on the employer's record. To adjust the debt, see Parental Leave Pay (PPL) debt raising, debt offsetting and recovery
  • No, procedure ends here

9

NSD in re-grant claim is a future date + Read more ...

A future NSD will not generate an immediate arrears payment. Any future awaiting payments are unable to be offset against any undetermined debt shells.

  • Finalise the manual acceptance
  • Annotate the Fast Note, and close

If debt exists, see Parental Leave Pay (PPL) debt raising, debt offsetting and recovery.
