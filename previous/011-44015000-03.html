Description

F2F Service Delivery network + Read more ...

  • Enrolment work received from a customer when attending a service centre

In the interim:

  • Call the Health Service Delivery Division (HSDD) Eligibility team, and
  • Ask for a copy of the customer's VEVO details

The HSDD Eligibility Service Officer will email details.

Do not escalate an enrolment application because of no VEVO access.

Health Service Delivery Division (HSDD) Eligibility team work types + Read more ...

  • Work received via PaNDA
  • Applicants for permanent residency
  • Ministerial Orders including parent visa categories
  • Newborn Child Declaration (FA081) enrolments
  • Child Protection Agency (DCP) requests
  • Pre-release prisoners
  • Third Medicare card requests
  • A Child Dental Benefits Schedule (CDBS) enquiry
  • Diplomat enrolments
  • Public Trustee/POAs

Note: HSDD Eligibility Service Officers must seek help from an Eligibility PSO.

3

Work types done by the Medicare Public helpdesk + Read more ...

The PSO will confirm if the query is a helpdesk work type. If escalation is required the PSO will:

  • escalate documents to Medicare Consumers Program Support helpdesk or
  • email enquiry where appropriate.

Note: Applications in PaNDA must be escalated via PaNDA using work type Medicare Eligibility Nat.Office Escalations.

These can include:

  • 410 visa holders
    • issued or applied for prior to 1 December 1998, and
    • who have never had a Medicare card
  • The removal of a 'Pending Bridging' visa holder
  • Complex enrolments that need to be escalated to the Department of Home Affairs
  • Victims of People Trafficking programme
  • Requests that relate to an Apprehended Violence Order (AVO) where children under 15 are involved
  • Request to remove children under 15 from a family Medicare card
  • Fourth (or more) Medicare card requests
  • Postcode locality update
  • Person enrolled on the incorrect card
  • Process Record Amendment (PRA) requests
  • Surrogacy arrangement where documentation provided does not meet standard requirements

Face to Face (F2F) Service Officer escalation for Document Verification System (DVS) checks

Table 2

Step

Description

1

Face to Face (F2F) Service Officers who do not hold a DVS licence + Read more ...

  • access DVS request link

2

Submit DVS request + Read more ...

  • select Submit DVS Request
  • enter PaNDA Work Item ID
  • select type of DVS consent
  • complete confirmation messages
  • select Submit

Once the request has been submitted it will appear under ‘My DVS Requests’.

3

DVS request completed + Read more ...

Once DVS request complete, an email will be sent to the processing Service Officer with the outcome.

Match results include:

  • Yes
    • Verification successful
    • Complete enrolment
  • No
    • Verification has been unsuccessful (2 attempts to verify have been made)
    • Return the original document to the customer with relevant letter
  • D (details input are recorded with issuer, but cannot be verified)
    • Verification has been unsuccessful
    • Return the original document to the customer with relevant letter
  • S (system issues)
    • Add processing note in PaNDA ‘DVS unsuccessful as system is down, unable to process’
    • Unassign the work item
  • Not Required
    • Customer can be identified via VEVO or Immigration data link
    • Complete enrolment

Where a request is submitted and cannot be processed due to the following reasons:

  • Acceptable identification not provided
  • Scan not legible
  • Form not signed
  • Form does not include DVS consent

Service officer will receive an email detailing appropriate action to take.

Face to Face (F2F) Service Officer escalation

Table 3

Step

Description

1

Try to resolve the enquiry + Read more ...

A F2F Service Officer must try to resolve the customer enquiry first by using:

  • Operational Blueprint
  • staff messages such as Daily Comms
  • comments in the Consumer Directory Maintenance System (CDMS)
  • a VEVO system check

2

Contact the HSD Eligibility team for help + Read more ...

If the F2F Service Officer is unable to resolve the customer’s enquiry call the HSD Eligibility team.

If the HSDD Eligibility Service Officer is not able to help with the enquiry the HSDD Eligibility Service Officer can call an Eligibility PSO. F2F Service Officer is put on hold while this occurs. The following process will take place:

  • PSO gives advice to the HSDD Eligibility Service Officer
  • HSDD Service Officer gives this advice to the F2F Service Officer
  • F2F Service Officer can then resolve the enquiry with the customer

3

The PSO is unable to help with the enquiry + Read more ...

If it is not urgent, the F2F Service Officer escalates the enquiry to the HSDD Eligibility Team. To do this the F2F Service Officer will:

  • Scan all of the enrolment documents
  • In the CDMS:
    • Add a Comment at the Group level (if the customer has a Medicare number). For example, the enrolment has been escalated
    • Add the HSDD Eligibility Service Officer login ID details in the comment

This work will be finalised by an HSDD Eligibility Service Officer or PSO.

If the enquiry is urgent, go to Step 4

4

Escalate when the enquiry is urgent + Read more ...

If an enquiry requires an immediate response the PSO will call the Medicare Enrolment and Eligibility Helpdesk for advice.

Note: if the helpdesk cannot resolve the enquiry over the phone refer to Escalations from a PSO to the helpdesk

Escalations from a HSDD Eligibility Service Officer to the PSO

Table 4

Step

Description

1

Escalating to the PSO + Read more ...

The HSDD Eligibility Service Officer must try to resolve the enquiry first by using all available resources.

If the enquiry is unable to be resolved, the HSDD Eligibility Service Officer will call an Eligibility PSO for help.

If the PSO asks the HSDD Eligibility Service Officer to send them the enquiry, the following details may be included in the request:

  • The file name Work Item ID
  • A Medicare number
  • The applicant’s name
  • Applications/documents/VEVOs if relevant

Note: Forward the details to the PSO via the digitised workflow

2

PSO response to HSDD Eligibility Service Officer + Read more ...

For a phone enquiry:

  • The PSO will provide a verbal response to the HSDD Eligibility Service Officer to complete the PSO tool template and email it to the HSDD Eligibility Service Officer
  • The HSDD Eligibility Service Officer will tell the F2F Service Officer how to complete the enrolment

For a workflow enquiry:

  • The PSO will:
    • assess the enquiry
    • send a written response back to the HSDD Eligibility Service Officer via the workflow
    • complete the PSO tool template and email it to the HSDD Eligibility Service Officer
  • The HSDD Eligibility Service Officer will:
  • retrieve the piece of work from the workflow, and
  • complete the enquiry

3

Escalate an urgent query to the PSO + Read more ...

Some examples of an urgent enquiry are:

  • a Ministerial or Ombudsman complaint
  • a potential breach of privacy
  • a Child Protection Agency request for a Medicare card for a child that needs urgent medical attention
  • any enrolment errors that
    • have a negative impact on the customer, or
    • stop them from claiming a Medicare benefit

When a HSDD Eligibility Service Officer cannot resolve an urgent query, they call an Eligibility PSO. They will explain that the query is urgent, and give the PSO all of the details.

The PSO will either:

  • tell them how to resolve the enquiry, or
  • ask them to send the details to the PSO via PaNDA using work type Medicare Eligibility escalations

If they cannot contact the Eligibility PSO, they should direct the urgent query to their Team Leader.

Escalations from a PSO to the helpdesk

Table 5

Step

Description

1

Escalate to the helpdesk + Read more ...

If the PSO is unable to resolve the enquiry, the PSO will:

  • escalate the enquiry via PaNDA using work type Medicare Eligibility Nat Office Escalations, or
  • email Medicare Consumers Program Support - Medicare Consumers Section - Medicare and Veterans Health Branch including:
    • all enquiry details, and
    • any forms and documents

The helpdesk will send a response within 10 business days.

2

Escalate an urgent enquiry + Read more ...

If an Eligibility PSO needs help with an urgent query, the Eligibility PSO can call Medicare Consumers Program Support helpdesk.

The PSO will explain that the query is urgent and give the helpdesk all of the details.

The helpdesk will:

  • tell them how to resolve the enquiry, or
  • ask them to send all of the details

Email requests

The PSO may email details as High Importance and will include PSO URGENT in the subject line.

In some cases, the PSO may need to tell the HSDD Eligibility Team Leader or Manager of the urgent enquiry. For example, a ministerial complaint or a breach of privacy.

The helpdesk will respond to urgent email requests within 2 business days, will assess all of the urgent requests and respond to them in order of priority.

The response time will be longer if the case needs external stakeholder advice.

PaNDA requests:

  • Applications escalated via PaNDA must be escalated using work type Medicare Eligibility and enrolment, Nat.Office Escalations.
  • Requests to remove children under 15 from a person Medicare card escalate using work type Nat.Office Escalation DCP

Escalate a Department of Child Protection (DCP) or a prisoner enquiry

Table 6

Step

Description

1

How to handle a F2F DCP or prisoner enquiry + Read more ...

F2F Service Officer must:

  • escalate the query to the HSD Eligibility team, and
  • scan all of the forms and documents

The DCP or a non-government organisation may make a verbal request for a child’s Medicare card details. Do not give out these details. The DCP or non-government organisation needs to call the HSDD Eligibility Team and give their password and agency ID.

Any request from a non-government body must not be actioned. For example, Anglicare or Wesley Mission. Call the HSDD Eligibility team for help.

2

DCP and prisoner queries for a HSDD Eligibility Service Officer + Read more ...

If a HSDD Eligibility Service Officer needs help call an Eligibility PSO.

If the PSO asks the HSDD Eligibility Service Officer to send them the enquiry, they must:

  • Forward the details to PSO via the digitised workflow
  • Details of the enquiry may include:
    • The file name Work Item ID
    • A Medicare card number
    • The applicant’s name

3

DCP and prisoner queries for the helpdesk + Read more ...

If a PSO needs help:

  • Call Medicare Consumers Program Support helpdesk, or
  • escalate via PaNDA using work type Medicare Eligibility Nat.Office Escalations DCP

The Helpdesk will respond to the PSO. A PSO or a HSDD Eligibility Service Officer will complete the enrolment.

Escalate a third Medicare card request

Table 7

Step

Description

1

A third card request + Read more ...

F2F or HSDD Eligibility Service Officers must escalate all third card requests to Eligibility PSO team via PaNDA. PSOs do not decide and grant third card approvals verbally.

Service Officers must ensure a completed application and all relevant documentation has been provided by the applicant.

Service Officers will escalate to Eligibility PSO via PaNDA under the work type:

  • DCP 3rd card request to Medicare Eligibility DCP Escalations.
  • Other 3rd card requests to Medicare Eligibility Escalations

The PSO will:

  • assess and confirm that the request is valid
  • include approval/rejection in PaNDA work item notes
  • re-assign the third card request back to the processing queue for the next HSDD Eligibility SO to complete

All 4th card (or more) must be escalated to the Medicare Public Eligibility Helpdesk. The PSO is to send request via PaNDA using work type Medicare Eligibility Nat.Office Escalations DCP.

Escalate a Child Dental Benefits Schedule (CDBS) query

Table 8

Step

Description

1

How to handle a F2F CDBS enquiry + Read more ...

A F2F Service Officer must try to resolve the customer enquiry. See Patient eligibility for Child Dental Benefits Schedule (CDBS) in Medicare.

If the F2F Service Officer is unable to help call the HSDD Eligibility team.

If unable to contact a HSDD Eligibility Service Officer:

  • complete the CDBS enquiry form
  • scan it using a Multi-Function Device (MFD)

2

Try to resolve the enquiry + Read more ...

The HSDD Eligibility Service Officer must try to resolve the enquiry.

If the Eligibility PSO needs help, email CDBS Medicare and Veterans Branch National Office
