View and amend immunisation history details

Table 2: when processing pended encounters, Service Officers must check the individual's immunisation history to see which vaccines may be conflicting with the current encounter. Previously recorded immunisation history may need amending to allow the current encounter at a P status to go to an A status.

Step

Action

1

Access Episode History-Basic screen + Read more ...

  • From the pended claim, navigate to YIAE – Encounter Details screen. See Steps 1 – 5 in Table 1.
  • Press [F14] – HIST

The YIRD – Episode History-Basic screen displays.

2

Evaluate the immunisation history + Read more ...

To allow the current encounter at P status go to A status, determine if:

  • the immunisation history details need to be amended, or
  • the immunisation details currently at P status need to be amended

If necessary, contact the immunising provider to clarify the information received or the immunisations previously recorded.

3

Contact the provider + Read more ...

For outbound calls, Service Officers must use the script in Step 5 in Verify the identity of recognised vaccination provider for AIR enquiries table.

Contact provider to confirm immunisations.

Locate provider contact details in the Provider Directory System (PDS). See the Log on to PDS table for help.

PDS search for provider details:

  • Search for the provider number using Program Registration (Medicare provider) or Organisation (Ancillary provider)
  • Select View. The Program Registration screen displays
  • In the *Search ID field, key provider number stem and location (first 7 digits Medicare provider or 6 digits Ancillary provider)
  • In the *ID Type field, select from drop down:
    • Medicare Stem and Location for Medicare providers, or
    • AIR Provider number for Ancillary provider
  • Select the radio button Program Registrations for this Provider/Practice/Organisation
  • Select Search. The Program Registration View screen displays
  • Check the Registration End Date column for AIR and Medicare end dates

Is contact successful?

  • Yes, go to Step 4
  • No, provider is unable to be contacted, go to Step 8

4

Make an amendment + Read more ...

To amend:

  • date of service, go to Step 5
  • vaccine and/or dose details, go to Step 6
  • vaccine recorded in error, go to Step 7

5

Amend a date of service + Read more ...

To amend a date of service:

  • Select (/) the encounter and press [F5] – ENCOUNTER. The YIAE – Encounter Details screen displays
  • Over-key the date of service
  • Press [Enter] and press [F9] to update. The YIAN – Antigens Details screen displays
  • If pend code is removed, press [F9]. The YIAE – Encounter Details screen displays

Go to Step 10.

6

Amend a vaccine and/or dose + Read more ...

To amend vaccine and/or dose details:

  • Select (/) the encounter and press [F6] – ANTIGEN. The YIAN – Antigens details screen displays
  • See Step 4 in the Reject or accept an episode from an individual’s immunisation history table
  • Add the correct vaccine and/or dose
  • Press [Enter]
  • Press [F9] to update

Go to Step 10.

7

Amend a vaccine recorded in error + Read more ...

To amend a vaccine recorded in error:

  • Select (/) the encounter and press [F6] – ANTIGEN. The YIAN – Antigens details screen displays
  • See Step 4 in the Reject or accept an episode from an individual’s immunisation history table

Go to Step 10.

8

Unable to contact provider + Read more ...

If unable to contact the provider:

  • Leave pend in a P status and write the comment unable to contact provider on DD/MM/YY to resolve pend on the individual’s Mainframe record
  • Make a maximum of 3 attempts to contact the provider by phone
  • If contact cannot be made the Service Officer must action the pend:
    • accept the first dose that has come in
    • over-key (R and Z) the second

9

Record a comment + Read more ...

Record a comment on the customer's Mainframe record:

  • Key the Medicare number on the YCUI – Registration Amendment Search screen and press [Enter]. The YCUA – Amendment Registration screen displays
  • Press [F3]. The YCCO – Comments List screen displays
  • Press [F3]. The YCCV – Comment Details screen displays
  • Add a comment. To confirm details, include:
    • provider contacted
    • correct date of immunisation
    • immunisation given
    • name of the person confirming/providing information
  • If unable to contact, write unable to contact provider on DD/MM/YY to resolve pend
  • Press [Enter]
  • Press [F9] to update

10

Complete next pend or select next claim + Read more ...

If there are multiple episodes in the claim, the next pend will display automatically. Use the relevant pend code table to process the warning message displayed.

Once the last pend for the claim is processed:

  • Press [F5]. The YIAE – Encounter Details screen displays
  • Press [F4]. The YICP – Claim Selection screen displays
  • Key search parameters to select the next claim
