Unable to find prescription

Use alternative search criteria including:

Action

1

HPOS (Form Upload) received prescriptions

  • Select the relevant HPOS folder - if unsure, escalate to a Complex Drugs LPS

2

Is the prescription to be processed by DVA/RPBS? + Read more ...

When a Department of Veterans' Affairs (DVA) file number is recorded on the Authority prescription or the Repatriation Pharmaceutical Benefits Scheme (RPBS) tick box has been highlighted, check RPQN in Mainframe or the Patient Search screen in OPA to confirm PBS eligibility.

HPOS (Form Upload) received prescriptions

  • Right click on the prescription file and select Save As
  • Select the folder to save the file into
  • Name the file 'DVA prescription' and select Save
  • Open 'File Explorer' and locate the file
  • Right click on the file and select Send to > Mail recipient. This will generate a new email with the file attached
  • Email Veterans' Affairs Pharmaceutical Advisory Centre (VAPAC) Queensland. See the Resources page for the correct email address
    • email Classification must be selected as SENSITIVE:PERSONAL before selecting 'Send'
  • Delete file from File Explorer
  • In HPOS Form Upload, respond to the HPOS item by using the 9 digit HPOS reference number to locate the item and send a reply. See the Resources page for the required wording to be included in the email response
  • In PaNDA, add a Processing note on the file indicating action taken and change work status to 'Completed'

Procedure ends here.

Prescriptions received by mail

  • Authority prescriptions for DVA patients are placed in a non-window facing envelope and sent to DVA for authority processing. See the Resources page for the correct DVA mailing address
  • Add a Processing note on the file indicating the action taken
  • Change work status to 'Completed'

Procedure ends here.

3

Check legislative requirements + Read more ...

If the prescription has a telephone approval number, see Process telephone Authority approval application.

If the prescription has a Streamlined code, see Process Authority required (STREAMLINED) item.

Check Authority prescriptions for all legislative requirements such as patient/prescriber name, patient/prescriber address, date of prescribing and prescriber signature.

Note: reject all post-dated Authority prescriptions. There is no legislative basis for approved prescribers to postdate Authority prescriptions.

4

Access Online PBS Authorities (OPA) - Record Authority + Read more ...

In the Medicare - eBusiness Gateway:

  • Log onto PBS Authority Application System from the eBusiness System dropdown
  • submit and select Request Authority

Make sure all prescriptions are keyed even if they are to be rejected.

5

Approved prescriber search + Read more ...

  • Key the approved prescriber number from the Authority prescription
  • Amend the date of prescribing to reflect the date on the prescription as this will default to the date of processing
  • Press Enter or select Search
  • Confirm approved prescriber details and select Next

6

Patient search + Read more ...

  • Search for the patient by either:
    • Keying the patient Medicare number in the Medicare number field, or
    • Selecting Patient as the search type and key the patient's surname and postcode
      Note: postcode is a mandatory field
  • Press Enter or select Search
  • Select the correct patient from the list and select Next

7

Authority prescription number + Read more ...

  • Key the Authority prescription number as it is written on the Authority prescription, leading zeroes are not required
  • Press Enter or select Next

Note:

  • If the number is incorrect:
    • a pop up will alert the Service Officer
    • rekey the Authority prescription number
  • If the number is still incorrect escalate to the:
    • Local Peer Support (LPS)
    • Nominated experienced Service Officer
    • Team Coach, or Pharmaceutical Adviser (PA)

If possible, all authority prescriptions are to be recorded in OPA.

Where an Authority prescription number does not exist, key the prescriber number in this field then reject the script and add an authority comment 'no authority ID number’.'

8

Item selection + Read more ...

  • Search for the item by either:
    • keying the name of the item in the Keyword field, or
    • using the item code
      Note: if the name of the item is being used, filter the search results by adding the strength in the Drug, Form and Strength field 10 for 10mg
  • Press Enter or select Search
  • Select the correct item from the list and select Next
  • The Prescribed Quantity and Prescribed Repeats will default to the PBS listed quantity and repeats for the item. These details can be changed if required
  • The Daily Dose must be keyed. The dose field accepts whole numbers as well as decimal doses, for example, 1.5
  • If a daily dose cannot be measured key X for items not measured by dosage, D if directions were given to the patient or N if no directions were given
  • After keying the correct quantity, repeats and dose, select Next

9

Select restriction + Read more ...

A screen will appear to select the restriction.

  • select the appropriate restriction as specified on the Authority prescription
    • Service Officers may need to select Display Restriction Detail if they cannot read the full restriction
  • select Next and confirm the indication
  • select Next
  • answer any associated question details if applicable to the item restriction, then select Next

10

Authority prescription details + Read more ...

  • The Authority prescription details will display. If required details need to be changed, this can be done by selecting 'Edit'
  • If changing the quantity or repeats requested on the written authority application, record this in the Authority Comments field. Example of what to key 'Approved with changes 60 & 5 to 60 & 4'. (See Step 12)

Read patient comments if available.

Authority Comments field

  • If approved with changes, record the dose, quantity and repeats originally requested
  • Rejection reason(s)
  • Dispatch type - returned to Prescriber or patient
  • Any other comments if required
  • P number or logon

Internal Application Type dropdown

For Written Authorities, select Written from the Internal Application Type dropdown.

Patient Comments field

If a patient comment is applicable, add the comment in the Patient Comments field.

11

Assessment result + Read more ...

Select Submit. The system will assess the request as approved or rejected.

If a Service Officer determines the written authority request meets all relevant criteria and should be approved but OPA has rejected the request the Service Officer may:

  • select Approved from the Assessment Result Override dropdown box
  • select reason for override or use the keyboard shortcut Ctrl+Shift+O and submit, or
  • escalate to LPS or PA for clarification

Rejections for ‘prescribed quantity and/or repeats is greater than allowed’ occurs due to the limits set to prevent prescribers obtaining more than a certain amount when using OPA. Operators can override these rejections in accordance with approval rules.

If the written Authority is:

  • rejected as 'too soon' Reason Code 94 and 95, or
  • has an X or D in the dose field
  • go to Step 12

Note: if recording another Authority for the same patient the date of prescribing will default to today’s date. Use the New Authority Same Prescriber function and change date of prescribing.

12

Rejected as 'too soon' + Read more ...

Written Authority applications with an X or D in the dose field will need to be assessed manually for 'too soon'.

The system will return Information Reason Code 132 to advise that 'MANUAL HISTORY CHECK REQUIRED - a patient history check could not be completed’.

If the Written Authority application is rejecting because it is 'too soon' reason code 94 and 95, or needs to be assessed manually, Service Officers must go into Search Authorities and:

  • select Display Patient History
  • use the returned history to calculate the 'date the supply will last to' in order to complete the rejection letter
    • To reject an application, use the reject button to record the rejection or use the keyboard shortcut Ctrl+Shift+R
    • Select a reject reason from the dropdown box

The system uses 12 months of history to calculate 'too soon' and considers the dispensing history and if there have been any overrides applied. Therefore, when manually assessing, be aware of the differences and use your usual method of assessment. Contact a LPS if further assistance is required.

For manual assessing of too soon:

  • Review authority approval history
  • Use a 6 month date range from date of processing , for example, if processing the application on 16/5/22, the date range would be 16/11/21 - 16/5/22. See Step 9 in the Requesting a telephone Authority prescription table
  • Use the Too soon and supply length calculator (new) to calculate the date the supply will last to

13

Saving an Application + Read more ...

If a Service Officer needs to stop processing, an Authority application can be saved:

  • To save an application select Save on any screen after the item has been selected or use the keyboard shortcut [Ctrl]+[Shift]+[S]
  • To continue assessing an application that has been saved, find the saved authority application using Search Authorities from the homepage in OPA
  • Select Resume against the authority which is saved. This opens the authority at the point that it was saved
  • Amend any previously entered information by selecting the Back button
  • Complete authority as usual

How to endorse and post a written authority approval request

Table 6: this table describes how Service Officers endorse and post a written authority approval request.

Step

Action

1

Written authority request results + Read more ...

For written authority requests with the assessment result of:

  • Approved, go to Step 2
  • Rejected, go to Step 3
  • Approved with changes, go to Step 4

2

Approved requests + Read more ...

For written authority requests with an assessment result of Approved, endorse the request:

  • circle or write the quantity and repeats being approved in red pen
  • annotate the PBS item code(s), and
  • annotate with OPA approval number, including applicable prefixes, for example TPX, SPX, PLC and HSD

Posting the written authority request

  • Remove the third copy and place in a secure orange bin for shredding
  • Check who is to receive the application - the prescriber or the patient
  • Generate an envelope for the correct recipient
  • Confirm name and address generated on the envelope matches the recipient details on the prescription
  • Confirm prescriber details on any attached letter to be sent with the prescription matches the prescription and the envelope
  • Place the original and the duplicate of the prescription, together with any attached letter, in the envelope
  • Make sure the Authority prescription is folded so no information can be read through the envelope
  • Place the envelope in the correct mailroom tray

Note:

  • when the Authority prescription must be returned to the patient, return to the address on the Authority prescription
  • when addressing hospital prescriptions, including the unit title on the address will improve delivery outcomes. For further information on addressing correspondence, see Quality assurance and quality checking for external mail in Medicare

Go to Step 5.

3

Rejected requests + Read more ...

For written authority requests with assessment result of Rejected, do not endorse the prescription.

Posting the written authority request

  • Do not remove the third copy
  • Print Rejection letter (Unapproved authority prescriptions)
  • Staple letter to prescription
  • Use window envelopes where available, folded with the prescriber address visible through the window and return to approved prescriber
    • otherwise, an envelope must be generated
    • confirm prescriber’s name and address on the envelope matches the prescription

Note: always return Rejection letters to the approved prescriber indicated on the written authority request.

Go to Step 5.

4

Approved with changes + Read more ...

For written authority requests that have had their quantity and/or repeats changed from what was originally written on the prescriptions, an assessment result of 'Approved with changes', endorse the request by doing the following:

  • circle or write the quantity and repeats being approved in red pen
  • annotate the PBS item code(s), and
  • annotate with OPA approval number, including applicable prefixes, for example TPX, SPX, PLC and HSD

Posting the written authority request

  • Remove third copy and place in a secure orange bin for shredding
  • Print Approved authority prescription with changes letter
  • Staple any letters to authority prescription
  • confirm prescriber’s name and address on the envelope matches the prescription

Note: always return Approved with changes to the approved prescriber indicated on the written authority request.

Go to Step 5.

5

Complete work item + Read more ...

On completion of the work item:

  • Select Complete in the Status field
  • A Complete reason is only required when:
    • the work item has been completed in the source system (appears to already be completed, that is a duplicate work item)
  • select Save

A message will display at the bottom of the screen advising that the work item has been successfully saved.

Note:

  • it may take up to 2 minutes for the work item to disappear from the Worklist
  • refer to Team Leader or LPS if required

Procedure ends here.

Item

Description

1

Determine the problem + Read more ...

Is the file:

  • unclear or cannot be read, go to Step 2
  • a form other than a prescription request, go to Step 3
  • a prescription that has previously been actioned and the prescriber has sent multiple emails through HPOS, go to Step 4

Unclear or cannot be read + Read more ...

Search for the email:

  • Select the ‘Open in HPOS’ link located in top right hand corner
  • open the attached file from the prescriber and check the response is for the correct patient
  • if incorrect patient, escalate to a Pharmaceutical Adviser (PA)

A form other than a prescription request + Read more ...

The form is not a prescription request, for example stationery or a pathology form or the form has been categorised incorrectly:

  • escalate to the team leader for instruction on how to action

4

Prescription has already been actioned + Read more ...

The prescription has already been actioned and the prescriber has sent in multiple emails:

  • Select the ‘Open in HPOS’ link located in top right hand corner
  • open the attached file from the prescriber and check the response is for the correct patient
  • if incorrect patient, escalate to a PA
  • press Reply
  • copy and paste the information below into the Text field:
    • Please note, this prescription has already been assessed and your response can be found in HPOS reference number XXXXXXXX

5

File has multiple patient prescriptions + Read more ...

If there are multiple patient prescriptions in one file:

  • Save the file:
    • Right click the file and select Save As
    • Save the file to the computer
  • Extract the additional prescriptions from the PDF file and save the new files to the computer:
    • In Acrobat DC, select Tools > Organize Pages > Extract
    • Highlight the pages to be extracted
    • If the pages are to be extracted as separate documents, tick the Extract Pages As Separate Files box before selecting Extract
    • Save the item(s). These item(s) can be found in the saved folder which has the same name and numerics to differentiate, for example 2 or as pages from <original title>
    • Locate the saved file (the file will have the same title and a numeric, for example, 2)
  • Create separate files in PaNDA for each person. To create new work items in PaNDA:
    • Select PaNDA Document Upload under the Health Services menu
    • Select Create New Work Item from Upload Type field
    • Select Browse and select a file from the pop up window
    • Change the Date of Lodgement (defaults to today’s date)
    • Select Program from the dropdown menu
    • Select Work Type from the dropdown menu
    • Add Primary Reference Number (Medicare number)
    • Select Upload to submit the new work item into the PaNDA allocation pool
  • Delete files from the computer
  • On the original file, put a processing note to indicate the other documents in the file has been extracted into new files.
