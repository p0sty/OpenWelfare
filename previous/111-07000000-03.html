This page contains information on Data Integrity Enquiry (DIE) System and screens.

Data Integrity Enquiry (DIE) screen codes

Description of DIE screen codes

Table 2: This table describes Data Integrity Enquiry (DIE) screens. These screens can be accessed from the DI (Data Integrity) Main Menu (DIMM) screen, with the exception of the DI Notes Update (DINT) screen.

Screen

Screen Descriptions

DIESD

DI Error Search Data Search screen

The DIESD screen displays all customer DI error record occurrences for a particular DI Error Code. It allows the user to make record selections to view record details, submit 'Fix' or 'Check' triggers, add keywords, update DI notes and print record lists. Additionally, counts of unique data corruptions can be displayed using the Tabulate option.

DIESL

DI Error Search List Screen

The DIESL screen displays all customer DI error records for a particular DI Error Code. It allows for cluster owned, full and wildcard DI Error Code searches. This screen allows the user to make record selections to view record details, submit 'Fix' or 'Check' triggers, add keywords, update DI notes and print record lists.

DIESS

DI Error Search – Statistics(DIESS) screen

The DIESS screen lists the count of totals, new and fixed customer DI error records for specific DI Error Codes. It allows for cluster owned, full and wildcard DI Error Code searches. This screen also allows the user to select a record to view a complete list of customers with the DI Error Code and print DI error record statistics.

DIFIXSL

The DI Adhoc Fix Search - List (DIFIXSL) screen

The DIFIXSL screen displays customer records that have been updated by 'adhoc' processing. This includes X! field (PF9), Batch2 processing via Release Management and Major Release 'Adhoc Fix' processing. It allows the user to make record selections to view 'Adhoc Fix' details and print the record list.

DIFSL

DI Fix Trigger Search - List (DIFSL) screen

The DIFSL screen is a search screen that displays all DI errors associated with 'Fix' triggers. It allows for cluster owned, full and wildcard DI Error Code searches. 'Fix' triggers can also be filtered by full or partial name searches. This screen allows the user to view the DI Error Code and 'Fix' trigger details and print the record list.

DICED

DI Customer Error Details (DICED) screen

The DICED screen displays detailed customer DI error record information for a specific customer and DI Error Code combination, as selected from the DIESL, DICES or DIESD screens.

DICES

DI Customer Error Summary (DICES) screen

The DICES screen displays all customer DI error records for a particular customer. It allows the user to make record selections to view record details, submit 'Fix' or 'Check' triggers, update DI notes and print record lists.

DIEDD

DI Error Description Details (DIEDD) screen

The DIEDD screen displays the description of DI errors, including all information held in the DI Error member in the Centrelink Repository and any 'Fix' trigger information.

DIFIXS

DI Adhoc Fix Summary (DIFIXS) screen

The DIFIXS screen allows users to view updates made to a customer's record by 'Adhoc Fix' processing. This includes X! field (PF9), Batch2 processing via Release Management and Major Release 'Adhoc Fix' processing.

DIFIXD

DI Adhoc Fix Details (DIFIXD) screen

The DIFIXD screen displays detailed 'Adhoc Fix' information for a specific customer and update type combination. This includes X! field (PF9), Batch2 processing via Release Management and Major Release 'Adhoc Fix' processing.

DIMM

DI Main Menu (DIMM) screen

The DIMM screen lists all the screens that comprise the DIE system.

DIMRPT

DI Multiples Report (DIMRPT) screen

The DIMRPT screen allows the user to create a report showing all the multiple CRNs created for a given period and office. The report will be generated in an XML spreadsheet format, which will be sent by email to the nominated Area positional mailbox. The XML report can be viewed in any database or spreadsheet program. It will usually open in Microsoft Excel (depending on your PC settings).

DINT

DI Notes Update (DINT) screen

The DINT screen is used to update notes and keywords for a specific customer DI error record. Blocks of entered text are marked as protected to maintain a history of notes. It can only be accessed through the DIESL, DICES, DICED and DIESD screens.

DIPRT

DI Print (DIPRT) screen

The DIPRT screen is used to direct a print request from the Data Integrity Enquiry (DIE) system to a printer. Records selected for printing from other DIE screens are stored in a buffer and displayed on the DIPRT screen, where they can be deleted from the list before printing.

DIRIT

DI Reference Identifier and Type (DIRIT) screen

The DIRIT screen allows the user to place a Reference Identifier and Type (RIT) in context to customer DI error records.

DIT

DI Triggers (DIT) screen

The DIT screen is used to enable users to create 'Check' triggers against a set of customers. Sets of customers are formed from Centrelink system triggers or activities that are selected with various filters.

XDIFC

DI Funload Check (XDIFC) screen

The XDIFC screen allows Funload checks to be run online.
