Preliminary checks

Preliminary record check before reactivating a suspended BasicsCard

Table 1: This table describes the preliminary checks to be undertaken before reactivating a suspended BasicsCard.

Step

Action

1

Customer or payment nominee contacts requesting BasicsCard reactivation + Read more ...

The customer, or payment nominee (if the BasicsCard was issued to a payment nominee), may contact when they find their previously lost or stolen BasicsCard and they request to have this card reactivated.

The BasicsCard may have been suspended previously due to it being reported as lost, stolen or damaged.

Confirm identification

  • If the customer has contacted, confirm the identity of the customer before proceeding with the interview. Go to Step 3
  • If the payment nominee has contacted, go to Step 2

2

Check payment nominee arrangement on the BasicsCard Summary page + Read more ...

Ensure that the BasicsCard was issued to the payment nominee.

  • Check that the payment nominee arrangement exists
  • Check the payment nominee's identity
  • If an income managed customer has a payment nominee, the BasicsCard will be issued to the payment nominee on behalf of the customer

3

BasicsCard Summary page: BasicsCard status + Read more ...

Check the BasicsCard status on the BasicsCard Summary page. Access the BasicsCard Summary page via Customer First>Workspace>Income Management and under the BasicsCard heading select BasicsCard Summary.

If the status is Blocked:

The BasicsCard PIN will be 'blocked' by the Card Transaction Provider (CTP) if customer enters their Personal Identification Number (PIN) incorrectly six or more times at a BasicsCard Merchant.

Customers/payment nominees that have a BasicsCard that is 'blocked', will remain 'blocked' until the customer contacts. These BasicsCards can be 'unblocked' by the customer changing or resetting their PIN.

Is the status showing as suspended?

  • Yes, go to Step 4
  • No, procedure ends here

4

Has another BasicsCard been issued as a result of the previous suspension? + Read more ...

  • Yes, go to Step 5
  • No, the BasicsCard can be reactivated. See Step 1 in the Reactivating a suspended BasicsCard table

5

Replacement card has already been issued + Read more ...

Advise the customer/payment nominee of the following:

  • The re-activation is not possible as a replacement card has already been issued and activated
  • The found BasicsCard is to be destroyed by them as it has been cancelled and replaced by a new BasicsCard

Record details on a DOC. Procedure ends here.

Reactivating a suspended BasicsCard

Table 2: This table describes the assisted processing instructions for workflows for reactivating a suspended BasicsCard.

Step

Action

1

Access the Change BasicsCard status workflow + Read more ...

Updates are made using the Change BasicsCard status workflow. BasicsCard Status can be viewed on the BasicsCard Summary screen.

Is Customer First Workspace available?

  • Yes, go to Step 2
  • No, see Step 1 in the Manual coding in the national index, BasicsCard Alternative Interface and BasicsCard Contingency Support System table

2

Reactivating suspended BasicsCard + Read more ...

When reactivating a suspended BasicsCard, ensure the card in the customer/payment nominee's possession matches the BasicsCard listed on the BasicsCard Summary screen.

Does the requested card match the system?

  • Yes, go to Step 3
  • No, advise the reactivation is not possible at this time and the customer/payment nominee will need to visit a Centrelink Service Centre, Remote Service Centre or Remote Servicing Team to receive a replacement BasicsCard. See BasicsCard replacement of card

3

Review BasicsCard suspension Auto DOC + Read more ...

Prior to running the Change BasicsCard Status workflow, review the Card Suspension Auto DOC for additional information as to why the BasicsCard was initially suspended.

Can the BasicsCard be reactivated?

  • Yes, go to Step 4
  • No, advise the re-activation is not possible at this time and the customer/payment nominee will need to visit a Centrelink Service Centre, Remote Service Centre or Remote Servicing Team to receive a replacement BasicsCard. See BasicsCard replacement of card

4

Run the workflow: Change BasicsCard Status + Read more ...

Run the Change BasicsCard status workflow to go to the BasicsCard Status page.

Does the customer have a nominee?

  • Yes, go to Step 5
  • No, go to Step 6

5

Continue the workflow: Authentication of nominee + Read more ...

An Authentication page displays.

  • If the customer has Display on Access (DOA) DOCs, confirm that all the DOA DOCs have been read
  • Confirm if the customer's record is password protected

Once this is completed, confirm the nominee's name and password. The workflow will not continue until this is done.

If a nominee cannot recall their password, refer to Appointing Nominees.

This page will display the customer/payment nominees:

  • card number
  • Expiry date
  • Card Status
  • reason for suspension
  • Option to 'opt in' to BasicsCard Balance Printed on EFTPOS receipt.

If the customer/payment nominee consents to the offer of BOR and 'yes' is selected, then a 'consent statement' will appear. The customer/payment nominee will need to read and accept the consent statement in order to 'opt in' to Balance on Receipt. Once the consent statement is accepted the system will allow the Service Officer to continue on.

Note: there are a group of national retailers that are BasicsCard Balance on Receipt merchants. The merchants have the functionality to print BasicsCard balances on EFTPOS receipts for customers.

Go to Step 7.

6

Continue the workflow: Authentication of customer + Read more ...

The Authentication page displays.

  • If the customer has Display on Access (DOA) DOCs, confirm that all the DOA DOCs have been read. Note: DOA DOCs will only appear if they are present on the customer's record
  • Confirm if the customer's record is password protected
  • Confirm how the customer was authenticated by selecting the method/methods by which the customer has been authenticated using the list provided

The workflow will not continue until the customer has been fully authenticated

This page will display the customer/payment nominees:

  • card number
  • Expiry date
  • Card Status
  • reason for suspension
  • Option to 'opt in' to BasicsCard Balance printed on EFTPOS receipt

If the customer/payment nominee consents to the offer of BOR and 'yes' is selected, then a 'consent statement' will appear. The customer/payment nominee will need to read and accept the consent statement in order to 'opt in' to Balance on Receipt. Once the consent statement is accepted the system will allow the Service Officer to continue on.

Note: there are a group of national retailers that are BasicsCard Balance on Receipt merchants. These merchants have the functionality to print BasicsCard balances on EFTPOS receipts for customers.

7

Change BasicsCard status: Action to be taken + Read more ...

  • Select the 'Reactivate' option from the Please select a BasicsCard status action dropdown list
  • Click Finalise
  • User message displays: 'BasicsCard reactivated. Current balance $......'

8

Next page: Auto DOC + Read more ...

Record on the DOC that the card has been reactivated, including the time and date. Delete or add information as required.

9

BasicsCard is reactivated + Read more ...

BasicsCard status shows as Active on the BasicsCard Summary page.

If the customer/payment nominee requires their Personal Identification Number (PIN) to be changed following re-activation, see BasicsCard request to change Personal Identification Number (PIN).

Review current priority needs and allocations to the BasicsCard. Procedure ends here.

7

Access the BasicsCard Alternative Interface web page + Read more ...

The BasicsCard Alternative Interface is an external website maintained by the Card Transaction Provider.

The Alternative Interface provides BasicsCard:

  • balance
  • status
  • transaction history
  • status updates, and
  • can be accessed to suspend, activate (reactivate previously suspended cards) and unblock BasicsCards when the BasicsCard number is known.

The BasicsCard number must be known to use this interface

Generally this would only be used when Customer First and the National Index are not available as the BasicsCard Alternative Interface does not contain any customer information. It cannot be used to locate a customer's record or to verify a customer's identity. Access and use of this facility is intended for Service Officers in the 24/7 Income Management Line or Level 1 and 2 Helpdesks.

Note: this must be used in conjunction with Income Management Business Continuity Procedures to authenticate customers.

Account or Card Search page + Read more ...

From the Account or Card Search page:

Select the 'Search by Card No' and enter the 16 Digit BasicsCard Number. Press [Search].

The Transaction History screen will display card balance, transactions and status.

To update card status select the 'Card Status' button. The Card Status screen will display.

9

Card Status screen + Read more ...

To reactivate previously suspended BasicsCard:

  • enter the Office Code
  • in the New Card Status box select the action required, 'ACTIVATE'
  • enter the Reason Code from the drop down box
  • select Save which will update the card status

Navigate Back to the Transaction History screen which will show card status as ACTIVE.

Service Officer will not be able to DOC the customer's record in the BC Alternative Interface. Manually record the customer's details to be updated on the customer's record when Customer First is available. Procedure ends here.

10

BasicsCard Contingency Support System (BCSS) + Read more ...

The BasicsCard Contingency Support system allows access to limited functions for BasicsCard when the Centrelink system, National Index and BasicsCard Alternative Interfaces are not operational.

The BasicsCard Contingency Support System will allow the following functions:

  • display current BasicsCard status
  • obtain BasicsCard balance
  • obtain Income Management account balance
  • suspend, reactivate or unblock a BasicsCard
  • change/reset a BasicsCard Personal Identification Number (PIN)

The BasicsCard Contingency Support System will automatically update details of any transactions completed when systems are restored.

To access the contingency system, use the link in Resources to the login page for the BasicsCard Contingency Support System. Note: add this address to 'Favourites' for future use.

See Resources for links to the training presentation available on the BasicsCard and TPO Services team homepage.

Note: BCSS will not open with the Firefox internet browser. Ensure that the BCSS is opened using Internet Explorer (IE).
