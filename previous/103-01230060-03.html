BasicsCard found by a third party

Table 1: This table details the assisted processing instructions for Customer First when a third party reports they have found a BasicsCard.

Step

Action

1

Third party contacts the Services Australia to report they have found a BasicsCard + Read more ...

Note: no details about the customer are to be provided to the third party.

If face-to-face, the Service Officer is to request the BasicsCard from the third party.

If contact is via Smart Centre Call, request BasicsCard number.

  • To search for the customer's record by the BasicsCard number in Customer First, go to Step 2
  • If Customer First is not available, see Step 1 in the Manual coding, the national index and the BasicsCard Alternative Interface table

2

Conduct search on the Reference Number Search Page + Read more ...

  • Access Customer First
  • Search by the BasicsCard number, go to the Reference Number Search Facility by typing 'RNS' in the Next: field

For BasicsCards:

  • partner agency: field, enter 'IND'
  • Either swipe the BasicsCard or enter the BasicsCard number manually into the Customer ID: field
  • ID Type: field, enter 'BCN'
  • Search National Index: field, enter 'Y'
  • Next Screen: field, enter 'PED' and press [Enter]

3

Searching National Index Records + Read more ...

Has the message '1018ZZ - No National Index records found meeting the search criteria' been presented?

  • Yes, the BasicsCard has already been cancelled and another reissued to the customer. Destroy the card. If contact is via Smart Centre Call, request third party to destroy the card. Record details on a DOC. Procedure ends here
  • No, select the displayed customer's record. This will open the customer's record. Go to Step 4

4

Code the Suspend/Reactivate card page + Read more ...

Access the Change BasicsCard Status page found under the Income Management/BasicsCard menu:

  • BasicsCard status action: field, select Suspend option from the drop down menu
  • Select the relevant Reason for Suspension option
  • Select Finalise. User message will be displayed, 'BasicsCard suspended. Current balance $......'. This signifies that the activity has been completed. Service Officer can navigate off this page
  • Destroy the card. If contact is via Smart Centre Call, request the third party destroy the card

Go to Step 5.

5

Record details on a DOC + Read more ...

An auto DOC will be generated in the workflow for the Service Officer to add or delete information as required.

Include the following details on the DOC:

  • Details of third party who found card
  • Time, date and place card found
  • The card was suspended
  • Reason card was suspended

6

Create Display on Access (DOA) DOC + Read more ...

Create a Display on Access DOC advising the BasicsCard has been found by a third party, the BasicsCard has been suspended and the card has been destroyed.

This will prompt the Service Officer during the next customer contact to review regular expenses and explain options for the customer to be issued a replacement BasicsCard.

Remote customers may not be able to attend a Service Centre and if the Remote Servicing Team is not available to the customer they may require alternative payment methods to meet priority needs, for example, direct deduction facilities.

Customer First not available + Read more ...

  • If a third party is reporting a card as found and Customer First is not available, access to the BasicsCard Alternative Interface is required to suspend the card
  • Generally the BasicsCard Alternative Interface function is only available through the 24/7 Centrelink Emergency Smart Centre Call or Level 1 and 2 helpdesks
  • Take possession of the BasicsCard and the relevant details, and contact the 24/7 Centrelink Emergency Smart Centre Call which is only available after business hours, or during business hours contact the helpdesk to have the card suspended
  • For Service Officers using the BasicsCard Alternative Interface, go to Step 5
  • For all other Service Officers, procedure ends here

As a final contingency, if Customer First and the BasicsCard Alternative Interface are not available, and the Third Party is able to advise the customer's full name or CRN, access the National Index. Go to Step 6.

5

BasicsCard Alternative Interface + Read more ...

The BasicsCard Alternative Interface is an external website maintained by the Card Transaction Provider.

The Alternative Interface provides BasicsCard:

  • balance
  • status
  • transaction history
  • status updates, and
  • can be accessed to suspend, activate (reactivate previously suspended cards) and unblock BasicsCards when the BasicsCard number is known

The BasicsCard number must be known to use this interface.

Generally, this would only be used when Customer First and the National Index are not available, as the BasicsCard Alternative Interface does not contain any customer information. It cannot be used to locate a customer's record or to verify a customer's identity. Access and use of this facility is intended for Service Officers in the 24/7 Income Management Line or Level 1 and 2 Helpdesks.

Note: this must be used in conjunction with Income Management Business Continuity Procedures to authenticate customers.

6

Customer First and the BasicsCard Alternative Interface are not available - National Index + Read more ...

This step is for Service Officers with access to Citrix.

  • Search within the National Index using the customer's full name or CRN. (Search by BasicsCard number is not available in the National Index)
  • When Customer First and the BasicsCard Alternative Interface are not available, limited activities can be done in the National Index System. This includes suspending and reactivating BasicsCards and obtaining BasicsCard balances
  • To log into the National Index system:
    • Open Empower, from the main menu (prior to logging in) press [F4]
    • Enter 'User ID' and Centrelink Staff Identification Card (CSIC) 'PIN' or 'Accesslink password' and press [F12] to select the National Index

7

Search for a customer using the National Index + Read more ...

  • Go to the IN screen with the system code set to NNS.
  • Search for the customer (same as if in Centrelink system)
  • Press [Enter]
  • The Client List (CL) screen will appear. Select the required customer
  • The National Index Person Details (PED) screen will display. This screen should provide enough information to authenticate the customer

8

Access BasicsCard - Suspend (BCSC) screen + Read more ...

  • The Card number: field will be pre-populated from customer's record
  • Code the following fields:
    • Suspend reason: use the appropriate code: LST (Lost), STL (Stolen) or OTH (Other)
    • Press [Enter]
    • The Card Balance will be displayed only after a successful response from the Card Transaction Provider (CTP) that the card has been suspended

9

Record details on a DOC + Read more ...

Include the following details on the DOC:

  • Details of third party who found card
  • Time, date and place card found
  • The card was suspended
  • Reason it was suspended

Note: if Customer First is unavailable, the Service Officer will not be able to DOC the customer's record in the National Index. Manually record the customer's details to be updated on the customer's record, when Customer First is available. Ensure to include the reasons the BasicsCard was suspended, who contacted, time/date of suspension and any other valid information.
