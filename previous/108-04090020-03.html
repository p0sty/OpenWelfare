4

Providing information + Read more ...

A large number of managed investments will have an Asia Pacific Investments Register (APIR) code. The system can search for the managed investment using the APIR code.

Investments that do not have an APIR code include:

  • Personal Investment fund portfolios, (also known as a master trust)
  • Funds that have not purchased an APIR code
  • Unlisted managed investments

A pseudo APIR code is issued to all managed investment funds on the Managed Investment (MIN) system that do not have an APIR code.

Can the customer provide an APIR code for the product?

  • Yes, go to Step 5
  • No, go to Step 6

5

Coding - APIR code available + Read more ...

  • Use the Income and Assets Update Task Selector (START)
  • Select Next to go to the Managed Investments (MIS, MIP, MIUS)
  • Select Add on the Managed Investment (MIP) line to access the Create Managed Investment (MIP) options. The system will use details entered to search the Managed Investment (MIN) system for the investment product.

Enter the investment's APIR code in the APIR Code field and press [Enter]

The APIR code:

  • must be exact to obtain a match
  • consists of 9 characters:
  • the first 3 are letters
  • the next 4 are numbers
  • the final 2 are 'AU'

Is there a match for the APIR code?

  • Yes, select the APIR code from the search results, see Step 1 in Table 2
  • No, go to Step 7

Coding - APIR code not available or no match found + Read more ...

Complete the following fields. To access screen help, key '?' in each field.

  • Fund Manager: is mandatory and must have at least 2 characters. The first must be a letter
  • Product Group: is optional, to narrow the investment search
  • Search by Product Option: is optional, to narrow the investment search

Select the Search icon.

A product list displays below the Search for APIR code fields.

Has the product been found?

  • Yes, select the relevant APIR code from the search results, see Step 1 in Table 2

Finalising a new managed investment

Table 2

Step

Action

1

Coding Create Managed Investments (MIP) + Read more ...

Once the search results locate the correct management investment product, the Fund Manager details will populate on MIP.

If bulk updates for 20 March or 20 September have been actioned

When updating a managed investment and the bulk update for 20 March or 20 September has been actioned, the Event date will display as a date in the future. For example, today's date is 15 March and the event date shows 20 March.

To complete the update, follow the Workaround for MIS/SIS updates immediately prior to 20 March or 20 September.

Key the following fields:

  • Fund Manager Details
    • Investment date: the date the investment was made (as shown on certificate/policy)
    • Policy number: key the policy/account/certificate number for the investment.
      (Note: if this number exceeds 10 characters, key the last 10 only)
  • Investment Details
    • Event date: the earliest of, the date the investment was made or the date the payment for the investment was transferred
    • Funeral exempt: (this field is only visible if the investment is a funeral bond) select Yes if the investment is to be disregarded from the income and assets tests
    • Customer owned % and Partner owned %: if the investment is owned jointly with the customer's partner
    • Asset deduction amount $: if applicable, key the loan balance amount which is deductible from the asset value in dollars and cents
    • Rural exemption code: (for customers claiming a rural payment) select:
      ASS if asset value is exempt
      BTH if both asset and deemed income is exempt
      NOT if neither asset value or deemed income is exempt
    • Number of units: if the investment is unit based. Must be a whole number. If the customer owns a partial number of units, round down to the nearest whole number
    • Account amount $:
      account based managed investments, record in dollars and cents
      margin loans, key the gross current market value
      personal investment funds, key the total value of the portfolio, including any cash component
  • Enter Receipt date: and select Channel
  • Select Save

The Investments Summary (MIS) shows the updated managed investment.

Go to Step 3

2

Coding the Unregistered Managed Investment (MIUS) for a customer + Read more ...

If bulk updates for 20 March or 20 September have been actioned

When updating a managed investment, the event date will be a date in the future.

For example, today's date is 15 March and the event date shows 20 March.

If completing the update, follow the Workaround for MIS/SIS updates immediately prior to 20 March or 20 September.

Select Add on the MIUS line to access the Create Unregistered Managed Investment (MIUS) options.

Key the following fields:

  • Header Detail
    • Product name:
    • Investment name:
    • Investment date (dd/mm/yyyy): the date the investment was made (as shown on certificate or policy)
    • Investment type: select the appropriate option-
    • Deeming Ex: select Yes if the investment is exempt from the deeming rules. Note: if deeming exempt, record an amount in the Interest Rate: field
  • Item Detail
    • Event date: the earliest of, the date the investment was made or the date the payment for the investment was transferred
    • Funeral exemption: select Yes if the investment is an exempt funeral investment
    • Customer owned %:
    • Partner owned %: (superannuation cannot be jointly owned)
    • Rural exemption code: (for customers claiming a rural payment) select:
      ASS if asset value is exempt
      BTH if both asset and deemed income is exempt
      NOT if neither asset value or deemed income is exempt
    • Number of units: A unit based investment must be a whole number. If the customer owns a partial number of units, round down to the nearest whole number
    • Unit value amount: if the investment is unit based, the value of each unit
    • Asset ded amt $: if applicable, key the loan balance amount which is deductible from the asset value in dollars and cents
    • Total asset value:
      account based use dollars and cents
      margin loans, key the gross current market value
      personal investment funds, key the total value of the portfolio, including any cash component
    • Interest rate: code if Deeming Ex: = Yes
  • select Save
  • update Receipt date and Channel, select Save

Go to Step 3.

3

What to tell the customer + Read more ...

  • there is a bulk update to revalue all listed shares and unitised managed investments each March and September
  • a customer may request a revaluation of their financial investments at any time
  • although their unitised managed investment prices are automatically revalued, they are required to notify changes to the:
    • number of units they own
    • value of their account-based managed investment(s)
    • balance of their margin loan(s)

Go to Step 4.

4

Finalising the process + Read more ...

For new managed investments obtained after withdrawal from existing investments, see Managed investments - updating existing investments before finalising the activity.

  • Select Assess
  • Check the outcome on Entitlements (ELD) and select Finish
  • Add any extra details to the Finalise Note which has been pre-filled with details of the income and assets update
  • Select Finalise to complete the transaction
  • Go to Step 4 in Table 2 of the Customer First tab to set up a manual follow up (MFU) review for:
    • Margin loans coded on Managed Investments (MIP)
    • Unregistered superannuation products, including Self-Managed Superannuation Funds (SMSF)
    • Personal investment funds, or
    • Investments advised by FINS Helpdesk
  • Otherwise, procedure ends here

Changes to timeshare assessment + Read more ...

Although timeshare investments are treated as managed investments, they do not have an APIR code and are coded on Direct Investments Account (SVDI).

Are there timeshare investments coded on Other Assets (OAS, OASS) or Managed Investments (MIS, MIP, MIUS) or, is income from timeshare coded on Other Income (OIN, OINS)?

Income already coded for a timeshare + Read more ...

  • Cease the asset value on OAS, OASS, MIS, MIP MIUS or the income on OIN, OINS,
  • Code the asset value on Savings (SVP, SVDI).

To cease the asset value:

  • Expand the time share investment
  • Add a new investment details entry
  • Event date: today's date. Use the same date of event when adding it as a Direct Investment Account (SVDI)
  • Total asset value: $0

To cease the income amount:

  • Expand the time share investment
  • Add a new investment details entry
  • Event date: today's date
  • Gross amount: $0
  • Frequency: blank

Note: income does not need to be coded. It will be assessed under the deeming provisions once the timeshare is coded on SVDI.

Key Direct Investment Account (SVDI) details + Read more ...

  • Go to SVDI and Add:
  • Header Details
    • Investment type code: select OTH – Direct investment
    • Investment name: the name of the organisation or timeshare
    • Investment date: the date of the purchase
    • Account number:
    • Customer owned %:
    • Partner owned %:
    • Deeming exemption: select Yes if the customer has applied for and been granted a deeming exemption for the timeshare investment. Note: the actual income from the investment is still assessable. Key Interest percentage %: to assess actual income
      Otherwise leave blank
      Note: to see if a product is exempt from deeming, see Deeming exemptions
    • Account closed: defaults to No and does not need to be changed when adding a new investment
  • Item Details
    • Event date:
    • Balance amount: the purchase price. In whole dollars
    • Interest percentage %: the actual rate of return. Leave blank if not deeming exempt
    • Assets test deduction amount: the amount of encumbrance (in dollars and cents) which is deductible from the balance of the investment, if applicable
  • Select Save
  • Update Receipt Date and Channel, select Save
  • Select Assess
  • Check the outcome on Entitlements (ELD) and select Finish
  • Add any extra details to the Finalise notes which has been pre-filled with details of the income and assets update
  • Select Finalise to complete the transaction

Aggregation + Read more ...

For Complex Assessment Officer (CAO) use only.

Managed investments owned by an entity are recorded on the entity record in Environment A.

For aggregation, all managed investments coded on the Managed Investments (MIN) system are assessed as non-primary production assets.

If a customer, trust or company runs a primary production business and invests some of its working capital in managed investments:

  • these may be recorded on the Other Assets (OAS) screen as primary production assets
  • manually code any updates to share/unit prices
  • only update current investments when details of all managed investments are provided

To determine if the investment is included in the Managed Investments (MIN) database, check if the investment is listed on the MIP screen.

Coding the MI screen + Read more ...

If bulk updates for 20 March or 20 September have been actioned:

When updating a managed investment and the bulk update for 20 March or 20 September has been actioned, the Event date will display as a date in the future. For example, today's date is 15 March and the event date shows 20 March.

To complete the update, follow the workaround for MIS/SIS updates immediately prior to before 20 March or 20 September.

Complex Assessment Officers - code managed investments owned by a trust or company, record the investment details on the entity’s record in Environment A.

Key investment details:

  • if the investment is unit-based, key the number of units in the Units: field. Note: the system will not allow the coding of decimals. If the customer owns a partial number of units, round down to the nearest whole number
  • if the investment is account based, key the amount (in dollars and cents) in the $: field
    • for margin loans, key the gross current market value
    • for personal investment funds code the total value of the portfolio including any cash component
  • if the investment is a funeral investment, key a valid field code in the Funeral exempt: field. Note: if the field code entered is 'Y', the investment is disregarded for the income and assets test
  • if the investment is owned in conjunction with the customer's partner, key the percentage owned by the customer and their partner in the % owned customer/partner: field
  • key the earliest of, the date the investment was made or the date the payment for the investment was transferred in the Event date: field
  • Policy number: key this field if the policy/account/certificate number is provided.
    (Note: if this number exceeds 10 characters, key the last 10 only)
  • key the date the investment was made (as shown on the certificate/policy) in the Investment date: field
  • key the loan balance amount (in dollars and cents) to be deducted from the investment value in the Assets test ded amt: field (if applicable). Note: the amount recorded is deducted from the investment amount for assets test purposes

Coding the MIUS screen for a customer + Read more ...

If bulk updates for 20 March or 20 September have been actioned:

Complex Assessment Officers - for managed investments owned by an entity:

Key the following fields:

    • Margin loans, key the gross current market value
    • Personal Investment Funds code the total value of the portfolio including any cash component
  • No of units: if the investment is unit based, the number of units. Note: the system will not allow the coding of decimals. If the customer owns a partial number of units, round down to the nearest whole number
  • Unit value: if the investment is unit based, the value of each unit
  • % owned customer/partner: the percentage owned by the customer and their partner (if superannuation, it cannot be jointly owned)
  • Event date: the earliest of, the date the investment was made or the date the payment for the investment was transferred

What to tell the customer + Read more ...

  • there is a bulk update to revalue all listed shares and unitised managed investments each March and September
  • customers may request a revaluation of their financial investments at any time
  • although their unitised managed investment prices are automatically revalued, customers are required to notify changes to:
    • the number of units they own
    • the value of their account based managed investment(s)
    • the balance of their margin loan(s)
  • for managed investments owned by entities (Complex Assessment Officers only), procedure ends here
  • for all other managed investments, go to Step 4 to set up a manual follow up (MFU) review for:
    • margin loans coded on MIP
    • unregistered superannuation products, including Self-Managed Superannuation Funds (SMSF)
    • personal investment funds
    • if required outside bulk updates, or as advised by FINS Helpdesk
  • otherwise, procedure ends here

4

Setting up a MFU review code for a new product + Read more ...

The following products are excluded from the bulk updates in March and September:

  • margin loans
  • unregistered superannuation products
  • personal investment funds
  • Self-Managed Superannuation Funds (SMSF)

Create a manual review for 20 March and 20 September:

  • for margin loans coded on MIP to request the latest statement from the lender showing the current market value of the managed investments and the current loan balance amount
  • for unregistered superannuation products (except Self-Managed Superannuation Funds – SMSFs) and personal investment funds recorded on MIIUS
  • if required outside the bulk updates, or as advised by FINS Helpdesk

For SMSFs, create a manual review every 12 months to request the latest financial statements for the fund to review the member balance

Procedure ends here.

Income already coded for a timeshare + Read more ...

Any income from timeshare investments coded on the OINS screen and any timeshare investment coded on the OAS/MIS screens must be ceased and the asset value recoded on the SVDI screen.

To cease the asset value, key:

  • Event Date: today's date (the same date of event is used for ceasing the timeshare when adding it on the SVDI screen)
  • Amount: $0

To cease the income amount, key:

  • Event Date: today's date
  • Amount: $0
  • Freq: blank

Note: income is not required to be coded as it will be assessed under the deeming provisions once the timeshare is coded on the SVDI screen.

  • Interest %: the actual rate of return, leave blank
  • Non FI deprvd income: the income amount for deprivation, if there is no asset value
  • % Owned: the percentage owned by the customer and/or partner
  • Event date: the date of event
  • Investment date: the date of purchase
  • Assets test deduction amt: the amount of encumbrance (in dollars and cents) which is deductible from the balance of the investment, if applicable
  • Deeming exempt: code 'Y' if the customer has been granted a deeming exemption for the timeshare investment, To see if a product is exempt from deeming, see Deeming exemptions, otherwise leave blank. If 'Y' is coded, actual income received from the investment is assessable - key the Interest %: field to enable assessment of actual income
  • The Source: and DOR: and Action: (using the code I) fields

Complete the activity on the AR screen.

Record the details on a DOC.
