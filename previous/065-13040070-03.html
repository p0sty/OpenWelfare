Checking a residential care service’s eligibility for Viability Supplement

Table 1: this table describes how to check residential care service's eligibility for Viability Supplement in the Aged Care Staff Portal (ACSP).

Note: the Aged Care Staff Portal (ACSP) automatically receives a service’s eligibility for the Viability Supplement via the Department of Health. No processing from Service Officers is required.

Step

Action

1

Check eligibility for Viability Supplement + Read more ...

Managing the Viability Supplement for home care

Table 2: this table describes how to process the Viability Supplement for home care recipients in Aged Care Management Payment System (ACMPS).

Step

Action

1

Manage the Viability Supplement for home care recipients using ACMPS + Read more ...

The Location Details event is used to maintain location details for care recipients receiving home care. Location details are used to determine viability supplement payments. The event can also be used to create a retrospective claim. A retrospective claim is a claim raised to make any modifications to a previously claimed event.

To Service Officers can manage the Viability Supplement for home care recipients in ACMPS by doing the following

  • To
    • view a Location Details event, go to Step 2
    • create a new Location Details event, go to Step 3
    • update an existing Location Details event, go to Step 4
    • delete a Location Details event, go to Step 5
    • create a retrospective Location Details event, go to Step 6
    • manage conflicting location Details, go to Step 7

2

View a Location Details event + Read more ...

Note: the Care Recipient Search option is located on ACMPS Home screen.

To view a Location Details event:

  • Log on to ACMPS
  • Select, Care Recipient Search option from the top menu
  • Key Care Recipient ID or search by surname, first name and DOB
  • Record death indicator as not set for current care recipient, or set for deceased care recipient
  • Select Search. The result list will display
  • Select the relevant care recipient. The Care Recipient Profile screen will display
  • Select the Assessments & Events tab
  • Location details event will display under the Location Details section

Procedure ends here.

3

Create a new Location Details event + Read more ...

All fields mark with an asterisk are mandatory. Optional fields will display and will vary depending on the type of event selected.

To create a new Location details event:

  • Log on to ACMPS
  • The Home screen displays
  • Select Event Create from the Create side menu or the Claim Events section
  • The New Event screen displays
  • Enter details in the following fields using the drop down list options:
    • Service*: field, key service ID stated on the form
    • Care Recipient*: field, key care recipient ID stated on the form
    • Care Type: field, defaults to Home care
    • Type of Event*: field, select Location Details
    • Requested Start: field, key start date or select date from calendar. This field is auto-populated with the current date. This date can also be on and/or after the entry date
    • Postcode: field, key the location postcode and press [Enter]. Service Officers will need to press [Enter] to prompt the drop down list in the Location Suburb
    • Location Suburb: select the suburb from the drop down list. This is only available once the location postcode has been entered
    • Requested End: field (optional), key end date of event or select date from calendar.This field is auto-populated with the default date of 31.12.9999. Request End dates should be after the entry date.
  • Select Save

The new Location Details event is now saved. The status is updated as 'Processed' after clearing validations.

Note:

  • If the event fails the validations, the status is updated as 'Invalid Location'. Invalid locations are work flowed to a payment processing team member to investigate and correct. They are reported on the Invalid Location Report
  • If a change in service by the care recipient results in conflicting location details being recorded by the new service, go to Step 7

If the event being processed by a user, other than a delegate, is outside the 2-year variation limit, it is then forwarded to a delegate for approval/rejection.

To create another event, select New to open a New Event screen and enter details.

Procedure ends here.

4

Update a Location Details event + Read more ...

To update a Location Details event:

  • Log on to ACMPS
  • The Home screen displays
  • Select Claim Events from the side menu
  • Select Event Search option, the Event Search screen displays
  • Key details into the displayed fields to search for Location Details event. Note: the search operation can be performed using a single parameter or multiple parameters. Select search parameters from the drop down box. If needed, search rows can be copied/removed by selecting the plus (+) or minus (-) icon
  • Select relevant operation for the search criteria
  • Enter the value for the search criteria
  • Select Search

Review the Claim events matching the search criteria displayed. To clear the search values, select Clear to clear the values in the Search fields, but retains any new field that might have been added.

To update a Location Details event:

  • Select the event to update from the search Results List
  • Select Edit and make the necessary updates to the fields as outlined in Step 2

Procedure ends here.

5

Delete a Location Details event + Read more ...

To delete a Location Details event:

  • Log on to ACMPS
  • The Home screen displays
  • Select Claim Events from the side menu
  • Select Event Search option, the Event Search screen displays
  • Key details into the displayed fields to search for Location Details event. Note: the search operation can be performed using a single parameter or multiple parameters. Select search parameters from the drop down box. If needed, search rows can be copied/removed by selecting the plus (+) or minus (-) icon
  • Select relevant operation for the search criteria
  • Enter the value for the search criteria
  • Select Search
  • Select the relevant event to delete from the search Results List
  • The Event Details screen will display
  • Select Cancel Event
  • Check the displayed status has changed to 'Deleted'
  • Select Save

Procedure ends here.

6

Create a retrospective Location Details event + Read more ...

A retrospective event can only be created for an event with an Event claimed status.

  • Log on to ACMPS
  • The Home screen displays
  • Select Claim Events from the side menu
  • Select Event Search option, the Event Search screen displays
  • Key details into the displayed fields to search for Location Details event. Note: the search operation can be performed using a single parameter or multiple parameters. Select search parameters from the drop down box. If needed, search rows can be copied/removed by selecting the plus (+) or minus (-) icon
  • Select relevant operation for the search criteria
  • Enter the value for the search criteria
  • Select Search
  • Select the relevant event to delete from the search Results List
  • The Event Details screen will display
  • Select Retro Claim to view the dialogue box displaying items of the retrospective event
  • Select the desired event
  • Select Choose
  • Review the displayed on the Retro Event screen to view the auto-populated data from the claimed event
  • Key the necessary changes
  • Select Save to create the retro

Procedure ends here.

7

Manage conflicting location details + Read more ...

A care recipient moves to a new service and the location event created does not have the exact location suburb and postcode details:

  • Location event entry fails validation or discrepancy is raised by service provider
    • Service Officers will investigate by comparing care recipient location details between previous and new service
  • New location details are correct
    • Payments team member contacts previous service to check location details, arrange for a manual fix or if appropriate create a retrospective location event
  • Previous location details are correct
    • Service Officer must contact the new service to check location details, arrange for a update
  • Conflicting location issue cannot be resolved through investigation or contact with services
    • Service Officer must escalate as a Tier 3 query to Aged Care Programmes Branch for assistance. See a Team Leader for the process
