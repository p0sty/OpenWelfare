Step

Action

1

Check partner details + Read more ...

A provisional record search can also be completed using the Application ID or Provisional ID of the enrolment application and/or the customer’s Consumer ID.

Review Application History table + Read more ...

Any newborn enrolments submitted appear in the Application History table.

  • Select the required enrolment radio button
  • Click Select
  • The Manage Provisional Enrolment Details screen displays

In PaNDA, multiple births are delivered as bundled work items. Process the Work Items in the order listed on the Newborn Child Declaration (FA081).

4

Review Manage Provisional Enrolment Details screen + Read more ...

Review the following information:

  • An Information Message appears if there are any potential issues relating to the consumer. These messages are displayed in red
  • Application Summary displays the Consumer who submitted the application for newborn child/children
  • Provisional Consumers displays the newborn/s to be enrolled
    • a newborn can be selected using the radio button on the left of the table
    • if multiple newborns were enrolled, they each have a unique Provisional ID. The Birth Order matches the order the applicant enrolled online. If the documents and/or any advice from customer suggest that this is wrong, process the enrolment manually, go to Step 8
  • If a consumer ID is recorded, search for it in Inquire on Consumer screen and check for any Australian Immunisation Register (AIR) record creation/activity. If any records exist process the enrolment manually, go to Step 8.
    See Individual registrations in the Australian Immunisation Register (AIR) for additional information or contact a Medicare and Aged Care - Local Peer Support (LPS) for advice
  • Additional enrolment information appears below in the following tabs. Potential eligibility or processing issue will present a warning message under Potential issues. The message will prompt the Service Officer to investigate the issue following business as usual processes before enrolling the child
  • Application Details tab displays the applicant’s responses to Medicare Safety Net, My Health Record and Duplicate Card request questions, their relationship and entitlement details and details for potential additional cardholders that they have elected
  • Applicant Details tab displays the applicant’s Consumer ID, Consumer Restrictions, name, gender, date of birth and address
    • Group Member Details displays the applicant’s name, gender, date of birth, entitlement type (Etm Type) and entitlement start date. See Complex eligibility and entitlement type of newborn child enrolments in Medicare for information about Medicare entitlement
    • If applicable, the date of death, consumer end date and entitlement end date also display
  • Medicare online accounts request:
    • proof of birth documents such as a birth certificate, or
    • a copy of adoption, guardianship, surrogacy or parental relationship
  • If the applicant has submitted all 4 pages of the Newborn Declaration (FA081) form, double check the address recorded in CDMS is up-to-date. If it does not match and the address recorded in the form is new, update the address accordingly. If unsure, attempt to contact the customer to confirm their address or contact a Tier 1 technical support - Local Peer Support (LPS) for assistance
  • Provisional details tab displays applicant’s response to newborn’s name, date of birth, gender, birth order (if applicable), descent status (if provided) and card name. The information in this tab can be edited. If there is a discrepancy between information displayed and details on the FA081, contact the applicant to confirm correct details
    Note: if the newborn has a long name, the card name will be truncated to fit on the Medicare card or the baby on own card checkbox will be selected if the parent has requested this
  • Documents tab displays the applicant’s relationship to the newborn/s and their entitlement. It also shows the submission date of document, document type submitted and document type to be recorded:
    • Confirm that the Document Type to be Recorded is accurate
    • Key the number or text of the document in Reference Text (for example, newborn child declaration form serial number located on back page, or registration number from birth certificate)
    • If required, key the Reference Date
    • Select the checkbox to the left of the Submission Date field
    • FAO81 Visually check the uploaded document to make sure it is genuine and there are no signs of tampering or forgery. If there are any of these types of concerns with the document do not discuss the application with the applicant or customer representative. Contact LPS for assistance. Leave the enrolment status as Open and do not continue with the application
    • Birth Certificate - see Table 9 for verifying identity document in DVS
  • Match Details displays any duplicate provisional consumer or consumer data matches
    • Provisional Consumer Data Match displays any unprocessed newborn applications whose details match with the selected application. Select the Provisional ID hyperlink to go to this enrolment
    • Consumer Data Match displays any existing consumer records whose details match with the selected application
  • Comments displays any comments relating to the enrolment. To add a new comment, key comment in Text field and select Validate
    These comments recorded do not transfer to the Comments field in the consumer’s CDMS record. Add comments to the CDMS personal record
  • Application Processing tab displays a summary of the enrolment
    • Status and Reason for rejection field displays the current status of the application, with an editable Status dropdown menu, Status options include Accept, Open and Reject, Reason for Rejection dropdown list (if applicable) and an Automatic Enrolment checkbox
    • If the Status is set to Reject, reasons for rejection include document issue, signature issue, duplicate registration and other issue
    • Select relevant checkbox/es about the Card and Decisions for Medicare Safety Net and My Health Record questions

Once all details have been provided and checked, do not select the Validate button. This is done when deciding the application processing outcome.

5

Accepting, rejecting or pending Provisional Registration Details + Read more ...

  • Select Application Processing tab
  • Resolve any potential issues before proceeding. For a full list of each issue, see Potential issues codes table
  • Make a decision on the application, select either Accept or Reject from the Status dropdown field
  • If no decision can be made, pend the work item in PaNDA and in the Application Processing tab in CDMS, leave the Status dropdown field as Open
  • If application is:
    • accepted, go to Step 6
    • rejected, go to Step 9
    • pended, resolve outstanding issue and then repeat Step 4

6

Application accepted + Read more ...

Status set to ACCEPT.

Make sure all checkboxes for the Card and Decisions headings are correct.

Card heading

  • Primary checkbox and field displays the Medicare card number that the newborn will be enrolled with
  • Duplicate checkbox is selected if applicant has requested a duplicate copy of their Medicare card
  • Additional checkbox and field displays the Medicare card number of an additional cardholder, as requested by the applicant
    Contact with additional cardholder is required to ensure they consent to have the newborn added to their card. For contact details of additional cardholder, go to the Application Details tab
    • If the additional cardholder does confirm consent to have their newborn added to their card, leave a note in the Comments tab, 'Additional cardholder contacted, permission granted to add newborn to card'
    • If the additional cardholder does not confirm consent, de-select Additional checkbox and continue with enrolment process. Leave a detailed note in the Comments tab
    • If contact with additional cardholder is unsuccessful, de-select Additional checkbox. Leave a detailed note in the Comments tab. The newborn can be added to another card after the enrolment is complete via copy/transfer process
    • If a Medicare card number was not provided, attempt to contact the applicant to confirm it. If contact with applicant is unsuccessful, de-select Additional checkbox. Leave a detailed note in the Comments tab. The newborn can be added to another card after the enrolment is complete via copy/transfer process
  • Baby’s Own Card checkbox is selected if the newborn is to be enrolled on their own Medicare card

Decisions heading

  • Medicare Safety Net checkbox is selected if the applicant has requested the newborn be added to an existing Medicare Safety Net, or be added to a newly created Medicare Safety Net. Check details of the applicant’s Medicare Safety Net, along with existing family members, in Inquire on Consumer screen in CDMS
    Note: if the newborn’s date of birth in a calendar year is before the Date Submitted, the system will not automatically add the newborn to the previous year’s Medicare Safety Net. If newborn is enrolled using automatic process (go to Step 7). Make sure that newborn is added to previous year’s Medicare Safety Net after the enrolment is complete, see Family registration and confirmation for Medicare Safety Net for assistance
  • Opt-out of My Health Record: if the checkbox is:
    • selected, the newborn will not receive a My Health Record
    • not selected, they will receive a My Health Record

Automatic enrolment streamlines the creation of a new consumer record without any re-keying of the information contained within. Make sure that all of the enrolment information is correct.

If the enrolment.

  • can be processed automatically with no further action, go to Step 7
  • requires further manual intervention to process, go to Step 8

7

Automatic enrolment + Read more ...

Leave the Automatic Enrolment checkbox selected then select Validate.

If the documents have not been validated in Documents tab, Service Officers are directed back before being able to proceed. Select the radio button beside each document/s and enter the required reference text.

The Confirm Provisional Enrolment Details screen appears. Make sure all the details are correct before proceeding:

  • Select Update to create a new consumer, or
  • Select Change to return back to make more changes

The Manage Provisional Enrolment Details screen returns with the following changes:

  • In the Provisional Consumers table, a new Consumer ID displays for the newborn
  • Only the Applicant Details, Provisional Details and Comments tabs display

If enrolling:

  • a single newborn, the procedure ends here
  • multiple newborns, return to Process Digital Newborn Enrolment screen, search for enrolment and select the second newborn using the radio button in the Provisional Consumer table
    Note: if the Additional card checkbox was selected, the newborn will be automatically added to the additional cardholders Medicare card

8

Manual enrolment + Read more ...

  • Uncheck the Automatic Enrolment checkbox and select Validate
  • The Confirm Provisional Enrolment Details – Manual Enrolment screen appears. Make sure all the details are correct before proceeding

Card table

  • Primary checkbox and field displays the Medicare card number that the newborn will be enrolled with
  • Duplicate checkbox is selected if applicant has requested a duplicate copy of their Medicare card
  • Additional checkbox and field displays the Medicare card number of an additional cardholder, as requested by the applicant. Contact with additional cardholder is required to make sure they consent to have the newborn added to their card. For contact details of additional cardholder, go to the Application Details tab
  • Baby’s Own Card checkbox is selected if the newborn is to be enrolled on their own Medicare card

Decisions table

  • Medicare Safety Net checkbox is selected if the applicant has requested the newborn be added to an existing Medicare Safety Net, or be added to a newly created Medicare Safety Net. Check details of the applicant’s Medicare Safety Net in Inquire on Consumer screen in CDMS
  • Opt-out of My Health Record. If applicant has selected:
    • yes, they will not receive a My Health Record
    • no, the newborn will receive a My Health Record

Actions

  • Select Update to create a new consumer, or
  • Select Change to return back to make further changes

Service Officer is directed to the Register New Consumer screen in Consumer Directory Maintenance System (CDMS). The details of the application appear in the Provisional Enrolment Summary page in a new window.

Depending on the data in the application, follow procedure as outlined in:

  • Process newborn enrolment to existing Medicare card in CDMS, or
  • Process newborn enrolment to their own Medicare card in CDMS

If enrolling:

  • a single newborn, the procedure ends here
  • multiple newborns, return to Process Digital Newborn Enrolment screen, search for enrolment and select the second newborn using the radio button in the Provisional Consumer table

Make sure comments are added in the Workload Manager (PaNDA) and the item is finalised. See View, add and edit processing notes to a work item table.

9

Application rejected + Read more ...

Status set to REJECT

  • Select an option from Reason for rejection dropdown menu:
    • Document Issue
    • Signature Issue
    • Duplicate Registration
    • Other
  • Select Validate

It is not necessary to select the relevant document and enter reference text in the Documents tab when rejecting an application.

If rejected due to Document Issue, Signature Issue or Other, try to contact the applicant to advise the outcome.

If contact is:

  • successful: advise them of the reason that their application cannot be accepted and suggest they use their Medicare online account to upload additional documents. This can be done by selecting Add documents to existing application in the applicant’s Medicare online account home page
  • unsuccessful: record a detailed note in the Comments tab, in the applicant’s consumer record Comments tab and the PaNDA work item

If rejected due to Duplicate, no further contact is needed.

The submission is displayed as Unsuccessful in the customer’s Medicare online account in the Newborn submission history page.

Make sure comments are added in the Workload Manager (PaNDA) and the item is finalised. See View, add and edit processing notes to a work item table.

7

View scanned images + Read more ...

  • Select Document Tools from the Quick Links menu
  • On the Document Tools screen, select View to display the scanned FA081

Scanned images can also be shown as unstructured documents (for example UNS007 or UNS008). These must be quickly opened and closed if not related to the newborn child’s enrolment. If any of these other images are used, annotate the MAENROL DOCs appropriately.

Sometimes only the last page of the FA081 form will be available. In this case, locate other scanned images uploaded after the FA081. For more information, go to Step 8.

To prevent errors on registering multiple births

  • Check the last page of the FA081 form to see if it is a multiple birth or not
  • Multiple birth newborns must be added in the right order of their birth as indicated by the midwife
  • If multiple births, notate twin 1 and twin 2 in personal comments in the Consumer Directory Maintenance System (CDMS)
  • If all the pages of the form are not available, contact the birth mother for confirmation

For multiple babies there will be multiple FA081 images and MAENROL DOCs. Open all images and read through.

All MAENROL DOCs must be annotated for the multiple birth babies.

Go to Step 9.

8

If only last page of FA081 available, locate newborn information + Read more ...

To locate information of newborn:

  • Go to Document List (DL) screen to view the Medicare Enrolment OLC DOC
  • The information is provided in the Text box

To prevent creating duplicate PINs

The submission of applications on multiple occasions occurs regularly. Service Officers can determine if the application has been submitted on multiple occasions and must not be actioned.

If the View Only warning appears within the Customer First record:

  • Do not begin the newborn enrolment. The View Only warning indicates that another Service Officer is in the record and is likely to be actioning the newborn enrolment for Medicare purposes
  • Go back to the Inbox and select Ready for work to get the next piece of work
  • Once the other Service Officer has completed the Medicare enrolment and completed all DOCs on the Centrelink record, the work will be removed from the workflow/inbox

9

Enrolling newborn if no information available in DL + Read more ...

If the Medicare Enrolment OLC DOC is not present:

  • do not proceed with the enrolment
  • attempt to contact the customer and advise them to supply the required documentation (full FA081) or complete a Centrelink online claim

Successful contact

If the customer advises other Centrelink claim DOCs (PPL/FTB) have been submitted and provides verbal permission to access and review the online claims, in Customer First:

  • key the customer’s CRN
  • select Workspace in the left menu
  • select Search Online Claims in Claims Online
  • select Search
  • select the relevant PPL/FTB online claim ID
  • select Review Claim from the top menu

The customer’s online claim information is displayed.

Enrol the newborn in Medicare using the information in the claim and the FA081 declaration.

Annotate the DOC and add comment on the Consumer Directory Maintenance System (CDMS) stating ‘Verbal permission obtained from customer to access PPL/FTB claim information for newborn enrolment’.

Go to Step 10.

Unsuccessful contact

If unable to contact applicant:

  • Add comment to Group in CDMS for applicant’s card:
    'unsuccessful attempt was made to contact applicant. Newborn not enrolled due to incorrect/missing OLC DOC'
  • Annotate the DOC (if there is one) on the Document List of CRN with the same details

Procedure ends here.

Enrol newborn in Consumer Directory Maintenance System (CDMS) + Read more ...
