Assess and locate information provided for Manual Income Confirmation (MIC) Interventions

Table 1: For Compliance Officers (CO).

Step

Action

1

Determine if a manual partner intervention is required + Read more ...

Determine if the updates will affect the customer's partner or ex-partner's payments. The Compliance Officer (CO) must ensure a partner intervention has been loaded and linked prior to a debt being raised.

If the partner intervention has not been loaded or linked, it is imperative that a partner intervention is created.

Do not load/initiate a manual partner intervention if the partner was not receiving a Centrelink payment during the period of the intervention or if Family and Domestic violence has been identified.

Is a manual partner intervention needed?

  • Yes, go to Step 2
  • No, go to Step 6

2

Create the manual partner intervention + Read more ...

On the New Review (IRNR) screen, complete the following fields in order:

  • Review Type: UII (Same type as the customer)
  • Review Reason: FDM for Family Day care. MPR for Trust.
  • Cycle number: match the one on the customer record
  • Service Reason: e.g. JSP, PPS, AGE
  • Review Method: same as customer
  • Review Origin: same as primary intervention on customer record

Once loaded, the partner intervention will show a status of REG (registered).

3

Link the customer record and partner record + Read more ...

In the Integrated Review System (IRS), go to the Link Summary (IRLS) screen and complete the following fields in order:

  • CRN column: code the other customer's Customer Reference Number (CRN)
  • Rev ID column: code the review ID of the customer's review that is to be linked
  • Lnk Rsn column: code a valid link reason, e.g. PTR - customers are partnered

For more on loading and linking reviews, see Linking reviews in the IRS.

4

Notify the partner + Read more ...

Speak to the partner during the initial call, or try an outbound call to the partner.

Is the partner available?

  • Yes, advise the partner of the intervention. Go to Step 5
  • No,
    • Issue a manual Q766 Notice of compliance review letter to the partner, including the Income Confirmation phone number
    • Create an Online advice
    • Go to Step 5

5

Start the partner intervention + Read more ...

To start the PTR intervention on the Review Contact Summary (IRCS) screen, code:

  • Method type: key 'OUT'
  • Date Contacted: key 'T' for today
  • Press [Enter]. When the Confirm Y/N box appears, key 'Y'

Press [Enter]. The Review Contact Details (IRCD) screen displays.

On IRCD screen code:

  • Expected response date: leave blank
  • Person contacted: enter the person contacted e.g. partner or ex-partner

Leave the intervention as STA.

6

Customer and third party responded to request for information + Read more ...

Has the request for information been provided from the customer or a third party?

  • Yes, go to Step 7
  • No, see Table 2

7

Locate income and assets information + Read more ...

Use the information obtained during the intervention to update the customer's and/or partner's record to determine:

  • historical qualification and entitlement
  • if any overpayment has occurred
  • current qualification and ongoing entitlement

For more information, see the user guide Assessing and coding income and assets.

If information relates to:

  • Family Day Care, see Table 3
  • Trusts, see Table 4

3

Apply Family Day Care match data to the customer's record + Read more ...

Before any updates on the customer's record, ensure the IRS intervention has a status of STA (started) and be in the Review Image.

If both the customer and third party have failed to respond to the request for information, the match data will be applied in its entirety.

Annualise FDC income if the customer started or ceased part way through a financial year. To work out the income to be assessed, use the formula:

  • Total amount of child care fees charged / number of day's income received x 365 = annualised total amount to be assessed

Complete the following action:

  • Code Real Estate/Business Identifying (REBI) screen fields
    • Business Type: Non Farming - Sole trader
    • Description: must state the customer's name and describe the business (Family Day Care)
    • Address: customer's address
  • Code Business Detail (BUS) screen fields
    • Event date: date started as per match data
    • Link New Real Estate: leave blank
    • Valuation Source: 'OTH'
    • Net Assets Customer $: $0
    • Income $: amount of Child Care Fees Charged as per the income available
    • Frequency: 'ANN'
    • % Income: 100

If the customer has multiple matches in a financial year, work out the income to be assessed. See Resources page for examples.

If the match data indicates the business has ceased update the BUS screen before finalising the activity. Code the following fields:

  • Event Date: code one day after ceased date
  • Income: code '0'
  • Income % code '0'
  • Frequency: 'ANN'

On the REB) screen, code Sold/Gifted/Ceased field: as 'CEA' (business has ceased).

If the match data indicates the business is no longer active (Cancelled or Closed), code the event date as 1 July for the next financial year.

Ensure all information and action taken is recorded in the Note.

After updating the income on the customer record, determine if there is a debt or adverse decision.

  • No, see Table 6

4

Determine JET Child Care fee assistance (JETCCFA) Additional Child Care Subsidy (ACCS) (Transition to Work) eligibility + Read more ...

Determine the customer's eligibility for ACCS.

For any periods the customer is not entitled to an income support payment, they are also not entitled to ACCS payments they may have received.

To help the Quality Development Officer (QDO) work out the amounts of ACCS and JETCCFA paid to the customer, send an email to the QDO via the QDO mailbox:

  • In the subject line: ACCS/ JETCCFA
  • In the email: 'Can you please provide the details of ACCS/ JETCCFA paid to (CRN), (customer's name) and (Date of Birth) from the (Date) to (Date)

ACCS/JETCCFA debts are raised and recovered under common law, under the Public Governance, Performance and Accountability Act 2013 (PGPA Act).

For more information, see Additional Child Care Subsidy (ACCS) (Transition to Work).

Determine, calculate and raise overpayments

Table 5: For Compliance Officers (CO)

Step

Action

1

Check if a Recovery Fee will apply + Read more ...

If the intervention includes verification of gross employment income paid including income from self-employment, determine if the Recovery Fee will apply.

The recovery fee will only be applied if the customer has not met their obligations. For example, they failed to provide information relating to their income confirmation.

The recovery fee will not be applied if the customer fulfilled their obligations. For example, they provided information relating to their income confirmation.

Note: employment income debts with a period of 14 days or less will be automatically raised on the completion of the income activity. The Recovery Fee is not applied to auto raised debts. The customer will receive the standard debt advice as part of the automatic process.

2

Determine debt and calculate + Read more ...

All debts that can be calculated via the Centrelink system and explained using ADEX should be done that way. Only use MultiCal if the Centrelink system calculation is not possible or the outcome is not correct.

Check the Assessment Result (AR) screen to make sure the system has correctly calculated any potential overpayment, for example:

  • Debt schedule - do the debt dates/date of event (DOV) match the period?
  • Have the updates/coding been entered correctly?
  • Check the following to make sure the updates are as expected:
    • Working Credit Balance Summary (WCBS)
    • Income Bank (NIB)
    • Employment Income Assessment (EIA)
  • Can a reassessment be made using the Benefit Action (BA) screen?
    • Use REA in relevant host system, or
    • RCR in PEN/PGA (for cancelled PEN and PGA records) and leave Reason blank

Note: check if Review of Assessment (ROA) coding is needed to generate allowable arrears. Do this when corrected income is less than declared, but the system did not generate arrears for those periods/fortnights.

If an ADEX debt calculation:

  • can be undertaken, the overpayment must be calculated with the customer over the phone where possible
  • is not possible, a MultiCal debt calculation may be required

For MultiCal debt calculation, get all information from the customer over the phone where possible to enable a debt calculation.

The CO is responsible for progressing the MultiCal activity. If they cannot action, their site is responsible for progressing the activity. The Team Leader will need to reassign the activity to another CO with the required skill set.

For more information, see:

  • Debt tools
  • Debt investigation
  • Actioning an undetermined debt on the Debt management and Information System (DMIS)
  • Using ADEX to explain a system generated debt
  • Using the correct date of effect when raising debts
  • Calculating Centrelink entitlements and gathering information when investigating debts
  • Waiving Centrelink debts

3

Contact customer and/or partner /ex-partner + Read more ...

  • Make 2 genuine attempts to contact the customer and the partner/ex-partner where it has been determined the partner/ex-partner has incurred a debt
  • Follow the phone contact requirements
  • Record all attempts to contact the customer in the Note

Was contact successful?

  • Yes, update the Review Contact Details (IRCD) screen to reflect customer contact. Go to Step 5
  • No, go to Step 4

4

Contact unsuccessful + Read more ...

After the first attempt:

  • update the Review Contact Details (IRCD) screen to reflect customer contact
  • code with 2attempt and add the due date of next business day

After the second attempt, if contact was:

  • unsuccessful proceed to raise the debt. Go to Step 6
  • successful, go to Step 5

5

Advise outcome + Read more ...

Advise customer

Advise the customer of the outcome of the intervention.

Thoroughly explain the decision, including any resulting debt.

Advise the customer of their review and appeal rights. See the Reviews and appeals of a Centrelink decision page on Services Australia website.

Advise partner/ex-partner

'Your partner/ex-partner (as appropriate) has been selected for a compliance intervention. The income from (income source) was not taken into account in the payments made to you, therefore you have incurred an overpayment of $xxxxx for the period from (date) to (date)'.'

Proceed to raise the debt.

6

Raise debt in accordance with guidelines + Read more ...

Staff should use the Debt Action script if possible.

  • To raise a JETCCFA debt, go to Step 7
  • For all other debts, go to Step 9

7

JETCCFA debts + Read more ...

Run the Debt Action script

  • At the Payment Type Selection choose: PGPA (Public Governance, Performance and Accountability Act 2013) followed by No Suitable Payment Found Under Any Category
  • The Debt Action script will stop at the Add Debt (OPAD) screen, prompting to add a manual code
  • Insert code 'JFA' (Jet Child Care Fee Assistance) on the Benefit type then continue to follow the script prompts
  • Insert code 'NEP' (Not eligible for payment) in the Reason field. Continue following prompts
  • Insert '11' in Section of ACT field and continue with script

8

JETCCFA debts issue manual Q417 + Read more ...

  • The script will prompt a manual Q417 letter to be sent to the customer
  • In the free text option of the letter, include the following wording:
    'You were not qualified to receive Jobs Education and Training Child Care Fee Assistance for the period xx/xx/xxxx to xx/xx/xxxx.'
  • Finalise the Debt Action script
  • Annotate the JETCCFA Debt DOC with the following information:
    • Debt raised under Rule 11 of the PGPA Act 2013
    • Debt Reason is NEP Not entitled to Payment
  • Scan all calculations for JETCCFA to the customer's record

9

Raise debt + Read more ...

Annotate the debt DOC with the following text:

'Intervention finalised as a result of the MIC Budget Initiative. Please transfer to 1800 086 400 if the customer can provide new information to support a reassessment of the initial overpayment, or if the explanation relating to the overpayment was unsuccessful.'

Suggested text to be used for Accounts Payable:

  • Customer suggested text
    'The correct amount of your income from (income source) was not taken into account in the payments made to you for the period from (date) to (date). This means you have been overpaid $(amount). We are, therefore, required to recover this amount.'
  • Partner/ex-partner suggested text
    'The correct amount of your partner's/ ex partner's income from (income source) was not taken into account in the payments made to you for the period from (date) to (date). This means you have been overpaid $(amount). We are, therefore, required to recover this amount.'

Raise debts as per current guidelines. See General debt raising information.

Debts < $50 can be waived under the current guidelines. See Small debt waivers.

Ensure the recovery fee question is answered.

Note:

  • The date entered into the date contact made with customer question should be the date the Debt Action script is run
  • If the debt amount generated via ADEX differs from the final debt amount being raised, fully document the reasons for this in the stand alone debt DOC

Deceased customer

If the customer is deceased and a debt is likely, before sending the debt advice, ensure a social worker is used to decide the best way to contact the executor. See:

  • MIC - Deceased Customers user guide
  • Requesting refunds and raising debts for deceased customer

Finalising MIC Interventions

Table 6: For Compliance Officers (CO)

Step

Action

1

Document the intervention + Read more ...

It is important all information is recorded in the Note to clearly explain:

  • customer's circumstances
  • information relating to the compliance trigger
  • scope of Intervention
  • customer's action
  • outcomes of the intervention

All Notes on a customer's record are to provide a clear and concise history of the customer's interactions. They must also comply with Online Document Recording (ODR) standards and Getting it Right (GIR) DOC minimum standards.

See the Resources page for requirements for documenting MIC interventions.

All compulsory scripts must be run to ensure that:

  • the DOC is appropriately updated, including the Extra Details line
  • all actions are recorded in the IRS

2

Finalising the intervention + Read more ...

Before finalising the intervention ensure all actions required by the CO are completed.

Use the Finalise Review script to finalise the intervention.

Check the following in IRS as part of the script flow:

  • Attribute/unattributed activities as per current guidelines
  • Choose an appropriate Result Reason as per current guidelines

If the customer has a linked review or an (ex) partner who also has an intervention, it is important to finalise all interventions at the same time.

Issuing a Q765 advice

Send a Q765 unless the intervention outcome:

  • has been advised over the phone
  • is related to a member of couple trigger and a decision letter has been issued
  • resulted in an adverse decision, e.g.
    • payment/card remains suspended/cancelled
    • payment was varied
    • debt advice has been issued

A Q765 can be issued if requested by the customer.

3

Further investigation required + Read more ...

After completing the intervention, if the CO believes there is a need for further investigation of declared income due to non-compliance outside scope, they can refer the case to Intelligence and Investigations Branch.
