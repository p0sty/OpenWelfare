Accessing the Participation Compliance Hub

Table 1

Item

Description

1

Accessing the workflows + Read more ...

In Process Direct:

  • select Customer Summary from the Process Direct Landing page
  • enter the customer's CRN, select Go
  • in the customer's record, key 'BIPCW' in the Super Key and press [Enter]
  • the Participation Compliance Hub will display

This is the launching point to access:

  • Generate Compliance Action workflow
  • Compliance Investigation workflow
  • Capability Assessment workflow
  • Comprehensive Compliance Assessment (CCA) workflow
  • Review Decision workflow

This screen also displays the details of completed compliance actions and the actions managed by providers (for example, participation suspensions).

The Help (?) icon will show information relevant to the current page.

The workflows are intuitive, displaying information and questions tailored to the compliance action in progress and workflow responses. This ensures staff discuss, consider and capture relevant details for that compliance action.

2

Generate Compliance Action + Read more ...

This workflow is used by Services Australia staff to manually generate a compliance investigation if:

  • an employer notifies that a job seeker:
    • refused to accept an offer of employment, or
    • did not start work as planned
  • a job seeker voluntarily ceases suitable employment or was dismissed from employment due to misconduct as a worker
  • a Comprehensive Compliance Assessment (CCA) is required for a Community Development Program (CDP) job seeker

See Table 2.

3

Outstanding compliance actions + Read more ...

The Outstanding compliance actions table, on the Participation Compliance Hub, lists compliance actions not yet completed.

Penalty investigations for job seekers managed under the Targeted Compliance Framework cannot be selected from this section until they are active failures.

If there is more than 1 outstanding compliance action, these display in the order in which the Participation Solutions Team (PST) skilled Service Officer should conduct the investigations. The order is:

  • unemployment failures
  • work refusal failures
  • all other compliance actions which are listed in the event date order, with the earliest to be actioned first

Under the event date of the compliance action, select the relevant workflow to open the appropriate investigation workflow. Note: this will vary depending on the compliance action to be taken:

  • Serious failures for persistent non-compliance: Comprehensive Compliance Assessment workflow
  • Unemployment failures, work refusal failures and mutual obligation failures: Compliance Investigation workflow
  • Unemployment Non-Payment Periods, No Show, No Pay failures, connection failures, non-attendance failures, reconnection failures and work related serious failures: Compliance Investigation workflow

A WNPPD, UNPP or PFEC IT security resource is required to complete the investigation. See Resources for more information.

4

Completed/Provider Compliance Actions + Read more ...

  • This section shows completed compliance actions and outstanding compliance actions the provider is responsible for managing
  • To view, expand the event date of the action to display detail about the completed compliance action
  • Select the Compliance workflow button to open the appropriate compliance workflow. Then select:
    • Participation Compliance Summary to view details on completed compliance investigation, Capability Assessment or Comprehensive Compliance Assessments
    • Re-Engagement to open the Re-engagement workflow
    • Review Decision to open the Review decision workflow
  • Participation suspensions under either job seeker compliance system can also be selected from the Completed/Provider compliance actions section to view details

5

Re-engagement + Read more ...

Services Australia is responsible for recording the re-engagement requirements for CDP job seekers. The Re-engagement page displays in the relevant workflows for certain Job Seeker Compliance Framework compliance events. It can also be accessed after completing a failure investigation.

Service Officers use this page to:

  • record if the job seeker does or does not need to be re-engaged
  • organise a provider re-engagement appointment using the link to the online diary
  • discuss the re-engagement requirement if the job seeker is present
  • record if re-engagement can occur within 2 working days
  • record the re-engagement outcome if 'no re-engagement' is required
  • deliver key messages to the job seeker advising the consequences of non-attendance at a re-engagement appointment

6

Save and Exit workflow + Read more ...

Some workflows have a Save and Exit workflow. The Save and Exit page should be used to capture the reason why compliance action cannot be completed and remove the contact trigger where appropriate.

Note: exiting the workflow using 'X' at the top right corner may not remove the contact trigger and not save any changes.

Recording a compliance investigation

Table 3: Using workflow during a compliance investigation.

Item

Description

1

Accessing the Compliance Investigation workflow + Read more ...

To access this workflow, select the Compliance investigation button for the event date of the compliance action from the Outstanding Compliance Actions section of the Participation Compliance Hub.

The workflow is used to undertake compliance investigations for both Targeted Compliance Framework and Job Seeker Compliance Framework.

A workflow menu displays on the left of the workflow window. This includes a list of workflow pages with the current page highlighted.

The workflow contains the following pages:

  • Other Related Information
  • Key Information
  • Investigation
  • Decision and Penalty
  • Re-engagement (Job Seeker Compliance Framework only)
  • Save and Exit

If appropriate, the workflow may also link to the Circumstances Impacting Compliance page.

The workflow is intuitive, displaying information, discussion points and questions tailored to the failure reason and the responses in the workflow. This ensures staff discuss, consider and capture relevant details.

All staff have general view access to the workflow. However, a compliance investigation can only be recorded by Service Officers with a WNPPD, UNPP or PFDEC IT security resource.

2

Other Related Information + Read more ...

This page displays links to information relevant to the compliance investigation.

Select this page from the link on the Compliance Investigation workflow menu.

Select links from this page to open another window displaying the required page or information. The Compliance Investigation workflow will remain open.

The following links display:

  • Provider Report
  • Job Plan
  • Provider Recorded Information
  • Online Diary
  • Job Seeker Calendar (Targeted Compliance Framework provider reported compliance action only)
  • Recent non-compliance
  • Services Australia Rates Estimator
  • Address History
  • View ESAt
  • Recent requirements and exemptions
  • Circumstances impacting compliance
  • Medical certificates

3

Key Information + Read more ...

This page is the starting point for a compliance investigation. It displays the information relevant to the job seeker and failure details on the failure event date.

Information includes:

  • Provider type
  • Payment type
  • Participation category
  • Indigenous Identifier
  • Existing circumstances impacting compliance: link 'View circumstances' is active where there are current circumstances recorded and the circumstance category will be listed
  • Failure event date
  • Failure reason
  • Potential penalty

The job seeker's current contact information is displayed, along with any language support identifiers.

If an interpreter has been engaged during the compliance investigation this must be recorded, and contact details confirmed.

4

Investigation Page + Read more ...

This page includes these sections:

  • Reason failure submitted
  • Excuse for failure
  • Prior contact
  • Further evidence

The questions are based on the type of compliance action under investigation, the job seeker compliance system applies and if the action was generated by the provider or Services Australia.

Reason failure submitted + Read more ...

The non-compliance event, the compulsory requirement and the potential penalty display, with a prompt to discuss this with the job seeker.

Read the Privacy statement as displayed in this section.

Excuse for failure + Read more ...

If the provider generated the compliance action, refer to the information reported by the provider as the starting point for the investigation:

  • For Targeted Compliance Framework failures, the workflow will display the reason given to the provider. The Service Officer is then prompted to ask the job seeker if there is anything else they want to add to explain the failure and record the job seeker statement
  • For Job Seeker Compliance Framework failures, the Service Officer reviews the report submitted by the provider by selecting the 'Provider Report Summary' link

There is a prompt to discuss and record the job seeker's reason for the failure.

If the job seeker statement suggests ongoing circumstances directly impacted their capacity to comply, access the Circumstances Impacting Compliance page from this section to assess the impact. The Compliance Investigation workflow will then resume from this section.

Based on the investigation, the Service Officer will be required to select the category and reason that best describes the job seeker's reason for non-compliance.

Depending on the reason selected, more information or questions may appear in relation to:

  • Using the Rate Estimator
  • Relocation assistance
  • Employment Separation Certificate (SU1)
  • Requirement in Job/Participation Plan
  • Requirement in job seeker calendar
  • Date and method of notification
  • Provider confirmation of suitability of work
  • Employment Income reported
  • Medical certificates recorded in last 12 months
  • Exemptions recorded in last 3 months

If a reasonable excuse assessment is required, the workflow presents questions that prompt the Service Officer to:

  • consider the reasonable excuse guidelines and
  • indicate if further investigation is required or if reasonable excuse has or was not established

If reasonable excuse is not established, the workflow will go to the Decision and Penalty page. A decision can be made to apply the compliance action.

See Assessing reasonable excuse for non-compliance with mutual obligation requirements.

Prior contact + Read more ...

This section will display only when an assessment is required.

Failure types requiring an assessment of prior contact are:

  • Targeted Compliance Framework:
    • PPA - Failure to attend provider appointment (or attend on time)
    • P3A - Failure to attend other provider appointment (or attend on time)
    • PAY - Failure to attend activity (or attend on time)
  • Job Seeker Compliance Framework:
    • RFA - Failure to attend provider reconnection appointment
    • COP - Failure to attend other provider appointment
    • FAR - Failure to attend regular provider appointment

Questions allow the recording of whether a job seeker made prior contact or if they had reasonable excuse for not doing so.

If the job seeker has not provided a reasonable excuse, the workflow will go to the Decision and Penalty page for a decision to apply the compliance action.

See Assessing reasonable excuse for non-compliance with mutual obligation requirements.

Further evidence + Read more ...

This section displays if responses in the workflow indicate there is enough evidence to make a decision. Further evidence should only be gathered if it will affect the decision. If possible, complete the compliance investigation on the day of contact.

Only 1 person can be selected. If more than 1 person has been contacted for further evidence, record the details of the primary contact, e.g. provider, employer or other. Capture all other contact details in the in the Summary of evidence requested/provided field, e.g. organisation/business name, contact person, contact number and a summary of the evidence provided.

The workflow does not give an option to hold Targeted Compliance Framework mutual obligation failure investigations. Decide using the available evidence. All other investigations may be held pending relevant evidence.

If it is appropriate to hold an investigation pending further evidence, record in this section:

  • what evidence has been requested
  • why this evidence will influence the outcome of the decision
  • the evidence due date

See Gathering information and weighing evidence to establish facts for a compliance investigation.

5

Copying information where multiple failures are being investigated + Read more ...

The Investigation page includes the option for information that has been recorded in the Excuse for failure section, Prior Contact section and the Evidence section to be copied to other failure investigations that require investigation at that time.

These are multiple non-attendance at activity failures (PAY) in the same instalment period (Targeted Compliance Framework).

The information will pre-fill in the workflow when starting the investigation for the selected failures.

6

Decision and Penalty Page + Read more ...

The Decision and Penalty page records the decision about the compliance action including the requirement to record information to support the decision to apply, or reject the compliance action.

The page comprises of the following sections:

  • Determine if failure occurred
  • Failure decision and penalty
  • Further action required

Determine if failure occurred

The section will display questions relating to the validity of the failure. Further questions specific to the failure type will appear (reasonable excuse, prior contact, inappropriate behaviour, voluntary unemployment or misconduct).

If rejecting the failure, select the rejection reason, record an explanation for decision and further information for the provider if appropriate.

Failure decision and penalty

The Service Officer is prompted to explain the decision and, if the decision is applied, the job seekerâ€™s review and appeal rights and to record if they intend to apply for a review of the decision.

Note: Where the decision and/or appeal rights are unable to be verbally advised, the PST-skilled Service Officers should finalise the decision in the workflow. Finalising the decision will issue a letter to the job seeker advising of the adverse decision and appeal rights.

For Job Seeker Compliance Framework serious failures and UNPPs:

  • the failure period can be adjusted
  • waiver provisions can be addressed

Further action required

This section includes prompts:

  • generation of a Comprehensive Compliance Assessment (CCA) (Job Seeker Compliance Framework only)
  • if the job seeker has requested a review of the decision to apply the failure, record Payment Pending Review (PPR) (Job Seeker Compliance Framework only)
  • Income Management
  • updating the Circumstances Impacting Compliance screen
  • finalise outstanding reporting

7

Re-engagement + Read more ...

This page helps to record the job seeker's re-engagement requirements. Services Australia has a responsibility to formally re-engage job seekers only when actioning the following Job Seeker Compliance Framework failures:

  • Failure to attend other provider appointment
  • Failure to enter into Job Plan
  • Serious failure - Pending Serious Failure Requirement

It is used by PST-skilled Service Officers to:

  • record if the job seeker does or does not need to be re-engaged
  • organise a provider re-engagement appointment using the link to the on-line diary
  • discuss the re-engagement requirement if the job seeker is present
  • record if re-engagement can occur within 2 working days
  • record the re-engagement outcome - if it is determined that 'no re-engagement' is required
  • notify the job seeker of the consequences of non-attendance at a re-engagement appointment

The Re-engagement page will not display for Targeted Compliance Framework compliance actions.

8

Save and Exit + Read more ...

This workflow presents after selecting the Exit button from any page in the workflow, or after selecting the Save and Exit page from the workflow menu.

It saves the changes made in the workflow before exiting. The page captures and displays the reason for leaving the workflow before completing a decision. The exit reason is visible to all staff when viewing the Participation Compliance Hub to help with general enquiries about a compliance investigation.

When exiting the workflow, the Service Officer must select the appropriate reason from the drop down list, and may need to complete free text.

The options are:

  • call unexpectedly terminated
  • pending evidence from customer (work refusal, unemployment failures and Job Seeker Compliance Framework failures only)
  • unable to complete on day of contact
  • check information not available in the workflow
  • unable to complete - system issues

Note: exiting the workflow using 'X' at the top right corner may not remove the contact trigger and not save any changes. The workflow will close without launching the Assessment Finalise workflow.

Key Information + Read more ...

  • Indigenous Identifier

The job seeker's current contact information displays, along with any language support identifiers.

4

Capability Assessment + Read more ...

This page prompts the Service Officer to explain to the job seeker the purpose of the Capability Assessment and read the privacy statement.

It also displays information about the recent Capability Interview undertaken by the job seeker's provider including:

  • demerit history that contributed to the current Capability Assessment
  • job seeker circumstances that were current as at the date of the Capability Interview
  • Job/Participation Plan that was reviewed during the Capability Interview
  • updates, new circumstances or actions taken since Capability Interview was conducted

The Service Officer must discuss with the job seeker if there is any other circumstances impacting capacity to comply that have not been considered as part of the Capability Interview.

If the job seeker advises they:

  • have other circumstances to be considered, the workflow navigates to the Circumstances impacting Compliance page for further investigation
  • do not have any other circumstances to consider, the workflow will navigate to the Capability Assessment Outcomes page

5

Circumstances Impacting Compliance + Read more ...

This page presents when the response for 'Does the job seeker statement suggest there are ongoing circumstances directly impacting their ability to comply with requirements?' is 'yes - has ongoing difficulties meeting requirements'.

The Circumstances Impacting Compliance page displays existing circumstance. If appropriate, circumstances can be reviewed and new circumstances added.

The following information will also be required:

  • Did the circumstances directly contribute to non-compliance events that occurred on or after the date circumstances started affecting?
  • Which requirement/s are they finding it difficult to meet?
  • How does the circumstance affect their ability to comply with the requirements?
  • What is the reason?
  • What dates/times can they attend?
  • What type of activities can they undertake?

If consultation with an experienced colleague, Participation Solutions Team (PST) social worker or specialist for assistance with assessing the impact of identified circumstances occurred, details of this consultation should be recorded.

Any free text recorded on this page must be specific and relevant to the circumstances recorded and the requirements that are impacted by the circumstances. Content must be clear, concise and factual. Sensitive personal information should not be recorded, instead the focus must be on the impact of circumstances and the affect this is having on the job seeker's ability to meet requirements.

Information on the Circumstances Impacting Compliance page is shared with the provider. Once the confirm change button has been selected, a message under the further action required section prompts the Service Officer to use the Save and Exit workflow to exit the Capability Assessment and review the Job Seeker Snapshot where circumstances are likely to impact. Once this action has been completed, the Capability Assessment should be resumed. This ensures any change that occurs as part of the Capability Assessment is reflected in the Capability Assessment outcomes.

For more information, see

  • Circumstances impacting capacity to comply with compulsory requirement, and
  • Capability Assessments

6

Capability Assessment Outcomes + Read more ...

This page is to record further action to be taken as a result of the Capability Assessment and the outcome.

The Service Officer must respond to the mandatory questions to indicate if:

  • there were errors identified in the Job/Participation Plan
  • an Employment Services Assessment (ESAt) is required
  • there was contact with the provider during the assessment and record a summary of the contact
  • further assessment is required such as a review of entitlement, temporary exemption from mutual obligation requirements

This page also captures who (name, logon, position) conducted the Capability Assessment and the date completed.

Based on the responses provided, the workflow will determine the outcome of the Capability Assessment. This is a display only and protected field. For more information, see Capability Assessments.

The workflow displays information relevant to the Capability Assessment outcome for the Service Officer to discuss with the job seeker.

7

Save and Exit + Read more ...

This workflow captures and displays why the Capability Assessment cannot be finalised during the job seeker's contact. This reason may help other staff quickly determine how to progress the Capability Assessment when contact with the job seeker is re-established.

When exiting the workflow, the Service Officer must select the appropriate reason from the drop down list and may also have to complete free text and select a review date.

The options are:

  • Unable to complete on day of contact
  • Call unexpectedly terminated
  • Pending evidence from customer
  • Exemption applied
  • Check information not available in the workflow
  • Unable to complete - system issues

Note: exiting the workflow using 'X' at the top right corner may not remove the contact trigger and not save any changes.

Comprehensive Compliance Assessment (CCA) workflow + Read more ...

Recording Contact + Read more ...

Review Record

The Review Record page displays information about the job seeker's current circumstances such as payment type, failure type, reason for CCA, work capacity, and requirements/exemptions. There are also links to other job seeker information that may be relevant to the CCA such as Provider Report Summary and Job Plan. CCA Details

The CCA Details page is used by Participation Solutions Team (PST)-skilled Services Officers when preparing a CCA and for CCA Specialist when conducting the CCA.

Preparing a CCA requires PST-skilled Service Officers to:

  • identify the appropriate CCA delivery method
  • record the type of notification issued to the job seeker
  • confirm that the job seeker was advised that their payment may be delayed if they don't comply with the notification
  • the date the CCA is expected to be conducted by
  • document any additional information given by the job seeker in relation to their non-compliance that will support the CCA

When starting the CCA, the CCA Specialist will review the information recorded.

Customer contact

The job seeker's current contact information is displayed, along with any language support identifiers.

Where an interpreter has been engaged during the preparation or completion of the CCA this must be recorded. Service Officers are required to confirm contact details are correct and record the details of the job seeker's contact.

3

Recording the investigation + Read more ...

Pages in the Investigation section of the workflow can be recorded by PST-skilled Service Officers (WNPPD) or PST Social Worker.

After recording the Investigation section, PST Social Workers will need to save and exit the workflow to hand off to the serious failure decision maker.

Third Party Consultation page

Where a third party has been consulted to gather further evidence in preparation or completion of the CCA, the details of the third party and summary of the relevant information collected is recorded in the Third Party Consultation page. Free text recorded on this page must be specific and relevant to the CCA. Content must be clear, concise and factual.

CCA Commencement page

CCA Specialist must declare in the workflow that they have:

  • explained the purpose of the CCA
  • explained the potential outcome of the CCA
  • read the Privacy Statement
  • explained which information will be shared with the provider

Circumstances Impacting Compliance page

This page presents for all CCAs. Any existing circumstances are listed and must be reviewed. New circumstances can be added. All circumstances the job seeker presents during the CCA must be considered to determine the impact on their ongoing capacity to comply.

The following information will be required:

  • category and current status selected from drop down menu options
  • free text description of the circumstance that is impacting compliance
  • evidence was used to confirm the impact of the circumstance from the list of displayed
  • free text explanation of how the circumstance impacts the job seeker's ability to meet their compulsory requirements

If consultation with an experienced colleague, PST social worker or specialist for assistance with assessing the impact of identified circumstances occurred, details of this consultation is also recorded.

Any free text on this page must be specific and relevant to the circumstances recorded, and the requirements they affect. Content must be clear, concise and factual. When recording content, CCA Specialists must consider that this information is forwarded to the job seeker's provider. If sensitive information is obtained during the CCA, take care to only record information about how the circumstance affects the job seeker's capacity to comply with requirements.

Interventions recorded in the most recent CCA display on the Circumstance Impacting Compliance page. Review the active interventions to ensure these are still relevant, where appropriate interventions can be added.

The CCA Specialist will record:

  • who is responsible for the intervention - Services Australia or Workforce Australia provider
  • the intervention category and intervention
  • outcome where appropriate
  • method of interview
  • details of the CCA

Other related information relevant for the CCA displays in this section. This includes further information recorded by the provider, e.g. Job Plan barriers, Job Seeker identified barriers, Job Seeker Snapshot and ESAt.

See Circumstances impacting capacity to comply with compulsory requirement.

4

Recording the serious failure decision + Read more ...

Pages in the Decision section of the CCA workflow require a WNPPD IT security resource to make updates.

Decision page

The serious failure decision maker confirms that they have assessed the following:

  • reasons why the job seeker may have committed failures
  • reasons why the job seeker may have failed to meet other requirements
  • whether the person has any barriers to employment
  • whether the person's participation requirements are appropriate

The serious failure decision maker will indicate if a pattern of non-compliance is persistent and then select the decision to apply or not apply a penalty.

If the decision is to not apply a penalty, further fields to complete include:

  • select the rejection reason
  • free text summary of why this decision has been made

If the decision is to apply, further questions will display to identify if a waiver may be appropriate. A free text summary of why this decision has been made must be completed.

The serious failure decision maker is prompted to explain the decision and for adverse decisions, their appeal rights and record if the job seeker intends to appeal.

Waiver Considerations page

This section will display where it was indicated on the Decision page that it is appropriate to consider waiving the penalty period.

It is used to record:

  • who initiated the waiver discussion
  • date of contact for assessment of waiver
  • for a UNPP
    • if serving the non-payment period would cause the job seeker severe financial hardship
    • if the job seeker is in a 'class of person' specified by the secretary
  • for a serious failure
    • if the job seeker has agreed to participate in a serious failure requirement
    • if the job seeker has the capacity to undertake a serious failure requirement
    • if serving the serious failure non-payment period will cause the job seeker to be in severe financial hardship
    • the reason for waiving the penalty period
    • in the case of a serious failure, link to the Re-engagement screen to book a waiver re-engagement appointment with the job seeker's provider

5

Re-engagement + Read more ...

Within the CCA workflow the Re-engagement page is used to record the re-engagement requirements and will only display where 'Pending Serious Failure Requirement' has been recorded on the Waiver Consideration page.

If the re-engagement is required, the re-engagement outcome is selected from the drop down menu.

6

Save and Exit Reason + Read more ...

The Save and Exit workflow captures and displays the reason the CCA is outstanding. When exiting the workflow, select the appropriate reason from the drop down list. If necessary, complete free text and select a review date.

The options are:

  • CCA pending consult with other specialist
  • CCA pending consult with third party
  • CCA pending evidence from customer
  • Check information not available in the workflow
  • Customer contact in relation to CCA
  • Customer did not attend specialist assessment
  • Decision not yet made - Other reasons
  • Further contact with customer required
  • Pending CCA Assessment
  • Pending contact with a third party
  • Pending ESAt/JCA for identified barriers
  • Pending face to face appointment
  • Record cancelled. Compliance investigation held
  • Return to PST for decision
  • Unable to complete - system issues

Note: exiting the workflow via X at the top right corner may not remove the contact trigger and not save any changes.

Recording a review of decision outcome

Table 6: Using the Review Decision workflow.

Item

Description

1

Review Decision workflow + Read more ...

This workflow is used to amend a failure decision, waive a serious failure period or Unemployment Non-Payment Period (UNPP), and re-engage the job seeker when Services Australia is required to do so.

A review of decision outcome can be the result of:

  • a request for explanation where a Subject Matter Expert (SME) changes the decision (Secretary initiated review)
  • a review by an Authorised Review Officer (ARO) or the Administrative Appeals Tribunal (AAT)

Review Decision + Read more ...

This displays the current status of the compliance action and is used:

  • for SME explanations, to record the date of request for the explanation
  • for ARO or AAT reviews, to record the date the review was requested
  • to update the status to reflect the outcome of the review of decision
  • to record the reason for the decision
  • to adjust the penalty components amount and/or penalty period start and end dates, if applicable

Status

The status field will display the current status of the failure. If the decision has been amended as a result of a review, the Service Officer will select the appropriate status from the options that are displayed. The options presented will depend on the failure type and the current status. The full list of options is:

  • Applied (APP)
  • Applied Without Penalty (AWP)
  • Payment Pending Review (PPR) (for Job Seeker Compliance Framework compliance actions only)
  • AAT Stay Order (STO) (for Targeted Compliance Framework compliance actions only)
  • Reapplied following Review (RAP)
  • Revoked (RVK)

Reason for the decision

The Reason for the decision field will only display where the status Revoked (RVK) is recorded. The Service Officer will be required to select the category and reason for revoking the applied compliance action.
