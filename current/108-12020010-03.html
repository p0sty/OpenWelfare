Coding a manual rate on the Rate Component Override (RCO) screen

Table 1:

Step

Action

1

Customers circumstances + Read more ...

See the relevant procedure if the customer requires RCO coding for the following reasons:

  • coding approved Overseas Full Time Study (OFS) for students
  • death of a dependent child of a JobSeeker Payment (JSP) or Youth Allowance (YA) customer
  • coding Special Benefit for an Australian Citizen Child or Australian Permanent Resident Children
  • assessing assets hardship provisions for allowances
  • completing the Accommodation Details (AC) screen and assessing Rent Assistance (RA)
  • assessing Self-Employment Assistance allowances
  • Initial remediation on a multiple or intertwined Centrelink Customer Reference Number (CRN)
  • assessing entitlement to the Long Term Income Support rate of Austudy for students or migrants studying English language courses
  • assessing entitlement to Youth Allowance (YA) special rate for Long Term Income Support Student or migrants studying English language courses
  • assessing and coding a transfer to Austudy or Youth Allowance (YA) student claim from another payment

Is a manual rate required for a reason not listed above?

  • Yes, and:
    • customer is in receipt of ABSTUDY, see ABSTUDY system coding for manual rate of payment. Procedure ends here
    • customer is in receipt of a non-NSS payment, see Non NSS systems coding for manual rate of payment. Procedure ends here
    • otherwise, go to Step 2
  • No, if the manual rate needs to be ended go to Step 7, otherwise procedure ends here

2

Manual rate required + Read more ...

Note: RCO coding is only required in limited circumstances. If unsure about RCO coding, consult with a Service Support Officer (SSO) or the Level 2 Policy Helpdesk.

Is RCO screen coding required to complete an unapproved workaround or change payment rate?

  • Yes, for:
    • Smart Centre Staff, escalate record to Local Peer Support
    • Service Centre Staff, can escalate to an SSO or the Level 2 Policy Helpdesk. Go to Step 3
  • No, and the approved workaround or change in payment rate is in an Operational Message or current NNU:
    • Follow the process to code RCO
    • DOC the record with the reason for RCO coding
    • Procedure ends here

3

Manual RCO screen coding because of system issues + Read more ...

Is an RCO workaround required due to a system issue impacting payment rate and/or calculating correct arrears?

  • Yes, escalate issue through MySupport for further investigation. Procedure ends here.
  • No, view the !SSLS screen to view SSL payment. If the customer:
    • received a Student Start-up Loan (SSL) during the manual rate period (s), go to Step 4
    • did not receive a Student Start-up Loan (SSL) during the manual rate period (s), go to Step 5

4

Customer received a Student Start-up Loan during manual rate period + Read more ...

Do not use workarounds with zero rate coding on RCO if the customer received a SSL during the manual rate period.

Is RCO coding applying a $0 rate in the Amount/No field?

  • Yes, do not use the RCO screen to complete the update. If no other options are available to complete the update, escalate the record to Level 2 Policy Help Desk:
    • CSAQuery: Education
    • CSAEnquiry: ABSTUDY/Austudy/YAL (Students)
    • Service/Payment Type: ABSTUDY/Austudy/Youth Allowance - Student. Note: check the Svc Rsn field on the !SSLS screen to identify the income support payment SSL was paid with
    • Procedure ends here
  • No, go to Step 5

5

Code the RCO screen + Read more ...

Access the RCO screen, key:

  • the start date for the override in the Start Date field
  • the end date in the End Date field. For a non-standard rate it will be for a defined period. Note: if an end date is not recorded consider coding a manual review on the Review Registration (RVR) screen to review RCO end date, such as 12 months
  • the override type in the Type field (use field help (?) to determine the relevant type)
  • the reason in the Reason field (use field help (?) to determine the relevant reason)
  • the new rate amount in the Amount/No field, if necessary (dollar amount is required depending on override type). See the References page for a link to the Payment Rates in the Social Security Guide for current and historical rates of payment
  • if necessary adjust the Working Credit via the W/C Manual Override (WCRC) screen
  • if the customer was receiving Adjusted Disability Pension from DVA before 1 January 2022 and the circumstance change is for a date before 1 January 2022:
    • check if the customer is eligible for any Defence Force Income Support Allowance (DFISA). If so authorised staff in the DVA Clearance Team must code a manual calculation on the DFISA Calculation Summary (DFS) and DFISA Calculation Details (DFD) screens
    • the system will transmit this information to the DVA system to pay the DFISA payment
  • Source and DOR fields. Press [Enter]

6

System result + Read more ...

The system will update the customer's record and pay the new rate of payment:

  • record information on a DOC advising reasons for non-standard rate of payment
  • create a Display on Access (DOA) DOC to alert Service Officer at reassessment a manual rate applies. See Creating, reviewing and deleting documents (including Fast Notes and DOA DOCs)
  • code a new manual review with appropriate due date no longer than 12 months. See How to code and action a manual review
  • create a Q999 to advise the customer of the new rate
  • finalise activity on Assessment Results (AR) screen

Procedure ends here.

7

Ceasing a manual rate + Read more ...

To cease non-standard rate for a customer, access the Rate Component Override (RCO) screen:

  • key 'C' beside the line to be amended
  • key the date the non-standard rate is to cease in the End Date field
  • key Source and DOR fields and press [Enter]
  • if necessarily adjust Working Credit via the W/C Manual Override (WCRC) screen
  • end date the Display on Access (DOA) DOC from the date the manual rate ceases to apply. See Creating, reviewing and deleting documents (including Fast Notes and DOA DOCs)
  • record details on a DOC advising non-standard rate of payment to cease
  • finalise activity on Assessment Results (AR) screen

Procedure ends here.

2

Ending manual rate of payment + Read more ...

End date the Manual rate on the RCO screen.

In Process Direct:

  • Go to the Rate Component Override (RCO) screen
  • Select Edit
  • End Date: code date manual rate no longer applies
  • Select Assess
  • Select Finish
  • DOC record with reasons manual rate no longer applies
  • Go to the Document List (DL) screen
  • Expire Display on Access (DOA) DOC. See Creating, reviewing and deleting documents (including Fast Notes and DOA DOCs)

In Customer First:

  • Next field: Go to Rate Component Override (RCO) screen
  • Correct 'C' current RCO line
  • End Date: code date manual rate no longer applies
  • code Source and Date of Reciept fields
  • Next field: go to Assessment Results (AR) screen
  • check if future payments are correct
  • select Finalise
  • DOC record with reasons for ending manual rate
  • Expire Display on Access (DOA) DOC. See Creating, reviewing and deleting documents (including Fast Notes and DOA DOCs)
