Telephone Authority approval application

Table 1: This table describes how Service Officers process a telephone Authority approval application via Mainframe.

Step

Action

1

Log on and identify patient + Read more ...

Log in to Mainframe under Record Telephone Authority.

Key the patient’s Medicare card number.

Is the Medicare card number valid?

  • Yes, go to Step 2
  • No, correct the error or key the patient’s name and date of birth in the appropriate fields to initiate a search in RPQN.

Note: read patient comments if available. If the request is for a patient who does not have a current Medicare number for example: a newborn, patient with an expired card who the prescriber considers entitled to PBS subsidy, or an overseas visitor, see Step 4 in the Recording new patient or visitor in Mainframe or Online PBS Authorities system

2

Enter information + Read more ...

Enter the following information:

  • authority prescription number
  • doctor's prescriber number
  • medication requested
  • patient's dosage

For Authority required (STREAMLINED) items – a warning message “THIS IS A STREAMLINED ITEM. PBS8105” will appear:

If the item is an Authority required (STREAMLINED), and the PBS list quantity and repeats are requested:

  • Service Officers can provide the Streamlined code after reminding the prescriber to make sure their patient meets the PBS criteria. When ending the phone call, remind the prescriber:
    • for items that are Authority required (STREAMLINED) for the PBS listed quantity and repeat(s), there is no need to contact Services Australia
    • current Streamline Codes can be found on pbs.gov.au by searching for the item
    • streamlined codes must be endorsed on the prescriptions after they have made sure that the patient meets the PBS criteria

If the request is for an increased quantity and/or repeat(s), and the restriction allows, an authority approval can be given.

Note: for Authority required items, the item restrictions will appear automatically after selecting the item.

See also: Authority approval processing for Pharmaceutical Benefits Scheme (PBS).

3

Determine if request is within guidelines + Read more ...

Calculate if request is within the guidelines and enter:

  • dosage directions
  • quantity and repeats requested

Note: if entering increased quantities and repeats for Unrestricted and Restricted items, the item restrictions appear.

See also: Authority approval processing for Pharmaceutical Benefits Scheme (PBS).

4

Check history + Read more ...

Press [PF10] to check history for previous applications. Assess for:

  • 'too soon', see Handle 'too soon' Authority prescription
  • dose changes

See also: Call enquiries in Pharmaceutical Benefits Scheme (PBS).

5

Enter assessment result + Read more ...

Enter assessment result and any applicable authority comments, see Codes in Mainframe for Pharmaceutical Benefits Scheme (PBS).

Press [PF8] to update and read approval number to approved prescriber.

Note: add patient comments if required.

6

Process approved prescriber request for another authority + Read more ...

If the approved prescriber requests another authority for:

  • the same patient press [PF6]
  • a different patient press [PF11]

Note: when call is complete, press [PF3] to return to the Record Authority - Patient Search screen.

Telephone approval Authority prescription received

Table 2: This table describes how to proceed if a previously phone approved Authority prescription item is inadvertently posted to/received by Services Australia.

Step

Action

1

Check paper prescription details + Read more ...

If the approved prescriber has put a telephone approval number (for example, Z1234AB or M1234RX)) on the prescription but has sent the paper prescription to Services Australia, the prescription does not require further approval.

Access Mainframe and check that all the details in the paper prescription match exactly to an entry on Mainframe.

  • If the details are correct go to Step 2
  • If some details are incorrect or a discrepancy is identified, escalate to Program Support Officer (PSO)/Pharmaceutical Adviser (PA)

2

Identify whether to send the prescription to the approved prescriber or the patient + Read more ...

For HPOS Form Upload prescriptions

Always return requests to the approved prescriber by responding to the original request in HPOS with:

  • Please read the attached letter for the outcome of your recent PBS authority approval application
  • Attach letter Your Patient’s authority prescription is already approved

For mailed prescriptions

If the prescription is marked for return to the patient:

  • copy the original Service Officer's details into Mainframe Authority comment and add Mainframe Authority comment 'Rx sent to Services Australia, now sent to patient'
  • post the prescription to the patient

If the prescription is not marked for return to the patient:

  • copy the original Service Officer's details into Mainframe Authority comment and add Mainframe Authority comment 'Rx sent to Services Australia, now sent to approved prescriber'
  • post the prescription back to the prescriber with letter "Your Patient's authority prescription is already approved"

The Resources page has a link to the letter outlined above.
