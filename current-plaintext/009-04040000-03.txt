| 4                                 | Contact carer + Read more ...     |
|                                   |                                   |
|                                   | Contact the carer to confirm      |
|                                   | details in the following          |
|                                   | situations.                       |
|                                   |                                   |
|                                   | Care receiver getting CA for      |
|                                   | another adult care receiver       |
|                                   |                                   |
|                                   | The care receiver cannot qualify  |
|                                   | for CA for another disabled       |
|                                   | adult. Before granting, rejecting |
|                                   | or cancelling any CA payments,    |
|                                   | additional information is         |
|                                   | required from the carer and the   |
|                                   | care receiver to clarify the      |
|                                   | circumstances and whether the     |
|                                   | care receiver has ceased          |
|                                   | providing care. Where another     |
|                                   | person is already/still providing |
|                                   | care for the care receiver, see   |
|                                   | Change of Care for Carer Payment  |
|                                   | (CP) and Carer Allowance (CA)     |
|                                   | customers.                        |
|                                   |                                   |
|                                   | When determining the start date   |
|                                   | for CA, the decision is           |
|                                   | independent of whether the care   |
|                                   | receiver has been paid CA, the    |
|                                   | date CA was cancelled or whether  |
|                                   | the care receiver is entitled to  |
|                                   | CA. The start date determination  |
|                                   | should follow the normal start    |
|                                   | date procedures that apply for    |
|                                   | new claims.                       |
|                                   |                                   |
|                                   | Multiple CP claims/care receivers |
|                                   |                                   |
|                                   | If a carer lodges a claim for CP  |
|                                   | for more than one care receiver,  |
|                                   | or multiple claims for CP for 2   |
|                                   | or more carer receivers, it is    |
|                                   | deemed that the customer has      |
|                                   | lodged a claim for each care      |
|                                   | receiver. Each claim must be      |
|                                   | assessed. If the carer is         |
|                                   | eligible for more than one care   |
|                                   | receiver, a determination of who  |
|                                   | the carer is claiming for must be |
|                                   | made.                             |
|                                   |                                   |
|                                   | More than one eligible care       |
|                                   | receiver                          |
|                                   |                                   |
|                                   | Contact the carer to discuss:     |
|                                   |                                   |
|                                   | -   CP can be paid for only one   |
|                                   |     care receiver                 |
|                                   | -   the circumstances of the      |
|                                   |     carer and care receivers      |
|                                   | -   the implications of the       |
|                                   |     decision on bereavement       |
|                                   |     provisions                    |
|                                   | -   which care receiver they want |
|                                   |     to claim for                  |
|                                   |                                   |
|                                   | Document contact with the carer   |
|                                   | on the carer's record. If the     |
|                                   | carer cannot be contacted, a      |
|                                   | letter must be sent requesting    |
|                                   | they contact before the claim is  |
|                                   | finalised.                        |
|                                   |                                   |
|                                   | When the carer advises which care |
|                                   | receiver they want to claim for:  |
|                                   |                                   |
|                                   | -   withdraw the additional       |
|                                   |     claims, and                   |
|                                   | -   finalise the claim for that   |
|                                   |     care receiver                 |
|                                   |                                   |
|                                   | If the carer cannot be contacted  |
|                                   | and does not respond to the       |
|                                   | request to contact Services       |
|                                   | Australia, reject the claims.     |
|                                   |                                   |
|                                   | Multiple care receivers:          |
|                                   |                                   |
|                                   | -   if no care receivers are      |
|                                   |     eligible, reject the claim    |
|                                   |     for each care receiver using  |
|                                   |     the most appropriate reason   |
|                                   | -   if only 1 care receiver is    |
|                                   |     eligible, reject the claim    |
|                                   |     for each of the ineligible    |
|                                   |     care receivers before         |
|                                   |     processing the claim for the  |
|                                   |     eligible care receiver        |
|                                   | -   create a SOA for each care    |
|                                   |     receiver for action           |
|                                   |                                   |
|                                   | All claims must be finalised,     |
|                                   | unless withdrawn.                 |

| 5                                 | Streaming of claims + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | All processing staff are to       |
|                                   | stream claims before making a     |
|                                   | referral or placing the claim on  |
|                                   | hold.                             |
|                                   |                                   |
|                                   | Streaming is the practice of      |
|                                   | coding of all information that    |
|                                   | has been provided or is available |
|                                   | when the claim is first           |
|                                   | allocated, including:             |
|                                   |                                   |
|                                   | -   Identity for carer and care   |
|                                   |     receiver                      |
|                                   | -   Medical Information           |
|                                   | -   Income and Asset details      |
|                                   | -   Care Receiver information     |
|                                   | -   Any other information that    |
|                                   |     requires coding, provided     |
|                                   |     with the claim                |
|                                   |                                   |
|                                   | This is to ensure that claims are |
|                                   | not being placed on hold          |
|                                   | unnecessarily, where a decision   |
|                                   | could be made about the claim.    |
|                                   |                                   |
|                                   | An example of this is, where it   |
|                                   | is identified that a claim        |
|                                   | requires a referral to an         |
|                                   | external assessment such as a     |
|                                   | Complex Assessment Officer, but   |
|                                   | the medical report provided       |
|                                   | indicates that the customer does  |
|                                   | not meet medical qualification    |
|                                   | for the respective payment.       |
|                                   |                                   |
|                                   | There is sufficient information   |
|                                   | to make a decision about this     |
|                                   | claim without proceeding with a   |
|                                   | referral.                         |
|                                   |                                   |
|                                   | Note: for claims that have been   |
|                                   | streamed, see Hold to user new    |
|                                   | claim criteria to determine if    |
|                                   | claim can be held to user.        |

| 9                                 | Residence screens + Read more ... |
|                                   |                                   |
|                                   | The customer's Australian         |
|                                   | residence qualification will be   |
|                                   | auto-assessed based on the        |
|                                   | residence information recorded.   |
|                                   |                                   |
|                                   | Residence information must still  |
|                                   | be recorded if the claim is to be |
|                                   | rejected for a reason other than  |
|                                   | residence (including for          |
|                                   | streamline rejections). If        |
|                                   | residence details are not         |
|                                   | recorded, the system will         |
|                                   | override any other rejection      |
|                                   | reason and the claim will reject  |
|                                   | for a residence related reason.   |
|                                   |                                   |
|                                   | If residence requirements are not |
|                                   | met but the customer has lived in |
|                                   | an Agreement country, they may be |
|                                   | qualified for Carer Payment under |
|                                   | an Agreement.                     |
|                                   |                                   |
|                                   | Does the customer qualify under   |
|                                   | an International Agreement?       |
|                                   |                                   |
|                                   | Yes:                              |
|                                   |                                   |
|                                   | -   ensure all residence screens  |
|                                   |     are coded                     |
|                                   | -   confirm the customer's        |
|                                   |     identity                      |
|                                   | -   ensure only required work     |
|                                   |     items are left open           |
|                                   | -   take the claim off hold and   |
|                                   |     categorise in Work Optimiser  |
|                                   |     to a CIS new claim work item  |
|                                   |     Note: no separate referral    |
|                                   |     necessary. Procedure ends     |
|                                   |     here                          |
|                                   |                                   |
|                                   | No, go to Step 12                 |
