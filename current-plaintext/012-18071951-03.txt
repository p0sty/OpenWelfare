This page contains the processes to follow for online registrations of
Healthcare Provider Identifier - Organisation (HPI-O) record for seed
organisations in Healthcare Identifiers (HI) Service, and how to resolve
error codes.

On this page:

Process online registrations of HPI-O records for seed organisations

Re-open and complete Documents tab – Authority to Act

Error codes and actions to escalate to the Provider HI Service and My
Health Record SME

Process online registrations of HPI-O records for seed organisations

Table 1: this table tells Service Officers how to process an online
registration for a seed organisation. The process below will occur when
an online application goes into a pending state.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Log on to the EHP (eHealth        |
|                                   | Program) Home Page + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | Open 2 EHP screens:               |
|                                   |                                   |
|                                   | -   one to process the online     |
|                                   |     application                   |
|                                   | -   the other to work through     |
|                                   |     issues of the pending         |
|                                   |     application, for example to   |
|                                   |     check details of existing     |
|                                   |     records or duplicate          |
|                                   |     applications                  |
+-----------------------------------+-----------------------------------+
| 2                                 | Is the WLM (Workload Management)  |
|                                   | work item available? + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   Yes, go to Step 3             |
|                                   | -   No, go to Step 4              |
+-----------------------------------+-----------------------------------+
| 3                                 | Locate application + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | In EHP:                           |
|                                   |                                   |
|                                   | -   select Application in the HI  |
|                                   |     Search menu bar               |
|                                   | -   paste the Work Item ID from   |
|                                   |     WLM into the Application ID   |
|                                   |     field                         |
|                                   | -   select Search                 |
|                                   |                                   |
|                                   | Go to Step 5                      |
+-----------------------------------+-----------------------------------+
| 4                                 | WLM work item is not available +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | In EHP the Search for individual  |
|                                   | screen displays.                  |
|                                   |                                   |
|                                   | On one EHP screen:                |
|                                   |                                   |
|                                   | -   select Application in the HI  |
|                                   |     Search menu bar               |
|                                   | -   select the ‘down arrow’ in    |
|                                   |     the Status field              |
|                                   |                                   |
|                                   | 6 status types will appear:       |
|                                   |                                   |
|                                   | -   Approved                      |
|                                   | -   Approved (Auto)               |
|                                   | -   Approved (Manual)             |
|                                   | -   Pending                       |
|                                   | -   Processing                    |
|                                   | -   Rejected                      |
|                                   |                                   |
|                                   | For applications that need        |
|                                   | operator intervention:            |
|                                   |                                   |
|                                   | -   select Pending                |
|                                   | -   select the Search button      |
|                                   | -   a list of pending             |
|                                   |     applications show in the      |
|                                   |     Search Results screen         |
|                                   | -   select the next Pending       |
|                                   |     application available in the  |
|                                   |     list                          |
|                                   |     Note: only use this step if   |
|                                   |     WLM (Workload Management) is  |
|                                   |     not available                 |
+-----------------------------------+-----------------------------------+
| 5                                 | Check Comments tab + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | Open Comments History tab to view |
|                                   | all errors first.                 |
|                                   |                                   |
|                                   | This allows at a glance to check  |
|                                   | if the Responsible Officer (RO),  |
|                                   | Organisation Maintenance Officer  |
|                                   | (OMO) or organisation already     |
|                                   | exists in EHP, as well as other   |
|                                   | errors that need resolving.       |
+-----------------------------------+-----------------------------------+
| 6                                 | Process application + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   Select the relevant tab to    |
|                                   |     list any errors that need     |
|                                   |     correction                    |
|                                   | -   Check Steps 7, 8, 9 and 10    |
|                                   |     for each tab                  |
|                                   | -   Include comments in the       |
|                                   |     Comments box                  |
|                                   |                                   |
|                                   | The one Comments box appears at   |
|                                   | the bottom of the screen for all  |
|                                   | tabs, not one for each tab. Add a |
|                                   | new line in the Comments box for  |
|                                   | all comments.                     |
|                                   |                                   |
|                                   | Do not select the Save Comments   |
|                                   | button. If selected the           |
|                                   | application closes and it will    |
|                                   | need to be reopened.              |
+-----------------------------------+-----------------------------------+
| 7                                 | Organisation tab + Read more ...  |
|                                   |                                   |
|                                   | If there are errors to process,   |
|                                   | cross check to see if there is a  |
|                                   | duplicate EHP record or a         |
|                                   | duplicate application.            |
|                                   |                                   |
|                                   | Check if the organisation already |
|                                   | exists in EHP or if there is a    |
|                                   | duplicate application.            |
|                                   |                                   |
|                                   | Organisation in EHP:              |
|                                   |                                   |
|                                   | -   Contact the applicant to      |
|                                   |     advise that the organisation  |
|                                   |     is already registered. Leave  |
|                                   |     the following message if      |
|                                   |     unable to speak to the        |
|                                   |     applicant:                    |
|                                   |     ‘Your application to register |
|                                   |     your organisation in the HI   |
|                                   |     Service was not successful    |
|                                   |     because this organisation is  |
|                                   |     already registered in the HI  |
|                                   |     Service. You can look up your |
|                                   |     Healthcare Identifier in      |
|                                   |     Health Professional Online    |
|                                   |     Services, available on the    |
|                                   |     Services Australia website.’  |
|                                   | -   Copy and paste the following  |
|                                   |     text into the Comments box:   |
|                                   |     ‘Organisation exists in EHP.  |
|                                   |     Contacted applicant. Rejected |
|                                   |     application.’                 |
|                                   | -   Reject application - select   |
|                                   |     Reject. Do not re-categorise  |
|                                   |     in WLM                        |
|                                   | -   An email will be sent to the  |
|                                   |     applicant’s Health            |
|                                   |     Professional Online Services  |
|                                   |     (HPOS) mail centre advising   |
|                                   |     them of the unsuccessful      |
|                                   |     outcome, and that they will   |
|                                   |     need to complete the          |
|                                   |     application again if they     |
|                                   |     still wish to register        |
|                                   | -   Procedure ends here           |
|                                   |                                   |
|                                   | Organisation is in another        |
|                                   | application:                      |
|                                   |                                   |
|                                   | If other application is a         |
|                                   | duplicate:                        |
|                                   |                                   |
|                                   | -   copy this application’s       |
|                                   |     application ID                |
|                                   | -   find and open the other       |
|                                   |     application                   |
|                                   | -   copy, paste and update the    |
|                                   |     following text into the       |
|                                   |     Comments box:                 |
|                                   |     ‘Duplicate application with   |
|                                   |     <other application ID>’       |
|                                   | -   reject other application,     |
|                                   |     select Reject                 |
|                                   | -   continue processing this      |
|                                   |     application, go to Step 8     |
|                                   |                                   |
|                                   | If other application is not a     |
|                                   | duplicate and the Responsible     |
|                                   | Officer (RO) is registering a     |
|                                   | different organisation:           |
|                                   |                                   |
|                                   | -   override errors, select       |
|                                   |     Override tick box that        |
|                                   |     relates to each individual    |
|                                   |     error, and                    |
|                                   | -   copy and paste the following  |
|                                   |     text into the Comments box:   |
|                                   |     ‘RO details checked. Overrode |
|                                   |     errors.’                      |
|                                   | -   go to Step 8                  |
|                                   |                                   |
|                                   | No organisation or duplicate      |
|                                   | application exists:               |
|                                   |                                   |
|                                   | -   override errors, select       |
|                                   |     Override tick box that        |
|                                   |     relates to each individual    |
|                                   |     error                         |
|                                   | -   copy and paste the following  |
|                                   |     text into the Comments box:   |
|                                   |     ‘OMO details checked.         |
|                                   |     Overrode errors’              |
|                                   | -   continue processing this      |
|                                   |     application                   |
+-----------------------------------+-----------------------------------+
| 8                                 | Responsible Officer tab + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If there are errors to process,   |
|                                   | cross check any duplicate details |
|                                   | using the second screen.          |
|                                   |                                   |
|                                   | Check if the Responsible Officer  |
|                                   | (RO) already exists in EHP or     |
|                                   | another application.              |
|                                   |                                   |
|                                   | RO in EHP:                        |
|                                   |                                   |
|                                   | -   Copy the Healthcare Provider  |
|                                   |     Identifier – Individual       |

|                                   |     dropdown list                 |

|                                   |     into the HPI/Admin number     |
|                                   |     field                         |
|                                   | -   Copy, paste and update the    |
|                                   |     following text into the       |
|                                   |     Comments box:                 |
|                                   |     ‘RO linked to existing record |
|                                   |     <HPI-I/Admin number>’         |
|                                   | -   Go to Step 9                  |
|                                   |                                   |
|                                   | If the error is not fixed when    |
|                                   | the HPI-I/Admin number has been   |

|                                   | ID to Provider HI Service and My  |

|                                   | Expert (SME) via email, and       |
|                                   | re-categorise the work item in    |
|                                   | WLM to Healthcare Identifier Tier |
|                                   | 3 Escalation                      |
|                                   |                                   |
|                                   | RO in another application:        |
|                                   |                                   |
|                                   | If other application is a         |

|                                   | -   copy this application’s       |
|                                   |     application ID                |
|                                   | -   find and open the other       |
|                                   |     application                   |
|                                   | -   copy, paste and update the    |
|                                   |     following text into the       |
|                                   |     Comments box:                 |
|                                   |     ‘Duplicate application with   |
|                                   |     <other application ID>’       |
|                                   | -   reject other application,     |

|                                   |                                   |
|                                   | If other application is not a     |

|                                   | -   override errors, select       |
|                                   |     Override tick box that        |
|                                   |     relates to each individual    |
|                                   |     error                         |
|                                   | -   copy and paste the following  |
|                                   |     text into the Comments box:   |
|                                   |     ‘RO details checked. Overrode |
|                                   |     errors.’                      |
|                                   | -   go to Step 9                  |
|                                   |                                   |
|                                   | No RO exists:                     |
|                                   |                                   |
|                                   | -   override errors               |
|                                   | -   copy and paste the following  |
|                                   |     text into the Comments box:   |
|                                   |     ‘RO details checked. Overrode |
|                                   |     errors.’                      |
|                                   | -   continue processing this      |

Re-open and complete Documents tab – Authority to Act

Table 2: this table tells Service Officers how to re-open an application
and complete the Documents tab where an Authority to Act has been
uploaded by the applicant.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Re-open application + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   Open Healthcare Identifier    |
|                                   |     Tier 3 Escalation in WLM      |
|                                   |     (Workload Management) and     |
|                                   |     select the appropriate        |
|                                   |     application                   |
|                                   | -   Follow Steps 1 to 2 in        |
|                                   |     Process online registration   |
|                                   |     of HPI-O records table        |
|                                   | -   Open Documents tab            |
+-----------------------------------+-----------------------------------+
| 2                                 | Has applicant uploaded the        |
|                                   | correct Authority to Act? + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Yes,                              |
|                                   |                                   |
|                                   | -   Select Resubmit               |
|                                   | -   An email will be sent to the  |
|                                   |     applicant’s Health            |
|                                   |     Professional Online Services  |
|                                   |     (HPOS) mail centre advising   |
|                                   |     them of the successful        |
|                                   |     outcome and HPI-O details     |
|                                   |                                   |
|                                   | No,                               |
|                                   |                                   |
|                                   | -   Copy and paste the following  |
|                                   |     text into the Comments box:   |
|                                   |     ‘Appropriate Authority to Act |
|                                   |     not provided by applicant     |
|                                   | -   Reject application, select    |
|                                   |     Application Invalid           |
|                                   | -   An email will be sent to the  |
|                                   |     applicant’s HPOS mail centre  |
|                                   |     advising them of the          |
|                                   |     unsuccessful outcome, and     |
|                                   |     that they will need to        |
|                                   |     complete the application      |
|                                   |     again if they still with to   |
|                                   |     register                      |

Error codes and actions to escalate to the Provider HI Service and My Health Record SME

Table 3: this table outlines the actions Service Officers need to take
when presented with error codes.

Note: once comments have been copied into Comments box return to Step 7
of Process online registration of HPI-O records table.

+-----------------------------------+-----------------------------------+
| Item                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Organisation name may be invalid  |
|                                   | or truncated + Read more ...      |
|                                   |                                   |
|                                   | Check documents to make sure      |
|                                   | organisation name is correct.     |
|                                   |                                   |
|                                   | If name is:                       |
|                                   |                                   |
|                                   | -   correct or truncated,         |
|                                   |     override                      |
|                                   | -   invalid, escalate to the      |
|                                   |     Provider HI Service and My    |
|                                   |     Health Record Subject Matter  |
|                                   |     Expert (SME)                  |
|                                   |                                   |
|                                   | Text for Comments box – override  |
|                                   | only:                             |
|                                   |                                   |
|                                   | -   ‘Organisation name checked.   |
|                                   |     Correct or appropriately      |
|                                   |     truncated’                    |
+-----------------------------------+-----------------------------------+
| 2                                 | Other errors + Read more ...      |
|                                   |                                   |
|                                   | If the following errors occur,    |
|                                   | escalate the application ID to    |
|                                   | the Provider HI Service and My    |
|                                   | Health Record SME via email, and  |
|                                   | re-categorise the work item in    |
|                                   | WLM to Healthcare Identifier Tier |
|                                   | 3 Escalation.                     |
|                                   |                                   |
|                                   | -   ABN is considered restricted  |
|                                   | -   ACN is considered restricted  |
|                                   | -   Organisation type is          |
|                                   |     considered restricted         |
