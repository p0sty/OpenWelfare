This page contains information regarding raising Pension Bonus Scheme
(PBS) debts.

Assessing Pension Bonus Scheme (PBS) debts

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Excess payment under the PBS +    |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Has the customer received an      |
|                                   | excess payment under the PBS for  |
|                                   | a payment made after 1 January    |
|                                   | 2008?                             |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, go to Step 3              |
+-----------------------------------+-----------------------------------+
| 2                                 | Excess Pension Bonus Scheme       |
|                                   | Payments made after 1 January     |
|                                   | 2008 + Read more ...              |
|                                   |                                   |
|                                   | Code the Modify/Assess Debt       |
|                                   | (OPMA) screen with PDB (Pension   |
|                                   | Bonus Payment) in the Benefit     |
|                                   | field.                            |
|                                   |                                   |
|                                   | The payment component code is PBP |
|                                   | on the OPCA (Component Amount)    |
|                                   | screen.                           |
+-----------------------------------+-----------------------------------+
| 3                                 | Excess PBS Payments made prior to |
|                                   | 1 January 2008 + Read more ...    |
|                                   |                                   |
|                                   | Code the Modify/Assess Debt       |
|                                   | (OPMA) screen with AGE in the     |
|                                   | Benefit Type field.               |
|                                   |                                   |
|                                   | The payment component code is PBP |
|                                   | on the OPCA screen.               |
+-----------------------------------+-----------------------------------+
| 4                                 | Appropriate action + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | If the excess payment is:         |
|                                   |                                   |
|                                   | due to administrative error, see  |
|                                   | Administrative error waivers      |
|                                   |                                   |
|                                   | due to fraud or                   |
|                                   | misrepresentation, these          |
|                                   | overpayments are recoverable. For |
|                                   | more information see Actioning an |
|                                   | undetermined debt on the Debt     |
|                                   | Management and Information System |
|                                   | (DMIS)                            |
|                                   |                                   |
|                                   | found in hindsight, to be excess: |
|                                   |                                   |
|                                   | -   finalise, no debt. See        |
|                                   |     Finalised no debt (FND) or    |
|                                   |     'zeroing' debts that are not  |
|                                   |     legally recoverable           |
|                                   | -   on the OPMA screen, key EXP   |
|                                   |     (Excess payment) in the FND   |
|                                   |     Reason field                  |
+-----------------------------------+-----------------------------------+
