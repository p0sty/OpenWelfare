|                                   |                                   |

| 4                                 | Changing undetermined debts +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Before proceeding with any        |
|                                   | changes, see Safeguarding         |

|                                   |                                   |
|                                   | Change an undetermined debt via   |
|                                   | the Modify/Assess Debt (OPMA)     |

|                                   |                                   |

|                                   | Customer Record to change the     |

|                                   | STA (started) activity from the   |

|                                   |                                   |

|                                   | screen. Key the following fields, |
|                                   | where appropriate:                |
|                                   |                                   |
|                                   | -   Authority field               |
|                                   | -   Estimated Amount $ field      |
|                                   | -   Benefit Type field            |
|                                   | -   Reason field                  |
|                                   | -   Section of Act field          |
|                                   | -   Date of Event field           |
|                                   | -   Date of Notification field    |
|                                   | -   Start Date field              |
|                                   | -   End Date field                |
|                                   | -   Calculated Amount $ field     |
|                                   | -   Addtnl Reference No field     |
|                                   | -   NZ Office field               |
|                                   | -   Was debt solely due to system |
|                                   |     or clerical error? field      |
|                                   | -   Position Responsible field    |
|                                   | -   Refer Prosecution field       |
|                                   | -   Acknowledgment Date field     |
|                                   |                                   |
|                                   | Key a '?' to see an explanation   |
|                                   | of how to fill in the fields      |
|                                   |                                   |
|                                   | Press [Enter]                     |
|                                   |                                   |
|                                   | On the Component Amount (OPCA)    |
|                                   | screen:                           |
|                                   |                                   |
|                                   | -   key the calculated amount for |
|                                   |     each component type in the    |
|                                   |     Calc Amt $ column             |
|                                   | -   if the calculated amount is   |
|                                   |     equal to the estimated amount |
|                                   |     and no change is required the |
|                                   |     OPCA screen will not display  |
|                                   | -   the Est Amt $ column will     |
|                                   |     complete for each component   |
|                                   |     if the debt comes from a host |
|                                   |     system reassessment           |
|                                   | -   the estimated component       |
|                                   |     amount(s) default in the Est  |
|                                   |     Amt $ and the Calc Amt $      |
|                                   |     fields for each component     |
|                                   | -   the sum of all components     |
|                                   |     must add up to the debt total |
|                                   |     amount                        |
|                                   |                                   |
|                                   | Press [Enter]                     |
|                                   |                                   |
|                                   | On the Account Payable (OPAOD)    |
|                                   | screen:                           |
|                                   |                                   |
|                                   | -   text details previously keyed |
|                                   |     can be changed if needed      |
|                                   | -   if the Account Payable is     |
|                                   |     suppressed, key manual advice |
|                                   |     details on the Key Dates      |
|                                   |     (OPKD) screen                 |
|                                   |                                   |
|                                   | To finalise the account on the    |
|                                   | Assessment Finalisation (OPAF)    |
|                                   | screen and to notify the customer |

|                                   | Account Payable letters           |
|                                   |                                   |
|                                   | Record details on a DOC           |
|                                   |                                   |
|                                   | Procedure ends here.              |
