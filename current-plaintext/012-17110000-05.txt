This page contains information on file fix requests, through the DVA
Maintenance System (DVAMS) and manually, system limitations, and file
fix reason codes and request field requirements. It also contains the
file fix request forms. Links on this page may be unavailable to
external audiences.

File fix process through the DVA Maintenance System

Table 1

+-----------------------+-----------------------+-----------------------+
| Stage                 | Who                   | Description           |
+-----------------------+-----------------------+-----------------------+
| 1                     | Requestor             | Submits the file fix  |
|                       |                       | request through the   |
|                       |                       | DVA Maintenance       |
|                       |                       | System.               |
+-----------------------+-----------------------+-----------------------+
| 2                     | Approver              | Reviews the request   |
|                       |                       | and either approves   |
|                       |                       | or rejects it, or     |
|                       |                       | changes the status to |
|                       |                       | Pending if the        |
|                       |                       | request needs to be   |
|                       |                       | edited.               |
+-----------------------+-----------------------+-----------------------+
| 3                     | System                | Collates and runs     |
|                       |                       | programme twice       |
|                       |                       | weekly – Wednesday    |
|                       |                       | night and over the    |
|                       |                       | weekend.              |
+-----------------------+-----------------------+-----------------------+
| 4                     | Requestor             | Checks DVAMS twice a  |
|                       |                       | week to ensure the    |
|                       |                       | file fix is           |
|                       |                       | successful. Close     |
|                       |                       | actioned file fixes.  |
+-----------------------+-----------------------+-----------------------+

System limitations for all claims

Includes information for when file fixes cannot be requested for all
claim modalities (Allied, Medical, Hospital, Manual and Online).

Table 3

+-----------------------------------+-----------------------------------+
| Type of file fix                  | Limitations for all claims        |
+-----------------------------------+-----------------------------------+
| Incorrect provider                | Provider can only be changed at   |
|                                   | the whole patient level and must  |
|                                   | be for all lines claimed.         |
+-----------------------------------+-----------------------------------+
| History - Rejected lines          | A rejected line cannot be changed |
|                                   | to a paid line if there are other |
|                                   | lines on that voucher.            |
|                                   |                                   |
|                                   | Note: to determine whether lines  |
|                                   | are on the same voucher, press    |
|                                   | Shift+F2 on the VHOI screen. This |
|                                   | will display the voucher number   |
|                                   | for each line.                    |
+-----------------------------------+-----------------------------------+

Technical limitations for paperless online claims

There are limitations to the information that can be file fixed for DVA
online claims.

Table 4

+-----------------------------------+-----------------------------------+
| Type of file fix                  | Limitations specific to paperless |
|                                   | claims                            |
+-----------------------------------+-----------------------------------+
| Incorrect provider                | -   Payee provider cannot be      |
|                                   |     changed. This applies when    |
|                                   |     the claim has a separate      |
|                                   |     payee provider or a servicing |
|                                   |     provider only                 |
|                                   | -   If the claim has both a payee |
|                                   |     provider and a servicing      |
|                                   |     provider, then the servicing  |
|                                   |     provider can be changed       |
|                                   | -   Where the claim has only a    |
|                                   |     servicing provider and that   |
|                                   |     provider should have been the |
|                                   |     payee provider, the servicing |
|                                   |     provider can be changed to be |
|                                   |     the payee provider and a new  |
|                                   |     servicing provider can be     |
|                                   |     added                         |
|                                   | -   Changes can only be made to   |
|                                   |     the whole claim, not          |
|                                   |     individual lines              |
+-----------------------------------+-----------------------------------+
| Patient                           | -   Can only move all of the      |
|                                   |     lines for one patient to      |
|                                   |     another single patient. It is |
|                                   |     not possible to move some of  |
|                                   |     the lines from one patient to |
|                                   |     another or to move all lines  |
|                                   |     to multiple patients          |
+-----------------------------------+-----------------------------------+

DVAMS file fix request field requirements

Not all fields may be relevant for all requests.

Table 6

+-----------------------------------+-----------------------------------+
| Field                             | Requirement                       |
+-----------------------------------+-----------------------------------+
| Claim ID                          | Claim ID of the claim being fixed |
|                                   | including suffixes such as @.     |
+-----------------------------------+-----------------------------------+
| Date of Lodgement                 | Date of lodgement for the claim   |
|                                   | being file fixed as DDMMYYYY.     |
+-----------------------------------+-----------------------------------+
| Payee provider                    | Payee provider number for claim   |
|                                   | being file fixed.                 |
+-----------------------------------+-----------------------------------+
| Claim type                        | Type of claim being file fixed:   |
|                                   | Allied Health (GST items), Allied |
|                                   | Health (Non-GST items), Hospital, |
|                                   | or Medical.                       |
+-----------------------------------+-----------------------------------+
| Change as a result of             | Select an option from this        |
|                                   | drop-down box if the file fix is  |
|                                   | a result of an overpayment or     |
|                                   | underpayment, otherwise leave     |
|                                   | blank.                            |
+-----------------------------------+-----------------------------------+
| Amount                            | Amount of the overpayment or      |
|                                   | underpayment. Do not include the  |
|                                   | $ sign.                           |
+-----------------------------------+-----------------------------------+
| Recovery status                   | Recovery status of the            |
|                                   | overpayment for the claim being   |
|                                   | file fixed.                       |
+-----------------------------------+-----------------------------------+
| Payment reference                 | Recovery reference number for the |
|                                   | claim being file fixed, obtained  |
|                                   | from SAP GUI, or short            |
|                                   | description i.e. Held payment,    |
|                                   | ADD GST etc.                      |
+-----------------------------------+-----------------------------------+
| Request reason                    | Select the file fix reason code   |
|                                   | applicable to the request. See    |
|                                   | the File fix reason code table.   |
+-----------------------------------+-----------------------------------+
| Add/Remove GST                    | Only used for Incorrect Provider  |
|                                   | when adding or removing the GST   |
|                                   | in the claim.                     |
+-----------------------------------+-----------------------------------+
| Professionally recognised         | Indicates that the service        |
|                                   | provider has the required         |
|                                   | speciality code to claim the      |
|                                   | service/s in the claim being file |
|                                   | fixed. Y (Yes) or N (No).         |
+-----------------------------------+-----------------------------------+
| GST registered                    | Indication on ABAA that the payee |
|                                   | provider is registered for GST. Y |
|                                   | (Yes) or N (No).                  |
+-----------------------------------+-----------------------------------+
| RCTI or ABN repository            | Indication on ABAA that RCTI      |
|                                   | Agreement has been signed. Y      |
|                                   | (Yes) or N (No).                  |
+-----------------------------------+-----------------------------------+
| ABN                               | Australian Business Number (ABN)  |
|                                   | the provider is linked to on      |
|                                   | ABAA. If the provider number is   |
|                                   | not linked to an ABAA on ABAA,    |
|                                   | leave this field blank.           |
|                                   |                                   |
|                                   | Note: the correct ABN details     |
|                                   | must be entered for all claims.   |
|                                   | If a provider number is linked to |
|                                   | an ABN on ABAA this number needs  |
|                                   | to appear on their statement.     |
+-----------------------------------+-----------------------------------+
| GST amount                        | Amount of GST to be paid or       |
|                                   | removed. Only used for Incorrect  |
|                                   | Provider, Add GST and Remove GST  |
|                                   | file fix requests.                |
+-----------------------------------+-----------------------------------+
| After                             | This field appears on the Amend   |
|                                   | Service screen in the History     |
|                                   | file fix request. Enter the       |
|                                   | details of the service line as    |
|                                   | they should appear after the file |
|                                   | fix.                              |
+-----------------------------------+-----------------------------------+
| Insert Summary                    | This field appears on the Insert  |
|                                   | Service screen in the History     |
|                                   | file fix request. Enter the       |
|                                   | details of the service line to be |
|                                   | inserted.                         |
