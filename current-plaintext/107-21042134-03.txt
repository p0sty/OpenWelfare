Manual Debt Advice Notice (DAN)

Table 1

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Manual Debt Advice Notice (DAN) + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | A DAN is sent from each program   |
|                                   | area to Medicare Recovery and     |
|                                   | confirms the reason for the debt. |
|                                   | A valid debt notification must    |
|                                   | have three documents:             |
|                                   |                                   |
|                                   | -   a DAN (amount must match with |
|                                   |     the overpayment letter and    |
|                                   |     duplicate statement)          |
|                                   | -   an overpayment letter, and    |
|                                   | -   a duplicate provider          |
|                                   |     statement                     |
|                                   |                                   |
|                                   | Are all the required documents    |
|                                   | available?                        |
|                                   |                                   |
|                                   | -   Yes, go to step 2             |
|                                   | -   No, contact the program area  |
|                                   |     by email to request the       |
|                                   |     missing documents             |
|                                   |                                   |
|                                   | The Resources page has an example |
|                                   | of a DAN form and links to the    |
|                                   | Medicare program areas.           |
+-----------------------------------+-----------------------------------+
| 2                                 | Check the SAP database + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Check if the customer already     |
|                                   | exists in the SAP database. Log   |
|                                   | into SAP:                         |
|                                   |                                   |
|                                   | -   Run Transaction FBL5N -       |
|                                   |     Customer Line Item Display    |
|                                   | -   Key the customer number and   |
|                                   |     select Search                 |
|                                   |                                   |
|                                   | Does the customer's record        |
|                                   | already exist in SAP?             |
|                                   |                                   |
|                                   | -   Yes: go to Step 3             |
|                                   | -   No: create the customer       |
|                                   |     record. See Table 1 in SAP    |
|                                   |     Invoices and Customer         |
|                                   |     Navigation. Procedure ends    |
|                                   |     here.                         |
+-----------------------------------+-----------------------------------+
| 3                                 | Create tax invoice and issue to   |
|                                   | customer + Read more ...          |
|                                   |                                   |
|                                   | Do not create a duplicate         |
|                                   | invoice. Check the customer       |
|                                   | record before proceeding.         |
|                                   |                                   |
|                                   | Log into SAP:                     |
|                                   |                                   |
|                                   | Run Transaction FBL5N - Customer  |
|                                   | Line Item Display                 |
|                                   |                                   |
|                                   | In Customer account, key the      |
|                                   | customer number and search ALL    |
|                                   | ITEMS, including:                 |
|                                   |                                   |
|                                   | -   DOS                           |
|                                   | -   claim ID, and                 |
|                                   | -   Medicare card details         |
|                                   |                                   |
|                                   | Verify the information if         |
|                                   | provided within 2 years.          |
|                                   |                                   |
|                                   | Are the details the same?         |
|                                   |                                   |
|                                   | Yes, send an email to Medicare    |
|                                   | Provider Services to check with   |
|                                   | the program area. Go to Step 4    |
|                                   |                                   |
|                                   | No,                               |
|                                   |                                   |
|                                   | -   create the invoice, see Table |
|                                   |     1 in SAP invoices and         |
|                                   |     customer navigation           |
|                                   | -   send the invoice to the       |
|                                   |     customer by post, unless      |
|                                   |     instruction in the DAN        |
|                                   |     advises otherwise             |
|                                   | -   go to Step 4                  |
+-----------------------------------+-----------------------------------+
| 4                                 | Follow up recovery of debts when  |
|                                   | due - 30 days + Read more ...     |
|                                   |                                   |
|                                   | Follow up includes:               |
|                                   |                                   |
|                                   | -   sending reminder letters. See |
|                                   |     Dunning for Medicare debt for |
|                                   |     the follow up process         |
|                                   | -   issuing a Statement of        |
|                                   |     accounts – Medicare           |
|                                   |     Overpayments (MCO) in SAP.    |
|                                   |     See Table 5 in SAP invoices   |
|                                   |     and customer navigation       |
|                                   | -   starting the Active recovery  |
|                                   |     process after Dunning level 3 |
|                                   |     or 7 day reminder for         |
|                                   |     compensation                  |
|                                   | -   contacting the customer       |
+-----------------------------------+-----------------------------------+
| 5                                 | Update SAP with Notes of the      |
|                                   | customer conversation + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Give a brief summary of the       |
|                                   | interaction and any follow up     |
|                                   | actions.                          |
|                                   |                                   |
|                                   | In SAP, run Transaction FBL5N -   |
|                                   | Customer Line Item Display:       |
|                                   |                                   |
|                                   | -   Update the Assignment field   |
|                                   |     to reflect the debt status    |
|                                   | -   In the Notes field, include   |
|                                   |     successful and unsuccessful   |
|                                   |     contacts with the customer    |
|                                   |     and why                       |
|                                   |                                   |
|                                   | When recording information and    |
|                                   | changing assignment fields after  |
|                                   | each contact. See the Resources   |
|                                   | page for the Assignment field     |
|                                   | values table.                     |
+-----------------------------------+-----------------------------------+
| 6                                 | EFT or cheque payments made by    |
|                                   | customer + Read more ...          |
|                                   |                                   |
|                                   | Clearing action is required to    |
|                                   | allocate payments to the correct  |
|                                   | tax invoice.                      |
|                                   |                                   |
|                                   | -   Run Transaction FB1D – clear  |
|                                   |     customer                      |
|                                   | -   account field, key the        |
|                                   |     customer number               |
|                                   | -   Select Enter                  |
|                                   | -   Select the payment and the    |
|                                   |     correct invoice by double     |
|                                   |     clicking on the amounts to    |
|                                   |     clear and post. This must     |
|                                   |     show zero on Not Assigned     |
