notice received via Unified Government Gateway (UGG) and booking an

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Income Management Contact Officer |
|                                   | (IMCO) receives Notice to         |
|                                   | Commence Income Management via    |
|                                   | the Unified Government Gateway    |
|                                   | (UGG) + Read more ...             |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | The IMCO can view all requests    |
|                                   | sent by a State or Territory      |
|                                   | Child Protection Authority by     |
|                                   | viewing the Third Party           |
|                                   | Information Summary Organisation  |
|                                   | (TPISO) screen in Environment B.  |
|                                   |                                   |
|                                   | Key 'TPISO' in next field.        |
|                                   |                                   |
|                                   | Complete the below fields:        |
|                                   |                                   |
|                                   | -   Application Type, select      |
|                                   |     'Welfare Payment Reform'      |
|                                   | -   Business Function, select     |
|                                   |     'Child Protection'            |
|                                   | -   Information Type, select      |
|                                   |     'Child Protection Send Notice |
|                                   |     for Income Management'        |

|                                   |     (the IMCO should be checking  |
|                                   |     this on a regular basis       |
|                                   |     throughout each working day)  |
|                                   | -   Status, for the purposes of   |

|                                   |     be searched                   |
|                                   | -   Select 'Search'               |
|                                   |                                   |
|                                   | This will display any unprocessed |
|                                   | notices that have been received.  |
|                                   |                                   |
|                                   | Review the details advised in the |
|                                   | notice, taking note of relevant   |
|                                   | information such as the duration  |
|                                   | to be income managed and the      |
|                                   | priority needs as ranked by the   |
|                                   | case worker.                      |
|                                   |                                   |
|                                   | Note: the system will attempt to  |
|                                   | auto-update any new Notice to     |
|                                   | Commence Income Management,       |
|                                   | change the status to pending and  |
|                                   | record an inactive notice on the  |
|                                   | customer's record. When automated |
|                                   | processing has occurred, no       |
|                                   | further coding of the notice is   |
|                                   | required by the IMCO. IMCOs can   |
|                                   | view the status of automated      |
|                                   | processing by viewing the         |

|                                   | pending).                         |
|                                   |                                   |
|                                   | -   If customer already has an    |
|                                   |     Active Notice on their        |
|                                   |     record, see Step 1 in Table 2 |
|                                   | -   Otherwise, go to Step 2       |

|                                   | requirements for Child Protection |
|                                   | Income Management + Read more ... |
|                                   |                                   |
|                                   | []                                |

|                                   | (See Legislation page Section     |
|                                   | 123UC of the Social Security      |
|                                   | (Administration) Act 1999). It is |
|                                   | important to thoroughly check the |
|                                   | eligibility as the customer's     |
|                                   | circumstances may have changed    |
|                                   | since the case worker performed   |
|                                   | the eligibility check and sent    |
|                                   | the Notice to Commence Income     |
|                                   | Management.                       |
|                                   |                                   |

|                                   | requirements for Child Protection |
|                                   | Income Management, the notice     |

|                                   | customer's record as it will be   |
|                                   | held for the period of the        |
|                                   | Notice. If in that time the       |
|                                   | customer meets the requirements,  |
|                                   | the notice will become Active and |
|                                   | Income Management will commence.  |
|                                   | If the notice expires without     |
|                                   | having become active, the notice  |
|                                   | will auto-convert to lapsed.      |
|                                   |                                   |
|                                   | If automated processing of the    |
|                                   | notice has not occurred, the IMCO |
|                                   | is required to code the Notice in |
|                                   | Centrelink system.                |
|                                   |                                   |
|                                   | Go to Step 3.                     |

|                                   | -   Status: Select Active or      |
|                                   |     Inactive as appropriate from  |
|                                   |     the drop down menu            |
|                                   | -   Start Date: will default and  |
|                                   |     is protected, do not change   |
|                                   | -   End date: will be calculated  |
|                                   |     from period of notice (field  |
|                                   |     is protected and cannot be    |
|                                   |     changed) and only displays    |
|                                   |     when the Notice status is     |
|                                   |     Active                        |
|                                   |                                   |
|                                   | Referring Organisation Details:   |
|                                   |                                   |
|                                   | -   State/Territory: Select the   |
|                                   |     applicable state/territory    |
|                                   |     from the drop down menu       |
|                                   | -   Organisation/Office: A list   |
|                                   |     of organisations/offices will |
|                                   |     display based on the          |
|                                   |     state/territory selected.     |
|                                   |     Select appropriate referring  |
|                                   |     authority from the drop down  |
|                                   |     menu                          |
|                                   | -   Contact: A list of contacts   |
|                                   |     will display based on the     |
|                                   |     Organisation/Office selected. |

|                                   |     case worker in the Notice     |
|                                   |     received                      |

|                                   | Select 'Continue'.                |

|                                   | more ...                          |

|                                   | with any information required     |
|                                   | under the guidelines for adding a |

|                                   |                                   |
|                                   | Note: If the case worker provided |
|                                   | information in the notice that    |

|                                   |                                   |
|                                   | Is the customer eligible to be    |

|                                   | -   Yes, go to Step 6             |
|                                   | -   No, see Step 9 in Table 2     |
+-----------------------------------+-----------------------------------+

|                                   | that is serviced by the Remote    |
|                                   | Servicing Team (RST)? + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | []                                |

|                                   | -   Yes, go to Step 7             |

Continuation Notice or Notice from a new Authority

Table 2: This table describes how to code a new or continuation notice
and finalise an assessment.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Continuation of a Notice + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | A State or Territory Child        |
|                                   | Protection Authority may lodge a  |
|                                   | new Notice to Commence Income     |
|                                   | Management to extend the period   |
|                                   | of Income Management beyond the   |
|                                   | current notice expiry date, or a  |
|                                   | notice may be received from a     |
|                                   | different Child Protection        |
|                                   | Authority.                        |
|                                   |                                   |
|                                   | Is the notice from the same Child |
|                                   | Protection Authority who lodged   |
|                                   | the current active notice?        |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, go to Step 3              |
+-----------------------------------+-----------------------------------+
| 2                                 | Automatic Continuation notice +   |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | A Continuation notice will        |
|                                   | auto-create when a customer with  |
|                                   | an active notice receives a       |
|                                   | continuation notification within  |
|                                   | 60 days of the current notice's   |
|                                   | end date.                         |
|                                   |                                   |
|                                   | The notice will have an inactive  |
|                                   | status as there can only be one   |
|                                   | active notice on a Child          |
|                                   | Protection customer's record at   |
|                                   | any time.                         |
|                                   |                                   |
|                                   | When the customer's current       |
|                                   | notice expires, the Continuation  |
|                                   | notice will automatically become  |
|                                   | active and Income Management will |
|                                   | continue uninterrupted.           |
|                                   |                                   |
|                                   | A message will display under      |
|                                   | Description on the TPISO screen   |
|                                   | indicating the results of the     |
|                                   | auto-processing, including where  |
|                                   | the automated process was         |
|                                   | unsuccessful in which case manual |
|                                   | processing is required.           |
|                                   |                                   |
|                                   | Was the automated processing of   |
|                                   | the continuation notice           |
|                                   | successful?                       |
|                                   |                                   |
|                                   | -   Yes, book an appointment for  |
|                                   |     the customer to discuss the   |
|                                   |     extension of Income           |
|                                   |     Management and review their   |
|                                   |     circumstances. Procedure ends |
|                                   |     here                          |
|                                   | -   No, manually record the       |
|                                   |     continuation notice, go to    |
|                                   |     Step 4                        |
+-----------------------------------+-----------------------------------+
| 3                                 | Notice from a new Child           |
|                                   | Protection Authority + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | If a new Notice to Commence       |
|                                   | Income Management is received     |
|                                   | from a different Child Protection |
|                                   | Authority than the existing       |
|                                   | active notice, an inactive notice |
|                                   | will auto-create on the           |
|                                   | customer's record (subject to     |
|                                   | usual business rules being        |
|                                   | satisfied).                       |
|                                   |                                   |
|                                   | A message will display under      |
|                                   | Description on the TPISO screen   |
|                                   | indicating the results of the     |
|                                   | automated processing, including   |
|                                   | where the automated process was   |
|                                   | unsuccessful and manual           |
|                                   | processing is required.           |
|                                   |                                   |
|                                   | Note: if the notice includes an   |
|                                   | Unrestricted Cash Payment (UCP)   |
|                                   | service offer, the customer will  |
|                                   | be identified as eligible for the |
|                                   | service offer in the automated    |
|                                   | process.                          |
|                                   |                                   |
|                                   | If the automated process is not   |
|                                   | successful IMCOs must manually    |
|                                   | create an inactive notice on the  |
|                                   | customer's record using the Child |
|                                   | Protection Assessment workflow.   |
|                                   |                                   |
|                                   | Was the automated processing of   |
|                                   | the new authority's notice        |
|                                   | successful?                       |
|                                   |                                   |
|                                   | -   Yes, the customer will be     |
|                                   |     sent a letter advising Income |
|                                   |     Management will be continuing |
|                                   |     under a new authority and     |
|                                   |     they will need to contact     |
|                                   |     Services Australia to arrange |
|                                   |     an interview. Procedure ends  |
|                                   |     here                          |
|                                   | -   No, manually record the       |
|                                   |     notice, go to Step 4          |
