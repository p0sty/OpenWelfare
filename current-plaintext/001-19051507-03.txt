|                                   |                                   |
|                                   | Calculate the start date for      |
|                                   | payment                           |
|                                   |                                   |
|                                   | The system automatically          |
|                                   | calculates a start date for       |
|                                   | eligible claims. It is important  |
|                                   | to confirm this start date is     |
|                                   | correct because system            |
|                                   | calculations are not always       |
|                                   | correct.                          |
|                                   |                                   |
|                                   | For example, a customer could     |
|                                   | reclaim a payment after having:   |
|                                   |                                   |
|                                   | -   a backdated cancellation of   |
|                                   |     the payment, and              |
|                                   | -   a debt raised for the period  |
|                                   |     they were not eligible        |
|                                   |                                   |
|                                   | The system will apply the         |
|                                   | backdating rule correctly and     |
|                                   | calculate the date of             |
|                                   | commencement as Date Paid To      |
|                                   | (DPT)+1, even if the date of the  |
|                                   | claim is within 14 days of DPT.   |
|                                   |                                   |
|                                   | Check the cancellation date and,  |
|                                   | if it is within 14 days of the    |
|                                   | claim date, do not apply DPT+1.   |
|                                   | The start date is the date the    |
|                                   | customer reclaims payment.        |
|                                   |                                   |
|                                   | If the system calculates the      |
|                                   | start date incorrectly and the    |
|                                   | correct date should be:           |
|                                   |                                   |
|                                   | after the DPT+1, code the correct |
|                                   | start date in the Manual Start    |
|                                   | Date field on NDC to override the |
|                                   | system calculated start date      |
|                                   |                                   |
|                                   | before the DPT+1 (which is very   |
|                                   | rare), check:                     |
|                                   |                                   |
|                                   | -   the customer is not being     |
|                                   |     paid again for a period they  |
|                                   |     were previously receiving     |
|                                   |     payment                       |
|                                   | -   if the customer was cancelled |
|                                   |     off their previous payment    |
|                                   |     and if a debt was raised for  |
|                                   |     the period they were not      |
|                                   |     eligible                      |
|                                   |                                   |
|                                   | If the date is being changed to a |
|                                   | date before DPT+1, do not code it |
|                                   | in the Manual Start Date field on |
|                                   | NDC. The start date will overlap  |
|                                   | with the previous payment dates,  |
|                                   | which may:                        |
|                                   |                                   |
|                                   | -   prevent the claim from        |
|                                   |     granting properly, or         |
|                                   | -   lead to a data corruption on  |
|                                   |     the record                    |
|                                   |                                   |
|                                   | To ensure the customer receives   |
|                                   | the correct payment:              |
|                                   |                                   |
|                                   | -   pay the days the customer was |
|                                   |     entitled to prior to the      |
|                                   |     DPT+1, as a one-off payment   |
|                                   | -   code the DPT+1 date in the    |
|                                   |     Manual Start Date field on    |
|                                   |     NDC, and                      |
|                                   | -   record a DOC detailing the    |
|                                   |     steps taken and the reason    |
|                                   |     for the start date            |
|                                   |     determination                 |
|                                   |                                   |
|                                   | Service Officer discretion is     |
|                                   | required when changing the start  |
|                                   | date from the one the system      |
|                                   | calculated.                       |
