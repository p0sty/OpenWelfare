This page contains the process a Compliance Officer (CO) must follow
when assessing and finalising a BIBI intervention within the Compliance
Assurance and Debt Operations Division.

How to determine, calculate and raise overpayments

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Determine whether a Recovery Fee  |
|                                   | is to be applied + Read more ...  |
|                                   |                                   |
|                                   | If the intervention includes      |
|                                   | verification of earned income     |
|                                   | (include income from              |
|                                   | self-employment), determine       |
|                                   | whether the Recovery Fee should   |
|                                   | be applied.                       |
|                                   |                                   |
|                                   | The recovery fee will only be     |
|                                   | applied if the customer has not   |
|                                   | met their obligations. For        |
|                                   | example, they failed to provide   |
|                                   | information relating to their     |
|                                   | income confirmation.              |
|                                   |                                   |
|                                   | The recovery fee will not be      |
|                                   | applied if the customer has       |
|                                   | fulfilled their obligations. For  |
|                                   | example, they provided            |
|                                   | information relating to their     |
|                                   | income confirmation.              |
|                                   |                                   |
|                                   | Note: the system will raise       |
|                                   | earnings debts with a period of   |
|                                   | 14 days or less upon completion   |
|                                   | of the earnings activity. The     |
|                                   | Recovery Fee is not applied to    |
|                                   | auto raised debts. The customer   |
|                                   | will receive the standard debt    |
|                                   | advice as part of the automatic   |
|                                   | process.                          |
+-----------------------------------+-----------------------------------+
| 2                                 | Determine debt and calculate +    |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | All debts that can be calculated  |
|                                   | via the Centrelink system and     |
|                                   | explained using ADEX should be    |

|                                   | MultiCal if the Centrelink system |
|                                   | calculation is not possible or    |
|                                   | the outcome is not correct.       |

|                                   | has correctly calculated any      |
|                                   | potential overpayment, for        |
|                                   | example:                          |

|                                   | dates/date of event (DOV) match   |
|                                   | the period?                       |
|                                   |                                   |
|                                   | Have the updates/coding been      |
|                                   | entered correctly?                |
|                                   |                                   |
|                                   | Check the following to make sure  |
|                                   | the updates are as expected:      |
|                                   |                                   |
|                                   | -   Working Credit Balance        |
|                                   |     Summary (WCBS)                |
|                                   | -   Income Bank (NIB)             |
|                                   | -   Employment Income Assessment  |
|                                   |     (EIA)                         |
|                                   |                                   |
|                                   | Can a reassessment be made using  |
|                                   | the Benefit Action (BA) screen?   |
|                                   |                                   |
|                                   | -   Use REA in relevant host      |
|                                   |     system, or                    |
|                                   | -   RCR in PEN/PGA (for cancelled |

|                                   | If an ADEX debt calculation:      |
|                                   |                                   |
|                                   | -   can be undertaken, the        |
|                                   |     overpayment must be           |

|                                   | -   is not possible a MultiCal    |
|                                   |     debt calculation may be       |
|                                   |     required                      |
|                                   |                                   |
|                                   | For MultiCal debt calculation,    |
|                                   | get all information from the      |
|                                   | customer over the phone where     |
|                                   | possible to enable a debt         |
|                                   | calculation.                      |
|                                   |                                   |
|                                   | The CO is responsible for         |
|                                   | progressing the MultiCal          |
|                                   | activity. If they cannot action,  |
|                                   | their site is responsible for     |
|                                   | progressing the activity. The     |
|                                   | Team Leader will need to reassign |
|                                   | the activity to another CO with   |
|                                   | the required skill set.           |
|                                   |                                   |
|                                   | For more information, see:        |
|                                   |                                   |
|                                   | -   Debt tools                    |
|                                   | -   Debt investigation            |
|                                   | -   Actioning an undetermined     |
|                                   |     debt on the Debt Management   |
|                                   |     and Information System (DMIS) |
|                                   | -   Using ADEX to explain a       |
|                                   |     system generated debt         |
|                                   | -   Using the correct date of     |
|                                   |     effect when raising debts     |

|                                   |     investigating debts           |
|                                   | -   Waiving Centrelink debts      |
