Mandatory detailed search in CDMS

Table 1: how to perform a detailed search for a customer where no
existing or previous Medicare number is available. This is mandatory
before creating a new Medicare registration.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Search for customer in CDMS +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | On the CDMS Home page:            |
|                                   |                                   |
|                                   | -   select Inquire on Consumer    |
|                                   |     Details                       |
|                                   | -   select the Detailed Search    |
|                                   |     tab                           |
+-----------------------------------+-----------------------------------+
| 2                                 | Enter search details + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | Enter search details as follows:  |
|                                   |                                   |
|                                   | -   Select All Programs from the  |
|                                   |     Program dropdown menu         |
|                                   |     (mandatory)                   |
|                                   | -   Key Family Name (mandatory)   |
|                                   | -   Select All from the Gender    |
|                                   |     field (mandatory)             |
|                                   | -   Key Age or Age Range          |
|                                   |     (mandatory)                   |
|                                   | -   Leave the Postcode or         |
|                                   |     Postcode range field as       |
|                                   |     default                       |
|                                   | -   Select Personal tab           |
|                                   |                                   |
|                                   | Note: staff must search under all |
|                                   | programs, as Individual           |
|                                   | Healthcare Identifiers (IHI) are  |
|                                   | not picked under the ‘Medicare    |
|                                   | Program (default)’ search.        |
|                                   |                                   |
|                                   | If information message 00005      |
|                                   | returns, follow the instructions  |
|                                   | in the message to refine the      |
|                                   | search or adjust search criteria  |
|                                   | by including:                     |
|                                   |                                   |
|                                   | the first initial, or             |
|                                   |                                   |
|                                   | for newborn enrolments:           |
|                                   |                                   |
|                                   | -   check other parent surname,   |
|                                   |     or                            |
|                                   | -   key 'baby of' or ‘Newborn’ in |
|                                   |     the First Name field          |
+-----------------------------------+-----------------------------------+
| 3                                 | Define search result + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | Services Officers must do extra   |
|                                   | checks to confirm no other        |
|                                   | consumer records have been        |
|                                   | previously created in CDMS. For   |
|                                   | example:                          |
|                                   |                                   |
|                                   | -   Does the name have any        |
|                                   |     hyphens or spaces? When       |
|                                   |     searching for a name with a   |
|                                   |     hyphen, do the search without |
|                                   |     the hyphen, for example.      |
|                                   |     Mary-Ann and Mary Ann         |
|                                   | -   Confirm the spelling of the   |
|                                   |     name/s                        |
|                                   | -   Did the customer have another |
|                                   |     surname as a child?           |
|                                   | -   Check for other variations of |
|                                   |     first name, for example, if a |
|                                   |     customer’s name is Oliver     |
|                                   |     also search Ollie             |
|                                   | -   Has the customer ever used or |
|                                   |     been known by any other name? |
|                                   | -   Family and given names may    |
|                                   |     have been transposed in the   |
|                                   |     past                          |
|                                   | -   Consider ethnic name          |
|                                   |     practices                     |
|                                   | -   Check other parent's surname  |
|                                   |                                   |
|                                   | Possible matches are displayed    |
|                                   | under Matched Consumers.          |
|                                   |                                   |
|                                   | Is a possible match been found    |
|                                   | after all checks?                 |
|                                   |                                   |
|                                   | -   Yes, go to Step 4             |
|                                   | -   No, see Step 1 in Table 6 to  |
|                                   |     do an address search, then go |
|                                   |     to Table 2                    |
|                                   |                                   |
|                                   | Note: make sure the correct       |
|                                   | record has been identified by     |
|                                   | confirming details such as legal  |
|                                   | name and date of birth before     |
|                                   | proceeding with the enrolment.    |
+-----------------------------------+-----------------------------------+
| 4                                 | Possible matches found + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Is there more than one match?     |
|                                   |                                   |
|                                   | Yes, go to Step 5                 |
|                                   |                                   |
|                                   | No, select the consumer:          |
|                                   |                                   |
|                                   | -   select radio button next to   |
|                                   |     the consumer                  |
|                                   | -   select Personal tab           |
|                                   |                                   |
|                                   | The Personal Details Inquiry      |
|                                   | screen displays.                  |
+-----------------------------------+-----------------------------------+
| 5                                 | More than one match found + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | -   Where the records show both   |
|                                   |     an IHI and an AIR SIN, select |
|                                   |     the ACTIVE IHI Record and     |
|                                   |     continue with the enrolment   |
|                                   | -   Once the enrolment has been   |
|                                   |     completed, see Duplicate PIN  |
|                                   |     and Consumer IDs in Medicare  |
|                                   |     to report potential           |
|                                   |     duplicates                    |

documents for enrolment purposes. This process applies to all staff.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | DVS details + Read more ...       |
|                                   |                                   |
|                                   | -   Identification documents must |
|                                   |     be verified for all customers |
|                                   |     in the DVS before processing  |
|                                   |     an application                |
|                                   | -   The DVS processing form must  |
|                                   |     be completed for each         |
|                                   |     document verification         |

+-----------------------------------+-----------------------------------+
| 3                                 | Customer fails DVS check and      |
|                                   | attends a service centre + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If a customer fails the DVS check |

Register a new customer

Table 3: how to create a new Medicare registration where no registration
for the customer already exists. Service Officers must do a detailed
search in CDMS before creating a new registration.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | New customer registration + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Is the customer being added to an |
|                                   | already existing Medicare card?   |
|                                   |                                   |
|                                   | Yes:                              |
|                                   |                                   |
|                                   | -   Select Register New Consumer  |
|                                   |     on the CDMS Home page         |
|                                   | -   The Registration Type screen  |
|                                   |     displays                      |
|                                   | -   Add card number to the ID     |
|                                   |     Number field                  |
|                                   | -   Select Get Details            |
|                                   | -   Make sure the results         |
|                                   |     displayed are for the correct |
|                                   |     customer                      |
|                                   | -   Select New Consumer           |
|                                   | -   The Search for Consumer       |
|                                   |     (Register) screen displays    |
|                                   |                                   |
|                                   | No:                               |
|                                   |                                   |
|                                   | -   Select Register New Consumer  |
|                                   |     on the CDMS Home page         |
|                                   | -   The Registration Type screen  |
|                                   |     displays                      |
|                                   | -   Select New Consumer           |
|                                   | -   The Search for Consumer       |
|                                   |     (Register) screen displays    |
+-----------------------------------+-----------------------------------+
| 2                                 | Enter customer's details + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Service Officers must complete    |
|                                   | Table 1 before registering a new  |
|                                   | customer.                         |
|                                   |                                   |
|                                   | Select the Immigration Data       |
|                                   | Search (tick will be displayed)   |
|                                   |                                   |
|                                   | Key details for the customer in   |
|                                   | the following fields:             |
|                                   |                                   |
|                                   | -   Family Name (mandatory) - add |
|                                   |     only name here                |
|                                   | -   First name (initial)          |
|                                   | -   Gender (mandatory)            |
|                                   | -   Date of Birth (mandatory)     |
|                                   | -   Postcode or Postcode range    |
|                                   |     (leave field as default, for  |
|                                   |     example 0001-9999)            |
|                                   |                                   |
|                                   | Select Search                     |
|                                   |                                   |
|                                   | Warning message 01875 will        |
|                                   | display if the person registering |
|                                   | is over 12 months old. Select Yes |
|                                   | to continue.                      |
|                                   |                                   |
|                                   | Search result displays:           |
|                                   |                                   |
|                                   | -   matched consumers             |
|                                   | -   exact or possible immigration |
|                                   |     clients on the Immigration    |
|                                   |     Clients tab                   |
|                                   |                                   |
|                                   | On the Immigration Clients tab,   |
|                                   | where the:                        |
|                                   |                                   |
|                                   | -   Etm (Entitlement) is a dash   |
|                                   |     (-) and Client Status is      |
|                                   |     Lodged, relationship          |
|                                   |     documents may be required     |
|                                   | -   Client Status is Refused      |
|                                   |     further documentation is      |
|                                   |     needed                        |
|                                   |                                   |
|                                   | Note: see Document types for      |
|                                   | Medicare eligibility and          |
|                                   | enrolment on the Resources page   |
|                                   | for the visa type for more        |
|                                   | details.                          |
|                                   |                                   |
|                                   | Follow the instructions in all    |
|                                   | warning messages.                 |
+-----------------------------------+-----------------------------------+
| 3                                 | Determine possible matches to     |
|                                   | customer search + Read more ...   |
|                                   |                                   |
|                                   | If no records found:              |
|                                   |                                   |
|                                   | -   A message displays indicating |
|                                   |     that no records were found    |
|                                   | -   Select OK to acknowledge the  |
|                                   |     message                       |
|                                   | -   Go to Step 4                  |
|                                   |                                   |
|                                   | If possible matches found:        |
|                                   |                                   |
|                                   | -   Consumer details display a    |
|                                   |     possible match of a customer  |
|                                   | -   Select the person by using    |
|                                   |     the radio button and select   |
|                                   |     Add to list                   |
|                                   | -   The Registration Type screen  |
|                                   |     displays                      |
|                                   |                                   |
|                                   | Asn, Curr Etm and Curr Etm Start  |
|                                   | Date fields will be blank if      |
|                                   | there is no Medicare entitlement  |
|                                   | recorded.                         |
|                                   |                                   |
|                                   | Note:                             |
|                                   |                                   |
|                                   | check the customers previously    |
|                                   | recorded personal details. For    |
|                                   | example address, contact details  |
|                                   | and bank details. Update as       |
|                                   | necessary before registering the  |
|                                   | customer entitlement              |
|                                   |                                   |
|                                   | if there is a system issue        |
|                                   | getting the consumer added to     |
|                                   | list:                             |
|                                   |                                   |
|                                   | -   register the customer’s       |
|                                   |     entitlement separately, on    |
|                                   |     the same record located       |
|                                   | -   repeat Steps 1 - 3, adding    |
|                                   |     the customer to the group     |
|                                   |     card                          |
|                                   | -   check all customer details    |
|                                   |     are added to the record       |
|                                   | -   contact Local Peer Support    |
|                                   |     (LPS) for help if needed      |
|                                   |                                   |
|                                   | See Table 4 to Register           |
|                                   | customer's Medicare entitlement.  |
+-----------------------------------+-----------------------------------+
| 4                                 | Record customer's details + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | -   Select New Consumer           |
|                                   |                                   |
|                                   | The Record New Consumer screen    |
|                                   | displays. The personal details    |
|                                   | are filled with any search        |
|                                   | details previously entered.       |
|                                   | Complete the following fields:    |
|                                   |                                   |
|                                   | -   Title                         |
|                                   |     Note: title not required for  |
|                                   |     children under 14 years of    |
|                                   |     age                           |
|                                   | -   Only Name Indicator (select   |
|                                   |     if customer is registering    |
|                                   |     with only one name)           |
|                                   | -   Second Name                   |
|                                   | -   Gender (mandatory)            |
|                                   | -   Multiple Birth Indicator      |
|                                   |     (select if customer is part   |
|                                   |     of a multiple birth)          |
|                                   | -   Aboriginal or Torres Strait   |
|                                   |     Islander Status (mandatory)   |
+-----------------------------------+-----------------------------------+
| 5                                 | Address or contact details + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | -   Select Address tab            |
|                                   | -   Select Address Source from    |
|                                   |     the dropdown box              |
|                                   | -   Key locality or postcode      |
|                                   | -   Select the locality from      |
|                                   |     dropdown box                  |
|                                   | -   Start to key the number and   |
|                                   |     street name on the Address    |
|                                   |     line                          |
|                                   | -   Select the address from the   |
|                                   |     dropdown box                  |
|                                   |                                   |
|                                   | Note:                             |
|                                   |                                   |
|                                   | -   Only add a residential        |
|                                   |     address if different from the |
|                                   |     postal address                |
|                                   | -   Do not record residential     |
|                                   |     addresses for children under  |
|                                   |     15 years                      |
|                                   | -   There is no need to enter     |
|                                   |     both a mailing and a          |
|                                   |     residential address in CDMS.  |
|                                   |     If the customer has only      |
|                                   |     provided a mailing (postal)   |
|                                   |     address such as a PO Box,     |
|                                   |     this is entered in CDMS as    |
|                                   |     the mailing address. See      |
|                                   |     Change of address for         |
|                                   |     Medicare enrolments for more  |
|                                   |     details                       |
+-----------------------------------+-----------------------------------+
| 6                                 | Record customer's contact         |
|                                   | details + Read more ...           |
|                                   |                                   |
|                                   | -   Select Contact details tab    |
|                                   | -   Key the customer's contact    |
|                                   |     details that are available    |
|                                   |     and any comments, key phone   |
|                                   |     numbers without any spacing   |
|                                   | -   Select an option from the     |
|                                   |     dropdown menu in the Source   |
|                                   |     field for any relevant        |
|                                   |     contact details keyed         |
|                                   |                                   |
|                                   | Note: do not record contact       |
|                                   | details for persons under 15      |
|                                   | years.                            |
+-----------------------------------+-----------------------------------+
| 7                                 | Record customer's bank details +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Select Banking details tab        |
|                                   |                                   |
|                                   | Key:                              |
|                                   |                                   |
|                                   | -   BSB number                    |
|                                   | -   account number                |
|                                   | -   account name (key 'and', do   |
|                                   |     not use ampersand '&')        |
|                                   |                                   |
|                                   | Select Notification Source from   |
|                                   | dropdown box                      |
|                                   |                                   |
|                                   | Note: Account start date field    |
|                                   | defaults to today's date and      |
|                                   | cannot be changed.                |
|                                   |                                   |
|                                   | See Store, confirm and end date   |
|                                   | bank account details for Medicare |
|                                   | EFT payments for procedures. Do   |
|                                   | not record bank details for       |
|                                   | persons under 14 years.           |
+-----------------------------------+-----------------------------------+
| 8                                 | Complete registration + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Scroll down and select Validate   |
|                                   |                                   |
|                                   | Action any warning messages and   |
|                                   | Validate                          |
|                                   |                                   |
|                                   | Select either:                    |
|                                   |                                   |
|                                   | -   Change to amend the details   |
|                                   |     if required                   |
|                                   | -   Update to confirm the details |
|                                   |     displayed. A message displays |
|                                   |     confirming that a consumer    |
|                                   |     record has been created       |
|                                   |                                   |
|                                   | Select OK to acknowledge the      |
|                                   | message                           |
|                                   |                                   |
|                                   | The Registration Type screen is   |
|                                   | displayed with the customer's     |
|                                   | details and customer ID           |
|                                   |                                   |
|                                   | If the details are correct,       |
|                                   | select Update and go to Table 4   |
